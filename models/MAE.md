# MAE (Masked Autoencoder)

## 基本信息

- **论文**: [Masked Autoencoders Are Scalable Vision Learners](https://arxiv.org/abs/2111.06377)
- **作者**: Kaiming He, Xinlei Chen, Saining Xie, Yanghao Li, Piotr Dollár, Ross Girshick (Meta)
- **发表**: CVPR 2022

## 创新点

1. **高掩码率 (75%)**: 比 NLP 的 MLM（15%）高得多，强制模型学习全局语义
2. **非对称编码器-解码器**: 编码器只处理可见 patch，轻量解码器重建像素，大幅降低计算量
3. **简单有效的像素重建**: 直接回归像素值，无需离散编码或复杂目标
4. **与 NLP 掩码预训练的深层类比**: 证明 MLM 范式可成功迁移到视觉领域

## 核心原理

### 掩码策略

- 将图像分成 16×16 的 patch
- 随机掩码 75% 的 patch（只保留 25% 可见）
- 可见 patch 数越少，计算量越少，学习难度越大

### 非对称编码器-解码器

- **编码器 (Encoder)**: 仅处理可见 patch（25%），使用 ViT
- **解码器 (Decoder)**: 处理全部 patch（可见 + 掩码），轻量级 Transformer

### 工作原理

1. 图像分割为 patch
2. 随机掩码 75% 的 patch
3. 仅编码可见 patch
4. 将编码结果 + 掩码 token（可学习）输入解码器
5. 解码器预测每个掩码 patch 的像素值
6. 仅计算掩码位置的 MSE 损失

## 预训练方法

### 核心思想：遮住 75%，让模型"脑补"缺失的画面

MAE 的预训练思路极其简单：**把一张图 75% 的区域遮挡住，只给模型看到剩下的 25%，然后让它把整张图重建出来**。这就像一个"极端的拼图游戏"——大部分碎片都被拿走了，你只能靠少量的碎片和全局的理解能力来还原画面。

> 这本质上和 BERT 的"完形填空"一样，只不过 BERT 是文字的填空，MAE 是像素的填空。区别在于：BERT 只遮 15%，MAE 遮高达 75%。

### 训练流水线（Step by Step）

#### Step 1 — 图像分块和掩码

将输入图像（如 224×224）分割为 $14 \times 14 = 196$ 个 $16 \times 16$ 的 patch。

**随机掩码 75% 的 patch**（约 147 个），只保留 25% 的可见 patch（约 49 个）。

```
完整图像: [P1][P2][P3]...[P196]
              ↓ 随机掩码 75%
可见 patch: [P1]    [P3]    ...[P195]  (49个)
掩码 patch:    [P2]    [P4]...[P196]  (147个)
```

> **关键设计**：掩码是**随机**的，没有任何结构（不像一些方法掩码整行/整列/整块）。这让模型无法利用空间连续性来"作弊"，必须学习真正的全局理解。

#### Step 2 — 编码器：只处理可见 patch

**非对称设计（MAE 最聪明的地方）**：编码器（ViT）**只处理可见的 49 个 patch**，忽略被掩码的 147 个 patch。

```
可见 patch (49个) → 编码器 (ViT) → 49 个编码后的特征向量
```

这样做的巨大好处：
- 计算量仅为处理全部 196 个 patch 的 **25%**
- 减少的内存占用可以训练更大的模型
- 编码器只需要对"真实信息"建模

#### Step 3 — 解码器：重建缺失的 patch

将编码后的可见特征与**可学习的掩码 token**（$[MASK]$）拼接回完整的 196 个序列：

```
序列 = [编码后的可见特征, [MASK], 编码后的可见特征, [MASK], ...]
```

所有 196 个 token 加上位置编码后，送入**轻量解码器**（仅 8 层 Transformer，隐藏维度 512）：

```
完整序列 (196个token) → 轻量解码器 → 196 个重建后的 patch
```

解码器对每个掩码位置预测其像素值（$16 \times 16 \times 3 = 768$ 维向量）。

#### Step 4 — 像素重建损失

**仅在掩码位置上计算损失**——可见 patch 不参与损失计算：

$$\mathcal{L}_{\text{MAE}} = \frac{1}{|M|} \sum_{i \in M} \| \hat{p}_i - p_i \|_2^2$$

其中 $M$ 是掩码位置的集合，$\hat{p}_i$ 是预测的像素值，$p_i$ 是真实像素值。

**为什么用归一化像素作为目标？**

直接回归原始像素（0-255）会导致模型过度关注亮度等简单统计特征。MAE 使用**patch 级别的归一化**——对每个被掩码的 patch，目标是其**归一化后的像素值**（减均值、除方差）。这让模型必须关注纹理和形状结构，而不能靠"猜平均颜色"偷懒。

#### Step 5 — 丢弃解码器，只保留编码器

预训练完成后，解码器被**完全丢弃**。编码器（ViT）直接用于下游任务——就像 BERT 预训练后丢弃 MLM 头。

### 为什么这样设计——深入理解每个选择

#### 1. 为什么掩码率是 75%？（而非 BERT 的 15%）

这是 MAE 最关键的发现之一：**图像和文本的信息密度不同**。

| 模态 | 信息密度 | 最优掩码率 | 原因 |
|------|---------|-----------|------|
| 文本 | 高 | 15% | 每个词携带很多语义，掩码太多无法恢复 |
| 图像 | 低 | **75%** | 像素高度冗余，相邻像素几乎相同，需要高掩码率强制全局推理 |

消融实验的惊人结果：

| 掩码率 | 微调 Top-1 (ViT-L) | 观察 |
|-------|-------------------|------|
| 15% | 较差 | 太简单，模型只学局部插值 |
| 50% | 中等 | 仍然能从周围像素"猜测" |
| **75%** | **最好（84.9%）** | 必须理解全局语义才能重建 |
| 85% | 略降 | 可见信息太少 |
| 95% | 仍较好（~84%） | 极其稀疏的信息仍然有用 |

> 75% 掩码率意味着模型能看到的只有原图的 **1/4**。在这种情况下要重建整张图，模型不得不学习物体的形状、结构、纹理等高级语义。

#### 2. 为什么编码器不处理掩码 token？

这是 MAE 与 BERT 的另一个重要区别：

| 方法 | 编码器处理的内容 | 设计选择 |
|------|---------------|---------|
| BERT | 完整序列（含 [MASK]） | 编码器需要看到 [MASK] 来学习其语义 |
| **MAE** | 仅可见 patch | 编码器不处理 [MASK]，节省计算 |

BERT 中 [MASK] 是一个特殊的词——编码器需要理解"这里有个空白"来学习填空。但在视觉中，像素是连续的，不存在"空白像素"的概念。编码器不需要知道"哪些位置被遮住了"——它只需要从可见部分提取特征，然后把重建任务交给解码器。

**这一非对称设计使 MAE 的训练速度比 BERT 风格的视觉掩码模型快 3× 以上**。

#### 3. 为什么解码器可以很轻量？

解码器的任务是像素重建——一个比语义理解"低级"的任务。因此可以用较小的解码器（8 层、512 维）来承担。消融实验表明：

| 解码器深度 | 微调 Top-1 | 说明 |
|-----------|-----------|------|
| 1 层 | 较差 | 解码能力不足 |
| 4 层 | 良好 | 基本够用 |
| 8 层 | **最好** | 标准选择 |
| 12 层 | 与 8 层相似 | 边际收益递减 |

解码器宽度也是 512 维就足够——再大没有额外收益。这说明像素重建任务本身不需要太复杂的架构，轻量解码器已经可以很好地"利用"编码器提取的高质量特征。

#### 4. 为什么用 MSE 而不是交叉熵？

NLP 的 MLM 使用交叉熵（因为目标是离散的词），但图像像素是连续值。MAE 对比了两种做法：

| 重建目标 | 方法 | 效果 |
|---------|------|------|
| 离散 token（dVAE） | 交叉熵 | 类似 BEiT，需要额外的 tokenizer |
| **归一化像素** | MSE | **更简单、效果更好** |

直接用 MSE 回归像素值是 MAE 的优雅之处——不需要预训练一个图像 tokenizer（如 dVAE），端到端的像素重建就足够了。

#### 5. 为什么不需要数据增强？

对比学习方法（SimCLR、MoCo、BYOL）严重依赖数据增强（裁剪、色彩抖动等）。MAE 几乎不需要数据增强——仅使用随机裁剪（random resized crop），不需要色彩抖动、高斯模糊等。

原因：**掩码本身就是最强的数据增强**。75% 的随机掩码每一次都不一样，模型被迫从任意稀疏的可见区域重建图像，这种"dropout-like"的随机性已经提供了足够的正则化和多样性。

### 预训练性能

| 方法 | 编码器 | 线性探测 | 微调 |
|------|--------|---------|------|
| MAE | ViT-B | 68.0% | 83.6% |
| DINO | ViT-B | **78.2%** | 82.8% |
| MoCo v3 | ViT-B | 76.7% | 83.2% |
| BEiT | ViT-B | 37.6% | 83.2% |
| MAE | ViT-L | 75.8% | **85.9%** |
| MAE | ViT-H | 77.0% | **86.9%** |

**关键观察**：MAE 的线性探测（linear probing）性能不是最好的（DINO 更好），但**微调（fine-tuning）性能是最好的**。这说明 MAE 学习到的特征更"非线性"——信息分散在各层中，需要微调才能充分释放。这也意味着 MAE 的表征更适合需要全面微调的下游任务（如目标检测、语义分割）。

### 迁移学习的实用价值

MAE 的意义远不止于一个预训练方法：

1. **证明了视觉 MLM 范式的可行性**：NLP 中 BERT 的掩码预训练范式终于被成功移植到视觉
2. **计算效率的革命**：非对称设计让 ViT 预训练速度提升了 3×+，使得资源有限的团队也能做 ViT 预训练
3. **对稠密预测任务友好**：MAE 预训练的模型在检测和分割上表现特别出色（保留了更多局部细节信息）
4. **启发了 VideoMAE、Point-BERT、AudioMAE** 等一系列跨模态掩码预训练工作
