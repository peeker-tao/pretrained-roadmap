# ClimaX

## 基本信息

- **论文**: [ClimaX: A Foundation Model for Weather and Climate](https://arxiv.org/abs/2301.10343)
- **作者**: Tung Nguyen et al. (UCLA / Microsoft)
- **发表**: ICML 2023

## 创新点

1. **天气与气候基础模型**: 统一的 Transformer 模型处理多种天气和气候任务
2. **多源数据融合**: 卫星、观测站、模型输出的统一处理
3. **多任务预训练**: 在不同分辨率、不同变量的数据上联合预训练

## 核心原理

### 架构设计

1. **变量 Tokenization**: 每个气象变量作为一个 token
2. **空间编码**: 经纬度位置编码
3. **Transformer 编码器**: 处理时空序列

### 多任务训练

- 不同分辨率的数据（粗粒度气候 + 细粒度天气）
- 不同变量组合（温度、降水、风速等）
- 不同任务（预报、降尺度、偏差校正）

## 预训练方法

### 核心思想：就像 GPT 学会了"理解通用文本"然后适配各种 NLP 任务，ClimaX 的目标是学会"理解通用天气和气候"——一个模型，在多种数据源（卫星、观测站、气候模型）和多种任务（预报、降尺度、偏差校正）上预训练后，能灵活迁移到各种天气应用

ClimaX 是第一个将 **Foundation Model 范式引入气象学** 的工作。它不针对单一预报任务优化，而是在多分辨率和多变量数据上联合预训练，学到一个通用的"天气表征"。

> ClimaX = 变量 Tokenization（每个气象变量一个 token）+ 时空位置编码 + Vision Transformer + 多任务多分辨率预训练。一个模型覆盖全球天气预报、区域降尺度、气候预测、偏差校正。

### 训练流水线（Step by Step）

#### Step 1 — 气象变量 Tokenization

ClimaX 将每个气象变量编码为一个独立的 token（与 ViT 的 patch 不同——整个变量是一个 token）：

```text
输入数据（全球网格 721×1440）:
  温度场: 721×1440 → 1 个 "温度 token"
  气压场: 721×1440 → 1 个 "气压 token"
  风速场: 721×1440 → 1 个 "风速 token"
  湿度场: 721×1440 → 1 个 "湿度 token"
  ...
  N 个变量 → N 个 token

每个 token: 通过变量特定编码器映射为 d 维向量
```

**变量聚合 + 空间编码**：

| 组件 | 说明 |
|------|------|
| 变量嵌入 | 气候变量 → d 维嵌入（区分温度、气压等） |
| 空间编码 | 经纬度 → 位置嵌入 |
| 时间编码 | 年/月/日/时 → 时间嵌入 |
| 分辨率编码 | 数据源的网格分辨率 → 分辨率嵌入 |

> Token 设计：每个气象场是一个 token，而非每个网格点——这使得不同分辨率的数据可以混合训练（网格点数不同，但变量 token 数相同）。

#### Step 2 — Vision Transformer 的时空处理

```text
N 个变量 token + 时空位置编码
  ↓
Transformer Encoder (多轮 Self-Attention)
  ↓
变量 token 之间交换信息
  - "温度" token 关注"气压" token（温度影响气压）
  - "湿度" token 关注"风速" token（风输送水汽）
  ↓
增强的变量 token → 预测头 → 各气象场的未来值
```

**为什么用 Variable-as-Token？**
- 标准 ViT（grid-based patch）依赖固定分辨率 → 不同数据集必须重新切 patch
- Variable-as-Token → 任何分辨率的输入都映射到同一套变量 token → 多分辨率混合训练成为可能

#### Step 3 — 多任务多分辨率联合预训练

这是 ClimaX 作为"基础模型"的核心机制：

| 数据源 | 分辨率 | 变量数 | 任务 |
|--------|--------|--------|------|
| ERA5（再分析） | 0.25° (~25km) | 67 个气象变量 | 天气预报 |
| CMIP6（气候模型） | 1-2° (~100-200km) | 多个模型输出 | 气候预测 |
| PRISM（观测） | 4km（仅美国） | 温度、降水 | 降尺度 |

**联合训练的挑战**：
1. ERA5 有 721×1440 网格，CMIP6 只有 180×360 → 分辨率不同
2. 不同数据源有不同的变量子集 → 变量数不同
3. ERA5 是"历史真实"（再分析），CMIP6 是"模型模拟" → 分布不同

**ClimaX 的解决方案**：

```text
每步随机选择:
1. 数据源: ERA5 / CMIP6 / PRISM
2. 时间窗口: 随机历史时段
3. 变量子集: 随机 mask 部分变量
4. 分辨率: 数据源的原生分辨率

→ "任何数据、任何变量、任何分辨率"都可以输入
```

这种随机采样 = 极端的数据增广 → 模型学到了**变量之间、分辨率之间、数据源之间的共性物理规律**。

#### Step 4 — 预测头与损失

ClimaX 是生成式预报模型：

```text
输入: 当前时刻的气象场 X_t
预测: 未来多个时刻 X_{t+6h}, X_{t+12h}, ..., X_{t+48h}

损失: $$\mathcal{L} = \sum_{t} \|X_t^{\text{pred}} - X_t^{\text{true}}\|^2$$
```

| 预报时效 | 典型应用 |
|---------|---------|
| 6h-24h | 短期预报（风暴、暴雨） |
| 24h-7d | 中期预报（台风路径） |
| 7d-30d | 次季节预报 |
| 1个月+ | 气候预测（季节预报） |

#### Step 5 — 完整训练与下游迁移

```text
预训练阶段: 多源数据混合（ERA5 + CMIP6 + PRISM）

下游微调:
  任务 1: 全球中期预报 → 冻结 backbone，只微调预测头
  任务 2: 区域降尺度 → 微调 backbone + 预测头
  任务 3: 气候预测 → 微调 backbone + 预测头
  任务 4: 偏差校正 → 微调（CMIP6 → ERA5 风格）
```

### ClimaX vs 传统 NWP

| 维度 | ECMWF IFS（NWP） | ClimaX |
|------|-----------------|--------|
| 方法 | 物理方程求解 | **数据驱动** |
| 推理速度 | 数小时（超算） | **<1 秒（单 GPU）** |
| 分辨率 | 9km | 25km（可降尺度） |
| 物理约束 | 硬约束（方程） | 软约束（数据习得） |
| 多任务 | 需重写模型 | **统一框架** |

### 预训练的实用价值

1. **气象学的基础模型范式**：一个模型覆盖多种天气/气候任务
2. **推理速度的变革**：<1 秒 vs 数小时 → 实时预警成为可能
3. **多分辨率兼容**：4km 到 200km 的数据都可以输入
4. **物理知识的隐性学习**：Transformer 的注意力模式中自然地出现了大气波的物理特征
5. **气象 AI 的通用平台**：类似 GPT 在 NLP 中的角色——下游任务只需微调
