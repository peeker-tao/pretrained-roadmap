# QLoRA (Quantized Low-Rank Adaptation)

## 基本信息

- **论文**: [QLoRA: Efficient Finetuning of Quantized Language Models](https://arxiv.org/abs/2305.14314)
- **作者**: Tim Dettmers et al. (UW)
- **发表**: NeurIPS 2023

## 创新点

1. **4-bit 量化 + LoRA**: 将预训练模型量化为 4-bit（NF4），再使用 LoRA 微调
2. **双量化 (Double Quantization)**: 进一步压缩量化常数，节省 0.4 bit/参数
3. **单卡 65B**: 可在单块 48GB GPU 上微调 65B 参数模型
4. **NF4 数据类型**: 正态分布 4-bit 量化，信息论最优

## 核心原理

### NF4 (NormalFloat4)

- 基于分位数量化：权重按正态分布的分位数划分
- 比均匀量化更适合神经网络权重分布

### 双量化

- 第一次量化：FP32 → NF4（存储量化权重）
- 第二次量化：量化常数（scale factors）→ FP8
- 额外节省 ~0.4 bit/参数

### 分页优化器

- 将优化器状态（momentum, variance）offload 到 CPU
- 使用统一内存分页，避免 OOM

## 适配方法

### 核心思想：LoRA 已经很省了？我们让它再省 4×——把模型先"压缩"到 4-bit

LoRA 只训练极少参数（< 1%），但预训练模型本身仍然占用大量显存——一个 65B 的模型即使在推理时也需要约 130GB（FP16）。QLoRA 要解决的问题是：**能不能连模型本身也"瘦身"，让单张 48GB GPU 就能微调 65B 模型？**

> QLoRA = 4-bit NormalFloat 量化 + 双量化 + 分页优化器 + LoRA。它证明了：用极致压缩的 4-bit 精度存储预训练模型 + LoRA 微调，几乎不会损失性能。

### QLoRA 三层技术栈

#### 第一层 — NF4（NormalFloat 4-bit）量化

标准 INT4 量化将值域均匀划分为 16 个区间。但神经网络的权重近似**正态分布**（均值为 0，大部分值集中在均值附近）。均匀量化在小值区域浪费了大量精度。

**NF4 的解决方案**：按正态分布的分位数划分区间——概率密度高的区域分得更细，密度低的区域分得更粗。

```text
INT4 (均匀):   [-1.0]  ...等距区间...  [1.0]  → 信息利用率低
NF4 (分位数):  [-1.0]  | | || ||| |  [1.0]  → 信息利用率高
                     密集区多分区，稀疏区少分区
```

数学上，NF4 是信息论最优的——对于正态分布的数据，它最大化了量化后的熵。

**量化过程**：
1. 对权重矩阵的每一块（block-wise），计算其最大值和最小值
2. 将 $[-1, 1]$ 按照正态分布 $N(0,1)$ 的分位数划分为 $2^4 = 16$ 个区间
3. 将每个权重值映射到最近的 NF4 值
4. 存储量化后的 4-bit 值 + 每个块的量化常数（scale）

#### 第二层 — 双量化（Double Quantization）

量化常数的存储也是一个开销——对于 64 个参数一个块（block size 64），量化常数占用 FP32（4 bytes）。

**双量化**：对量化常数再进行一次量化（FP32 → FP8）：

| 层级 | 数据 | 精度 | 每参数开销 |
|------|------|------|-----------|
| 原始权重 | FP32 权重 | — | 32 bit |
| 第一次量化 | 权重 → NF4 | 4 bit | 4 bit + 常数开销 |
| 第二次量化 | 量化常数 → FP8 | 8 bit | 额外节省 ~0.4 bit/参数 |

> 双量化将总存储从 ~4.5 bit/参数降低到 ~4.1 bit/参数——虽然只节省了 0.4 bit，但对 65B 模型来说就是 ~3GB 的额外节省。

#### 第三层 — 分页优化器（Paged Optimizer）

LoRA 只需要优化极少参数，但优化器状态（AdamW 的 momentum 和 variance）仍然需要存储。当 GPU 显存紧张时，QLoRA 使用**统一内存分页**：

- 优化器状态放在 CPU 内存中
- 仅在需要时通过 CUDA unified memory 分页调入 GPU
- 自动处理 out-of-memory 错误——通过内存分页而非直接崩溃

### QLoRA 训练流程

```text
1. 加载预训练模型 → NF4 量化（冻结）
2. 双量化量化常数
3. 添加 LoRA 适配器（FP16/BF16，可训练）
4. 正向传播：NF4 权重 → 反量化 → FP16 → 计算
5. 反向传播：仅计算 LoRA 梯度
6. 优化器状态通过分页管理
```

**关键**：正向传播时，NF4 权重被**动态反量化**为 FP16/BF16 进行计算。梯度只传回 LoRA 参数——量化的预训练权重不参与梯度计算。

### QLoRA vs LoRA vs 全量微调

| 维度 | 全量微调 | LoRA (FP16) | QLoRA (NF4) |
|------|---------|------------|------------|
| LLaMA-7B 显存 | > 56GB | ~16GB | **~8GB** |
| LLaMA-13B 显存 | > 100GB | ~28GB | **~14GB** |
| LLaMA-65B 显存 | 无法单卡 | ~64GB | **~42GB** |
| 性能差距 vs 全量 | 基线 | < 0.5% | **< 1%** |
| 训练速度 | 1× | 1× | ~0.7×（量化反量化开销） |

### 详细配置

| 参数 | 典型值 | 说明 |
|------|--------|------|
| 量化格式 | NF4 | 正态分布最优量化 |
| Block Size | 64 | 量化粒度 |

### 预训练的实用价值

1. **大模型微调的"终极民主化"**：单张 24GB RTX 3090/4090 可微调 LLaMA-7B/13B
2. **NF4 数据格式的创新**：信息论最优的 4-bit 量化方法
3. **与 LoRA 完美互补**：QLoRA 压缩存储 + LoRA 压缩训练 = 极致效率
4. **Guanaco 模型的训练方法**：QLoRA 论文中训练的 Guanaco-65B 在单卡上达到 ChatGPT 99.3% 的表现
5. **成为消费级 GPU 微调的标准工具**：HuggingFace PEFT 中的默认推荐方法
