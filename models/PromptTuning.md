# Prompt Tuning

## 基本信息

- **论文**: [The Power of Scale for Parameter-Efficient Prompt Tuning](https://arxiv.org/abs/2104.08691)
- **作者**: Brian Lester et al. (Google)
- **发表**: EMNLP 2021

## 创新点

1. **仅调整输入层软提示**: 比 Prefix Tuning 更简单，仅修改输入嵌入
2. **随规模提升**: 模型越大，Prompt Tuning 越接近全量微调
3. **单组参数/任务**: 仅需存储几百个浮点数即可切换任务

## 核心原理

### 软提示 (Soft Prompt)

在嵌入层前拼接可训练的嵌入向量：

$$[P_1, P_2, ..., P_m, X_1, X_2, ..., X_n]$$

- $P_i$: 可学习的 soft prompt token（$m$ 通常 5-100）
- $X_i$: 原始输入 token

### 规模效应

| 模型大小 | Prompt Tuning vs 全量微调 |
|---------|--------------------------|
| ~100M | 差距较大 |
| ~1B | 接近 |
| ~10B+ | **匹敌** |

### 与其他 Prompt 方法的对比

| 方法 | 修改范围 | 参数量 |
|------|---------|--------|
| Prompt Tuning | 输入嵌入 | 极低 |
| Prefix Tuning | 每层 K-V | 低 |
| P-Tuning v2 | 每层输入 | 低 |

## 适配方法

### 核心思想：模型越大，越"听话"——大模型只需要在输入端加几个可训练的"软提示词"就能完美适配

Prompt Tuning 将 Prefix Tuning 的思想推到极致：**不需要修改每一层，只在输入端拼接几个可学习的嵌入向量**。这听起来太简单了——只在输入层加几十个可训练参数，真的能匹敌全量微调吗？答案出人意料：**当模型足够大（10B+）时，可以**。

> Prompt Tuning = 输入端拼接 $m$ 个可训练嵌入 + 冻结所有模型参数 + 仅反向传播到这些嵌入。参数量仅为 Prefix Tuning 的 1/10，全量微调的 0.001%。

### 训练流水线（Step by Step）

#### Step 1 — 软提示的构成

输入序列变为：

$$[P_1, P_2, ..., P_m, X_1, X_2, ..., X_n]$$

其中每个 $P_i \in \mathbb{R}^d$ 是一个可训练的嵌入向量（$d$ 是模型隐藏维度）。

**与离散 Prompt 的本质区别**：

| | 离散 Prompt | Prompt Tuning |
|------|-----------|--------------|
| 形式 | "Translate to French:" | $P_1, P_2, ..., P_m$（连续向量） |
| 对应词汇 | 有具体 token | **无对应词汇——纯信号** |
| 可微分 | ✗（离散→不可微） | **✓（连续→可微）** |
| 优化方式 | 人工搜索 | **梯度下降** |
| 存储 | 几个 token | $m \times d$ 个浮点数 |

> 这些软提示向量不映射到任何实际词汇——它们是"纯信号"，直接作用于模型的嵌入空间，告诉模型"往这个方向走"。

#### Step 2 — 规模效应（"The Power of Scale"）

这是 Prompt Tuning 最惊人的发现：**模型越大，软提示越有效**。

| 模型大小 | Prompt Tuning vs 全量微调 | 差距 |
|---------|--------------------------|------|
| Small (< 100M) | 远不如 | > 10% |
| Medium (300M-1B) | 接近但不敌 | 3-5% |
| Large (1B-10B) | 接近 | 1-2% |
| **XL (10B+)** | **匹敌** | **< 0.5%** |

**为什么大模型更"听话"？**

大模型在预训练阶段见过更多模式，拥有更丰富的内部表征空间。软提示只需要在这个空间中"选定一个方向"——空间越大，越容易找到通向任务最优解的方向。

> 类比：小模型像一个只懂基本单词的翻译器——你需要给它详细的指令（全量微调）；大模型像一个语言学家——给它一个关键词（软提示），它就理解你想要什么。

#### Step 3 — 初始化策略

软提示的初始化对训练稳定性至关重要：

| 初始化方式 | 效果 | 说明 |
|-----------|------|------|
| 随机初始化 | 差 | 优化极不稳定 |
| 从词汇表采样 | 中 | 有语义锚定 |
| **从类别标签嵌入初始化** | **好** | 最强语义引导 |

**类别标签初始化**的具体做法：
- 如果任务是分类，用类别名称的嵌入来初始化软提示
- 例如情感分类 → 用"positive"、"negative"的嵌入
- 翻译 → 用目标语言的 token 嵌入

> 这就像给优化器一个"好的起点"——与其从零开始搜索，不如从"接近正确答案"的地方出发。

#### Step 4 — 训练过程

```text
1. 冻结预训练模型所有参数
2. 初始化 m 个软提示嵌入
3. 每个 batch：
   a. 将软提示拼接在输入嵌入前
   b. 通过冻结的模型进行前向传播
   c. 计算任务损失
   d. 反向传播仅更新软提示嵌入（梯度不流入模型）
4. 保存最终的软提示嵌入
```

**训练技巧**：
- 使用 standard Adam（而非 AdamW——不需要权重衰减）
- 学习率通常比全量微调高 10-100×
- 因为只训练极少量参数，可以承受更高的学习率

#### Step 5 — 多任务部署

每个任务只需存储 $m \times d$ 个浮点数——对于 $m=100, d=4096$（LLaMA-7B），仅约 1.6MB。切换任务时只需：

```text
任务 A → 加载 soft_prompt_A.pt（1.6MB）
任务 B → 加载 soft_prompt_B.pt（1.6MB）
```

> 全量微调需要为每个任务存储整个 7B-175B 模型（14-350GB）。Prompt Tuning 的存储成本降低了 10,000× 以上。

### 为什么输入端就足够？（vs Prefix Tuning 的每层都加）

| 方法 | 修改位置 | 信息流 | 效果 |
|------|---------|--------|------|
| Prompt Tuning | 仅输入 | 通过 self-attention 传播到所有层 | 靠模型自身传播 |
| Prefix Tuning | 每层 K-V | 直接干预每层注意力 | 更直接，参数更多 |

Prompt Tuning 的洞察：在大模型中，输入端的信号通过 self-attention **自然会传播到每一层**——不需要在每层都插入额外的向量。Self-attention 机制天然地将输入信号"扩散"到所有位置和所有层。

### 详细配置

| 参数 | 典型值 | 说明 |
|------|--------|------|
| 提示长度 $m$ | 5-100 | 任务越难，需要越长 |
| 初始化 | 类别标签嵌入 | 强烈推荐 |
| 优化器 | Adam | 无权重衰减 |
| 学习率 | 0.3-3.0 | 比全量微调高 10-100× |
| 可训练参数 | < 0.01% | 仅 $m \times d$ |
| 模型要求 | ≥ 1B（最好 10B+） | 规模效应 |

### 预训练的实用价值

1. **极简主义的极致**：仅训练输入嵌入，达到全量微调性能——"规模即一切"
2. **最低存储成本**：每个任务仅需 KB-MB 级别
3. **规模效应的理论贡献**："The Power of Scale"不仅是经验观察，更是对大模型能力的深刻洞察
4. **软提示的通用性**：同一软提示可以跨不同输入实例使用——不是"每个例子一个提示"
5. **与离散 Prompt 的互补**：离散指令（如思维链）+ 软提示（任务适配）= 最佳效果
