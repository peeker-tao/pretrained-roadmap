# AudioMAE

## 基本信息

- **论文**: [AudioMAE: Masked Autoencoders are Efficient and Scalable Audio Learners](https://arxiv.org/abs/2207.06405)
- **作者**: Po-Yao Huang et al. (Meta)
- **发表**: Interspeech 2023

## 创新点

1. **音频掩码自编码器**: 将 MAE 范式应用于音频频谱图
2. **高掩码率**: 掩码 80% 的音频 patch 仍能有效重建
3. **简单高效**: 与 MAE 一致的极简设计

## 核心原理

### 频谱图掩码

1. 将音频频谱图分割为 patch
2. 随机掩码大部分 patch（80%）
3. 非对称编解码器重建被掩码的频谱 patch

### 与 MAE 的相似性

- 完全相同的架构设计（ViT + 轻量解码器）
- 仅将图像 patch 替换为频谱图 patch
- 重建目标为频谱图的像素值

## 预训练方法

### 核心思想：MAE 在图像上遮掉 75% 来重建像素——音频频谱图本质就是一张"声音图像"。AudioMAE 把音频频谱图当作图像来处理，遮掉 80% 的频谱 patch，让模型从"碎片"中重建完整声音。令人惊讶的是，遮掉 80% 后模型仍然能重建出清晰的声音——说明了音频信息的高度冗余性

AudioMAE 是 MAE 范式在音频领域的最直接迁移。它将音频频谱图视为"图像"，使用完全相同的非对称 Encoder-Decoder 设计：高掩码率（80%）+ ViT Encoder（仅处理可见 patch）+ 轻量 Decoder（重建所有 patch）+ MSE 像素回归。

> AudioMAE = MAE + 音频频谱图（Mel-Spectrogram）。改动极小——仅将图像 patch 替换为频谱图 patch，掩码率调整到 80%。在 AudioSet 上以更少的训练时间超越了对比学习方法。

### 训练流水线（Step by Step）

#### Step 1 — 音频 → 频谱图 Tokenization

**为什么用频谱图？**
- 原始音频波形（44.1kHz）是 1D 信号 → 时间序列太长（10 秒 = 441,000 个采样点）
- 频谱图将 1D 波形转换为 2D 图像（时间 × 频率） → 天然适配 ViT 的 patch 处理

```text
原始音频 (10 秒, 16kHz):
  波形: [0.001, -0.002, 0.003, ...]  (160K 个值)

Mel 频谱图:
  短时傅里叶变换 (STFT)
  → 时间×频率: 1024 × 128 = 131,072 个值
  → 类似灰度图像 (1024×128×1)

Patch Tokenization:
  Patch 大小: 16×16
  → 64 × 8 = 512 个 token
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 音频时长 | 10 秒 | AudioSet 标准 |
| 采样率 | 16kHz | 降采样 |
| Mel 频带 | 128 | 频率分辨率 |
| 时间帧 | 1024 | 时间分辨率 |
| Patch 大小 | 16×16 | 同 MAE |
| Token 数 | 512 | — |

#### Step 2 — 80% 掩码率

| 方法 | 模态 | 掩码率 |
|------|------|--------|
| MAE | 图像 | 75% |
| VideoMAE | 视频 | 90-95% |
| **AudioMAE** | **音频** | **80%** |

**为什么音频 80%？**

| 因素 | 图像 | 音频 |
|------|------|------|
| 信息冗余 | 中 | **高（声音有很强的时频相关性）** |
| Token 数 | 196 | 512（更多） |
| 最优掩码率 | 75% | **80%（经实验）** |

> 音频邻域信息重叠：相邻时间帧的频谱几乎相同（除非有快速变化），相邻频率的幅度也高度相关（谐波关系）。

#### Step 3 — 非对称 Encoder-Decoder

完全复刻 MAE 架构：

```text
Encoder (ViT-B, 仅处理可见 token):
  可见 token: 512 × 20% = 102 个
  12 层 Transformer
  只在 102 个 token 间计算 Self-Attention
  → 输出: 增强的 102 个 token 特征

Decoder (4 层, 处理所有 token):
  Encoder 输出 (102 token) + [MASK]*410
  4 层轻量 Transformer
  → 预测: 每个 token → 16×16 频谱块的值

损失: 仅对被遮区域计算 MSE
  L = Σ_{i∈masked} ||x_i_pred - x_i_true||²
```

#### Step 4 — 完整预训练

```text
1. 音频 → Mel 频谱图 (1024×128)
2. Patch 化 → 512 token (16×16 patch)
3. 随机 mask 80% → 410 个 token 被遮
4. 仅 102 个可见 token 进入 Encoder
5. Decoder 重建被遮部分
6. MSE 损失反向传播
```

| 参数 | AudioMAE-B | AudioMAE-L |
|------|----------|----------|
| 预训练数据 | AudioSet-2M | AudioSet-2M |
| 音频时长 | 10 秒 | 10 秒 |
| 采样率 | 16kHz | 16kHz |
| Mel 频带 | 128 | 128 |
| Patch 大小 | 16×16 | 16×16 |
| 掩码率 | 80% | 80% |
| Encoder 层数 | 12 | 24 |
| Decoder 层数 | 4 | 6 |
| 优化器 | AdamW | AdamW |
| 学习率 | 1.5e-4 | 1e-4 |
| Epoch | 100 | 100 |

#### Step 5 — 下游任务微调

| 任务 | 数据集 | AudioMAE 性能 |
|------|--------|-------------|
| 音频分类 | AudioSet | 47.3% mAP (B) |
| 音频事件检测 | AudioSet Strong | 取得 SOTA |
| 语音情感分类 | IEMOCAP | 超越 wav2vec2 |
| 环境声分类 | ESC-50 | 94.1% |

### AudioMAE vs 音频对比学习

| 维度 | 对比学习 (Audio CLIP-style) | AudioMAE |
|------|--------------------------|---------|
| 方法 | 正负例区分 | **掩码重建** |
| 数据需求 | 需大量负例 | **无需负例** |
| 训练效率 | 中（需大 batch） | **高（Encoder 仅处理 20% token）** |
| 监督类型 | 实例判别 | **像素回归** |
| 编码器质量 | 好 | **好（且训练更快）** |

### 预训练的实用价值

1. **MAE 在音频的完美适配**：证明了 MAE 的通用性——图像、视频、音频都可以用同一范式
2. **训练效率**：Encoder 只处理 20% token → 比对比学习快 2-3×
3. **音频自监督的新范式**：掩码重建超越了之前的对比学习方法
4. **频谱图 + ViT 的简洁性**：不需要为音频专门设计架构 → 降低了音频 AI 的门槛
5. **跨模态一致性**：图像/视频/音频共享同一预训练框架 → 多模态融合更自然
