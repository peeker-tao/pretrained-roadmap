# PaLM (Pathways Language Model)

## 基本信息

- **论文**: [PaLM: Scaling Language Modeling with Pathways](https://arxiv.org/abs/2204.02311)
- **作者**: Aakanksha Chowdhery et al. (Google)
- **发表**: JMLR 2023

## 创新点

1. **5400 亿参数**: 当时最大的稠密语言模型之一
2. **Pathways 系统**: 在 6144 个 TPU v4 芯片上实现高效分布式训练
3. **涌现能力**: 展示了大规模模型的推理和多任务能力

## 核心原理

### 架构

标准的仅解码器 Transformer 架构，使用 SwiGLU 激活、RoPE 位置编码、并行注意力/FFN 计算。

## 预训练方法

### 核心思想：用 6144 个 TPU 训练 5400 亿参数的"超级大脑"

PaLM 是 Google 在 2022 年展示的"极致规模工程"——5400 亿参数、7800 亿 token、6144 个 TPU v4。它的核心信息是：**当模型达到足够大的规模时，不需要任何特殊技巧，仅靠标准的下一个 token 预测就能涌现出惊人的推理和多任务能力**。

> PaLM = 标准 GPT 式自回归 + 极致的工程规模 + Pathways 分布式训练系统。它在 540B 参数量下展示了 chain-of-thought 推理、多语言翻译、代码解释等多种涌现能力。

### 训练流水线（Step by Step）

#### Step 1 — 预训练数据处理

PaLM 使用了 7800 亿 tokens 的多语言+代码混合数据：

| 数据源 | 占比 | 特点 |
|--------|------|------|
| 社交对话 | 50% | 多语言对话（提升对话能力） |
| 网页文档（过滤后） | 27% | 高质量网页内容 |
| 书籍 | 13% | 长文本、复杂叙述 |
| Wikipedia | 4% | 结构化知识 |
| 代码（GitHub） | 5% | 编程能力 |
| 新闻 | 1% | 事实性文本 |

**数据处理流程**：
1. **去重**：文档级和近似去重
2. **质量过滤**：基于困惑度和语言模型打分
3. **毒性过滤**：移除有害内容
4. **个人信息过滤**：移除 PII（个人身份信息）

> 注意 50% 是"社交对话"数据——这是 PaLM 能展示优秀对话和推理能力的重要原因。而 5% 的代码数据虽然占比不高，但却是代码能力的"催化剂"。

#### Step 2 — 模型架构

PaLM 使用标准的**仅解码器 Transformer**，但做了几个关键改进：

| 组件 | PaLM 选择 | 说明 |
|------|---------|------|
| 激活函数 | **SwiGLU** | 在 FFN 层使用，比 ReLU/GELU 更好 |
| 位置编码 | **RoPE** | 旋转位置编码，支持长序列外推 |
| 注意力/FFN | **并行计算** | 注意力和 FFN 并行（而非串行），节省 15% 训练时间 |
| LayerNorm | **无偏置项** | 仅缩放不偏移 |
| 嵌入矩阵 | 与输出层**共享权重** | 减少参数量 |

**并行计算的关键**：传统 Transformer 中，注意力和 FFN 是串行的——先做注意力，再做 FFN。PaLM 改为并行：

```text
传统: x → [Attention] → [FFN] → 输出
PaLM: x → [Attention] → 合并 → 输出
     x → [FFN]      → 合并 → 输出
```

这虽然增加了每步的 FLOPs，但因为 Attention 和 FFN 可以同时计算，**实际墙钟时间减少了约 15%**——在大规模训练中，这个节省非常可观。

#### Step 3 — 超大 Batch 的自回归训练

PaLM 的 batch size 设计非常讲究——采用**渐进式增长**策略：

| 训练阶段 | Batch Size | 原因 |
|---------|-----------|------|
| 初期（Warmup） | 小 batch | 学习率 warmup，避免梯度不稳定 |
| 中期 | 逐渐增长 | 利用 TPU 的计算能力 |
| 稳态 | 1952 序列 ≈ 3M tokens/batch | 最大利用硬件 |

**序列长度**：2048 token

**损失函数**：

$$\mathcal{L} = -\frac{1}{T}\sum_{t=1}^{T}\log P(x_t | x_{<t})$$

标准的因果语言模型交叉熵。

#### Step 4 — Pathways 分布式训练系统

PaLM 是 Google Pathways 系统的一次大规模验证。6144 个 TPU v4 芯片跨 2 个 TPU v4 Pod 进行训练。

**分布式策略**：

| 并行方式 | 说明 |
|---------|------|
| **数据并行** | 不同的 TPU 处理不同的 batch 子集 |
| **模型并行** | 将模型参数分片到多个 TPU（Megatron-style tensor parallelism） |
| **流水线并行** | 将模型层分割到不同的 TPU 上 |

**训练效率**：
- **MFU（Model FLOPs Utilization）**：46.2%——对于 6144 个芯片的规模来说，这已是极高的利用率
- **吞吐量**：约每秒处理 38M tokens

> 46.2% 的硬件利用率看起来低？实际上对于 6144 芯片的超大规模分布式训练，这已是工程奇迹。大多数超大规模训练的实际利用率在 30-45% 之间。

### 详细训练配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 参数量 | 540B | 118 层，18432 隐藏维度，48 注意力头 |
| FFN 维度 | 73728（4× 扩展） | SwiGLU 激活 |
| 词汇量 | 256K（SentencePiece） | 多语言分词 |
| 训练数据 | 780B tokens | 多语言 + 代码混合 |
| 优化器 | AdamW | β₁=0.9, β₂=0.95 |
| 峰值学习率 | 1e-2 | 对于大模型来说相当高 |
| 学习率调度 | Warmup + 余弦衰减 | Warmup 约 375M tokens |
| 权重衰减 | 0.1 | — |
| 梯度裁剪 | 1.0 | — |
| Batch Size | 3M tokens (动态增长) | — |
| Dropout | 0.0 | 无 dropout（大规模数据自然正则化） |
| 训练硬件 | 6144 TPU v4 | 跨 2 个 Pod |
| 训练时间 | 约 50 天 | — |

### 涌现能力

PaLM 在 540B 规模下展现的涌现能力是论文最引人注目的部分：

| 能力 | 小模型 (8B) | PaLM (540B) | 说明 |
|------|-----------|------------|------|
| 多步推理 | 几乎为 0 | 显著 | Chain-of-thought |
| 笑话解释 | 0% | 63% | 理解双关和幽默 |
| 代码修复 | 较差 | 优秀 | 不光是生成，还能 debug |
| 翻译 | 中等 | 接近专用系统 | 零样本多语言翻译 |
| 逻辑推理 | 较差 | 较强 | 需要多步推理的任务 |

> PaLM 最惊人的示例之一：它可以在被要求解释一个笑话时，用 chain-of-thought 逐步分析笑点、识别双关、并给出幽默来源的解释——这个能力在 8B 和 62B 模型上完全不存在。

### 预训练的实用价值

1. **涌现能力的实证**：540B 规模的涌现能力确立了"再大一些"的研究方向
2. **Pathways 系统的验证**：证明了跨 Pod 的 6000+ 芯片训练是可行的
3. **并行计算改进**：Attention+FFN 并行的设计被后续很多模型采用
4. **数据配方的启发**：社交对话数据对模型对话能力的提升具有重要意义
5. **Google Gemini 的前身**：PaLM 和 PaLM-2 的技术积累直接服务于 Gemini
