# CoCa (Contrastive Captioners)

## 基本信息

- **论文**: [CoCa: Contrastive Captioners are Image-Text Foundation Models](https://arxiv.org/abs/2205.01917)
- **作者**: Jiahui Yu et al. (Google)
- **发表**: TMLR 2023

## 创新点

1. **对比 + 生成联合训练**: 同时进行对比学习（判别）和文本生成（生成）
2. **单编码器 + 双解码器**: 共享视觉编码器，对比头和生成头并行训练
3. **多任务统一**: 一个模型同时支持检索和生成任务

## 核心原理

### CoCa 架构

1. **图像编码器**: ViT 提取视觉特征
2. **对比解码器**: 单层 Transformer，输出用于对比学习的嵌入
3. **生成解码器**: 多层 Transformer，以图像特征为条件生成文本

### 联合训练

- 同时优化对比损失 (Contrastive Loss) 和生成损失 (Captioning Loss)
- 对比损失确保嵌入对齐，生成损失确保生成能力

## 预训练方法

### 核心思想：对比检索 + 图像描述，一个模型全搞定

CLIP 擅长检索（"这张图和这段文字配不配？"），但不擅长生成（不能看图说话）。BLIP 能做两者，但需要复杂的多任务交替训练。CoCa 的设计哲学是：**用一个极其简洁的架构——共享的视觉编码器 + 两个并行的文本解码器（对比 + 生成）——同时训练对比和生成，而且两种能力互相增强**。

> CoCa = **Co**ntrastive **Ca**ptioner。对比头做"找图"，生成头做"说图"，一个模型两件事。

### 训练流水线（Step by Step）

#### Step 1 — 共享的视觉编码

图像通过一个 ViT 编码器（或 CoAtNet——CNN+Transformer 混合架构），提取视觉特征。

ViT 的输出有两部分：
- **$[CLS]$ token 输出** → 用于对比学习（全局图像级表征）
- **Patch 级输出** → 用于生成解码器（局部细节供给生成任务）

#### Step 2 — 对比解码器（单层 Transformer）

对比解码器只有**1 层**（是的，一层就够了！）：

1. 文本经过 BPE 编码，通过文本嵌入层
2. 在 1 层自注意力 + 交叉注意力（以视觉特征为条件）后，输出文本表征
3. 取最后一个 token 的输出（或 [EOS] 位置）作为整个文本的嵌入向量
4. 与图像的 $[CLS]$ 嵌入做对比损失

$$\mathcal{L}_{\text{Contrastive}} = -\frac{1}{B}\sum_{i=1}^B \left[ \log \frac{\exp(\text{sim}(v_i, t_i)/\tau)}{\sum_{j}\exp(\text{sim}(v_i, t_j)/\tau)} + \log \frac{\exp(\text{sim}(t_i, v_i)/\tau)}{\sum_{j}\exp(\text{sim}(t_i, v_j)/\tau)} \right]$$

这就是 CLIP 的对称对比损失。

#### Step 3 — 生成解码器（多层 Transformer）

生成解码器是**多层**的（如 12 层），做标准的自回归文本生成：

1. 以视觉 patch 特征为交叉注意力条件
2. 自回归生成文本 token
3. 交叉熵损失 $\mathcal{L}_{\text{Captioning}}$（Language Model Loss）

```text
[图像] → ViT → Patch特征
                    ↓ (交叉注意力)
[BOS] → 嵌入 → 多层Transformer解码器 → "a" → "cat" → "sitting" → [EOS]
```

#### Step 4 — 注意力池化器（Attentional Pooler）

一个精妙的设计：生成解码器不直接使用 ViT 的所有 196 个 patch token（太长），而是通过一个**注意力池化器**将 $N$ 个 patch 特征压缩为 $M$ 个（$M \ll N$，如 $M=256$）。

```text
[N个patch特征] → 可学习的[M个查询向量]做交叉注意力 → [M个压缩特征]
```

> 注意力池化器是 CoCa 的一个关键工程优化——它使得生成解码器的交叉注意力计算量可控，同时又保留了视觉特征的细粒度信息。

#### Step 5 — 联合训练

$$\mathcal{L}_{\text{CoCa}} = \lambda_{\text{con}} \mathcal{L}_{\text{Contrastive}} + \lambda_{\text{cap}} \mathcal{L}_{\text{Captioning}}$$

两个任务**共享视觉编码器**，但使用不同的文本解码器。这种设计的妙处在于：
- 对比损失确保视觉特征与文本的全局对齐
- 生成损失确保视觉特征包含足够的细节来支持描述生成

### 为什么 CoCa 的设计有效？

#### 对比学习和生成学习的互补

| 任务 | 学什么 | 视觉特征需求 |
|------|--------|------------|
| 对比学习（检索） | 全局语义"是什么" | 类别级别的粗粒度 |
| 生成学习（描述） | 细节信息"有什么" | 实例级别的细粒度 |

两者一起训练，视觉编码器被迫同时学习**全局语义判别**和**局部细节描述**——这种"双重压力"产生了一个更全面的视觉表征。

#### CoCa vs CLIP vs BLIP

| 特性 | CLIP | BLIP | CoCa |
|------|------|------|------|
| 对比检索 | ✓ | ✓ | ✓ |
| 图像描述生成 | ✗ | ✓ | ✓ |
| VQA/理解任务 | ✗ | ✓ | ✓（通过生成回答） |
| 架构简洁性 | 简单 | 复杂（三任务交替） | **中等（双解码器并列）** |
| 训练效率 | 高 | 中（多任务调度） | **高（两个损失并行）** |

### 详细训练配置

| 参数 | CoCa-Base | CoCa-Large |
|------|-----------|------------|
| 视觉编码器 | ViT-B/16 | ViT-L/14 |
| 对比解码器 | 1 层 Transformer | 1 层 Transformer |
| 生成解码器 | 12 层 Transformer | 12 层 Transformer |
| 注意力池化 | 256 个查询 | 256 个查询 |
| 参数量 | ~300M | ~500M |
| 训练数据 | JFT-3B + ALIGN | JFT-3B + ALIGN |
| 优化器 | Adafactor | Adafactor |
| 学习率 | 1e-3 | 1e-3 |
| 批次大小 | 32768 | 32768 |
| 图像分辨率 | 288×288 | 288×288 |
| 训练迭代 | 约 500K 步 | ~500K 步 |
| 对比温度 τ | 可学习 | 可学习 |

### 预训练性能

CoCa 展现了惊人的"全才"能力——一个模型在多个任务上达到或超越专用模型：

| 任务 | CoCa | 专用 SOTA |
|------|------|----------|
| ImageNet 零样本分类 | **86.3%** | 76.2% (CLIP) |
| COCO 图像描述 (CIDEr) | **143.6** | 138.7 (LEMON) |
| Flickr30K 检索 (R@1) | **88.0%** | 87.5% |
| VQA | **82.3%** | — |
| 微调 ImageNet | **91.0%** | — |

CoCa 是第一个在**检索和生成两方面都达到 SOTA** 的统一视觉-语言模型。

### 预训练的迁移价值

1. **优雅的统一架构**：对比 + 生成在同一个模型中共存，互不干扰、互相促进
2. **注意力池化器**：一个轻量但有效的方法，解决了长视觉序列对生成解码器的影响
3. **单层对比解码器**：证明对比学习只需要很浅的文本编码器
4. **视觉-语言基础模型的新标杆**：CoCa 证明了统一框架可以同时达到检索和生成的双 SOTA
5. **启发了 PaLI、Flamingo 等更大规模的多模态模型**
