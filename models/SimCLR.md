# SimCLR (A Simple Framework for Contrastive Learning of Visual Representations)

## 基本信息

- **论文**: [A Simple Framework for Contrastive Learning of Visual Representations](https://arxiv.org/abs/2002.05709)
- **作者**: Ting Chen, Simon Kornblith, Mohammad Norouzi, Geoffrey Hinton (Google)
- **发表**: ICML 2020

## 创新点

1. **简洁完整的对比学习框架**: 数据增强 → 编码器 → 投影头 → InfoNCE Loss
2. **系统性消融研究**: 揭示了对比学习各组件的关键作用
3. **数据增强的决定性作用**: 随机裁剪 + 色彩抖动是最关键的数据增强组合
4. **投影头 (Projection Head) 的重要性**: 非线性投影头显著提升表征质量
5. **大 Batch Size**: 4096+ 的必要性

## 核心原理

### SimCLR 框架

1. **数据增强**: 对每个样本生成两个不同的增强视图
2. **编码器**: ResNet 提取特征
3. **投影头**: 3 层 MLP 将特征映射到对比空间
4. **对比损失**: NT-Xent (Normalized Temperature-scaled Cross Entropy)

### NT-Xent Loss

$$\mathcal{L} = -\log \frac{\exp(\text{sim}(z_i, z_j)/\tau)}{\sum_{k=1}^{2N} \mathbb{1}_{[k \neq i]} \exp(\text{sim}(z_i, z_k)/\tau)}$$

其中 $N$ 为 batch 大小，$\tau$ 为温度系数。

### 关键发现

| 组件 | 影响 | 最佳选择 |
|------|------|---------|
| 数据增强 | **决定性** | 随机裁剪 + 色彩抖动 |
| 投影头 | 显著提升 | 2 层 MLP + ReLU |
| Batch Size | 越大越好 | 4096+ |
| 温度系数 $\tau$ | 重要 | 0.1–0.5 |
| 编码器大小 | 同比例提升 | 越大越好 |

## 预训练方法

### 核心思想：拉近同类，推开异类

SimCLR 预训练遵循一个朴素但强大的原则：**对同一张图片的不同"视角"，网络应该输出相似的表示；对不同图片，网络应该输出差异较大的表示**。就像教一个小孩——"看，这张猫的照片和那张猫的照片（不同角度）是同一只猫，但这不是狗"。

### 训练流水线（Step by Step）

#### Step 1 — 构建正负样本对

假设一个 batch 有 $N$ 张图片 $\{x_1, x_2, ..., x_N\}$：

1. **双重数据增强**：对每张图片 $x_i$ 应用**两次不同的随机增强**，得到两个视角 $x_i^{(1)}$ 和 $x_i^{(2)}$
2. 批次中共有 $2N$ 个增强视图
3. **正例对**：来自同一原图的 $x_i^{(1)}$ 和 $x_i^{(2)}$ 组成一对正例
4. **负例对**：batch 中所有其他 $2N-2$ 个增强视图都是负例

> 如果 batch 中有 4096 张图片，则产生 8192 个增强视图。对每个视图，有 1 个正例和 8190 个负例——这就是为什么 SimCLR "需要大 batch size"。

#### Step 2 — 编码和投影

每个增强视图通过：

$$\text{增强视图} \xrightarrow{\text{编码器 } f(\cdot)} h \in \mathbb{R}^{2048} \xrightarrow{\text{投影头 } g(\cdot)} z \in \mathbb{R}^{128}$$

- **编码器 $f$**：通常是 ResNet-50（无最后的分类层），输出 2048 维特征向量 $h$
- **投影头 $g$**：一个 3 层 MLP（2048→2048→128），将特征映射到对比学习空间。**关键发现**：用 $z$ 计算损失比直接用 $h$ 更好，因为投影头可以"忘掉"数据增强引入的无关变换信息

#### Step 3 — 计算对比损失（NT-Xent）

对于一对正例 $(i, j)$，InfoNCE 风格的损失为：

$$\ell(i,j) = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k=1}^{2N} \mathbb{1}_{[k \neq i]} \exp(\text{sim}(z_i, z_k) / \tau)}$$

其中：
- $\text{sim}(u, v) = \frac{u^T v}{\|u\|\|v\|}$ 是**余弦相似度**（归一化后的点积）
- $\tau$ 是**温度系数**（temperature），控制分布的"锐度"

**直觉解读**：分子是"我和我的正例有多相似"，分母是"我和所有人（包括正负例）的相似度总和"。损失越小，意味着正例之间的相似度在分母中占据主导。

最终损失是所有正例对的对称平均：

$$\mathcal{L} = \frac{1}{2N} \sum_{k=1}^{N} [\ell(2k-1, 2k) + \ell(2k, 2k-1)]$$

#### Step 4 — 训练完成后丢弃投影头

预训练结束后，**投影头 $g$ 被丢弃**，只保留编码器 $f$。下游任务使用 $f$ 输出的 $h$（2048 维）作为图像表征。这个设计非常巧妙——投影头相当于一个"临时工"，在对比学习阶段承担了过滤噪声的任务，但最终交付的是更纯净的特征 $h$。

### 为什么这样设计——深入理解每个组件

#### 1. 数据增强为什么如此重要？

SimCLR 最核心的发现是：**数据增强策略直接决定了对比学习的成败**。

| 增强方式 | 单独使用 Top-1 | 组合效果 |
|---------|---------------|---------|
| 随机裁剪 | 32.3% | **最关键**——强制模型关注不同空间区域 |
| 色彩抖动 | 9.1% | 防止模型用颜色"作弊" |
| 裁剪 + 色彩抖动 | **55.8%** | 1+1>2 —— 两种增强的信息互补 |
| 裁剪 + 色彩抖动 + 高斯模糊 | 63.2% | 进一步的提升 |

**为什么裁剪和色彩抖动是黄金组合？**

想象一下——如果只用裁剪，两张猫的图片可能因为颜色相似被判定为同类（即使不是同一只猫），模型学会了依赖颜色。如果加上色彩抖动，同一个猫在不同增强下颜色也不一样了，模型被迫去学习**形状、纹理等更本质的视觉特征**。

#### 2. 投影头为什么重要？

投影头 $g$ 是一个非线性 MLP，将特征 $h$ 映射到对比空间 $z$。消融实验表明：

| 投影头配置 | Top-1 |
|-----------|-------|
| 无投影头（直接用 $h$） | 53.7% |
| 线性投影 | 60.9% |
| 1 层 MLP + ReLU | 63.0% |
| 2 层 MLP + ReLU | **63.2%** |
| 3 层 MLP + ReLU | 63.0% |

**为什么需要投影头？**

直觉解释：编码器 $h$ 包含的信息是多方面的（颜色、纹理、形状、语义……），但并不是所有信息都对"判断两张图是否是同一物体"有用。投影头 $g$ 充当一个**信息过滤器**——它学会了只从 $h$ 中提取那些对对比学习有用的特征，而把"噪音"留在 $h$ 中。训练后丢掉 $g$，我们得到的是一个保留了更丰富信息的 $h$。

#### 3. 为什么需要大 Batch Size？

在 SimCLR 中，**负例来自同一个 batch**。Batch size = $N$ 意味着每个正例有 $2N-2$ 个负例。

| Batch Size | 负例数 | Top-1 |
|-----------|--------|-------|
| 128 | 254 | 57.2% |
| 256 | 510 | 58.4% |
| 512 | 1022 | 60.9% |
| 1024 | 2046 | 62.8% |
| 2048 | 4094 | 64.5% |
| 4096 | 8190 | **66.3%** |

**为什么越多负例越好？**

对比学习的本质是"区分"——负例越多，区分难度越大，模型必须学习更精细的特征才能区分它们。想象你要从 10 个人中找出你的朋友（容易）vs 从 10000 人中找出你的朋友（难）——后一种情况你需要更细致的辨识能力。

但这不是免费的——4096 的 batch size 需要大量 GPU 显存。这也是 MoCo 用队列存储负例来解耦 batch size 和负例数量的动机。

#### 4. 温度系数 $\tau$ 的作用

温度系数 $\tau$ 控制 softmax 分布的"形状"：

| $\tau$ 值 | 效果 | Top-1 |
|----------|------|-------|
| 0.05 | 分布极尖锐，只关注最难区分的负例 | 59.4% |
| 0.1 | 较尖锐，给难负例更高权重 | 63.2% |
| 0.5 | 适中，均匀关注所有负例 | **63.8%** |
| 1.0 | 较平滑，所有负例权重相近 | 62.5% |

$\tau$ 太小 → 只关心最难分的负例（hard negatives），忽略简单负例 → 学习不稳定
$\tau$ 太大 → 所有负例权重均匀 → 模型不关注区分难度 → 学习效率低

### 详细训练配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 编码器 | ResNet-50（1×, 2×, 4× 宽度） | 标准残差网络，去掉分类头 |
| 投影头 | 3 层 MLP：2048→2048→128 | ReLU 在中间层，Linear 在最后一层 |
| 输出维度 | $h$: 2048, $z$: 128 | $z$ 用于损失计算，$h$ 用于下游 |
| 优化器 | LARS | 专为大 batch 设计的优化器 |
| 学习率 | $4.8$（BS=4096，Linear Scaling） | $lr = 0.3 \times \text{BatchSize}/256$ |
| 学习率调度 | 余弦衰减（无 Restart） | 从初始值衰减到 0 |
| Warmup | 前 10 epoch | 线性 warmup |
| 权重衰减 | 1e-6 | 极小的 L2 正则化 |
| 动量 | 0.9 | SGD 动量的标准值 |
| Batch Size | 4096 | 使用 128 个 Cloud TPU v3 核心 |
| 训练 Epoch | 100 / 200 / 400 / 800 / 1000 | 更长训练持续提升性能 |
| 温度 $\tau$ | 0.1（v1）/ 0.5（v2） | — |
| 数据增强 | RandomResizedCrop + ColorJitter + RandomGrayscale + RandomHorizontalFlip + GaussianBlur | — |
| 全局 BN | 是（SyncBN 跨设备同步） | 大 batch 必需，否则 BN 统计不稳定 |

#### 增强参数细节

| 增强 | 参数 |
|------|------|
| RandomResizedCrop | 尺寸 224×224，面积比 0.08~1.0 |
| ColorJitter | 亮度 0.8，对比度 0.8，饱和度 0.8，色调 0.2 |
| RandomGrayscale | 概率 0.2 |
| GaussianBlur | 核大小 23×23，σ ∈ [0.1, 2.0] |
| RandomHorizontalFlip | 概率 0.5 |

#### SimCLR v2 的改进

SimCLR v2 在 v1 基础上做了三点升级：

1. **更大的模型**：ResNet-152（3× 宽度）替代 ResNet-50
2. **更深的投影头**：3 层 → 4 层 MLP，且在微调阶段保留前 2 层
3. **知识蒸馏**：使用自监督预训练模型作为"教师"，蒸馏到更小的"学生"模型

### 预训练性能

| 配置 | ImageNet Top-1（线性探测） | ImageNet Top-1（微调） |
|------|--------------------------|---------------------|
| ResNet-50, 100 epoch | 66.5% | — |
| ResNet-50, 200 epoch | 68.3% | — |
| ResNet-50, 400 epoch | 69.8% | — |
| ResNet-50, 800 epoch | 70.4% | 76.5% |
| ResNet-50, 1000 epoch | 71.1% | — |
| ResNet-50 (2×), 800 epoch | 74.2% | 79.3% |
| ResNet-50 (4×), 800 epoch | **76.5%** | 80.9% |
| 监督基线 (ResNet-50) | 76.5%（监督预训练） | 76.5% |

关键观察：
- 800 epoch 时，SimCLR（4× 宽度）的线性探测性能**追平了监督预训练**
- 微调时，SimCLR 甚至可以**超越**监督预训练（79.3% → 76.5% / 80.9% → 76.5%）
- 训练时间越长，性能越好（没有饱和迹象）

### 迁移学习的实用价值

SimCLR 的贡献不仅是性能，更是**系统性消融实验**揭示的对比学习规律：

1. **数据增强 > 模型架构**：好的增强策略带来的提升远大于换更大的模型
2. **投影头不可或缺**：非线性投影头是让对比学习有效的关键
3. **大 batch = 多负例 = 好性能**：需要足够多的负例让对比学习有区分力
4. **训练时间很重要**：对比学习收敛慢，需要比监督学习更多的迭代
