# SimSiam (Simple Siamese Network)

## 基本信息

- **论文**: [Exploring Simple Siamese Representation Learning](https://arxiv.org/abs/2011.10566)
- **作者**: Xinlei Chen, Kaiming He (Meta)
- **发表**: CVPR 2021

## 创新点

1. **极简框架**: 仅需 stop-gradient 即可防止表征坍塌
2. **无需负例、无需动量编码器、无需大 batch**
3. **理论解释**: Stop-gradient 相当于隐式 EM 算法

## 核心原理

### SimSiam 架构

1. 对输入图像生成两个增强视图
2. 共享权重的编码器 + 投影头 + 预测头
3. 计算余弦相似度作为损失
4. **Stop-gradient**: 其中一个分支不接收梯度

### 为什么 Stop-gradient 可以防止坍塌？

SimSiam 的理论分析表明：
- Stop-gradient 将一个对称网络转化为**两个角色的交替优化**
- 相当于隐式的 EM 算法
- 网络同时扮演"预测器"和"目标"两个角色

### 与 BYOL 的对比

| 方法 | 目标网络 | 动量更新 | 预测头 | Stop-gradient |
|------|---------|---------|-------|-------------|
| BYOL | 有 | 是 | 有 | 隐式 |
| SimSiam | 无 | 否 | 有 | **显式** |

## 预训练方法

### 核心思想：最简单的孪生网络 + Stop-gradient

SimSiam 用一句简单的话总结了自监督学习：**不需要负例、不需要动量编码器、不需要大 batch——只需要 stop-gradient 就能防止表征坍塌**。它的理念是"去繁就简"，揭示非对比式学习中真正必要的最小组件。

> SimSiam 的名字（Simple Siamese）准确地描述了它的本质——两个完全相同（共享权重）的网络（孪生网络），用最简单的设计实现有效的自监督学习。

### 训练流水线（Step by Step）

#### Step 1 — 数据增强生成两个视图

对输入图像 $x$ 应用两次随机数据增强，得到 $x_1$ 和 $x_2$。

#### Step 2 — 共享编码器 + 投影头 + 预测头

两个视图通过**完全相同（共享权重）**的网络：

$$p_1 = h(g(f(x_1)))$$
$$z_2 = g(f(x_2))$$

其中：
- $f$：编码器（如 ResNet-50，提取特征）
- $g$：投影头（3 层 MLP，映射到对比空间）
- $h$：预测头（2 层 MLP，在投影头之上）

#### Step 3 — Stop-gradient 操作

这是 SimSiam **最关键的操作**：

$$\mathcal{L} = -\frac{p_1}{\|p_1\|_2} \cdot \text{sg}\left(\frac{z_2}{\|z_2\|_2}\right)$$

其中 $\text{sg}(\cdot)$ 表示 **stop-gradient**——$z_2$ 不接收梯度，在反向传播中被当作常量处理。

用更直白的话说：
- $p_1$ 正常接收梯度更新
- $z_2$ 不接收梯度——它的值被用来计算损失，但不会被损失"推着走"

#### Step 4 — 对称损失

同样交换角色再算一遍，两个方向的损失求平均：

$$\mathcal{L}_{\text{SimSiam}} = \frac{1}{2}D(p_1, \text{sg}(z_2)) + \frac{1}{2}D(p_2, \text{sg}(z_1))$$

其中 $D(p, z) = -\frac{p}{\|p\|_2} \cdot \frac{z}{\|z\|_2}$ 是负余弦相似度。

### 为什么这样设计——深入理解 Stop-gradient

#### 1. Stop-gradient 为什么能防止坍塌？

这是 SimSiam 论文最核心的理论贡献。想象一下没有 stop-gradient 的情况：

$$\mathcal{L} = D(p_1, z_2)$$

此时 $p_1$ 和 $z_2$ **都**接收梯度。一个显然的最优解是：

$$p_1 = z_2 = \text{常量向量（如全 0）}$$

此时 $D(p_1, z_2)$ 达到最小，但表征毫无意义——这就是"坍塌"。

**Stop-gradient 打破了这个退化路径**。当 $z_2$ 不接收梯度时，它不能主动向 $p_1$ 靠拢，只能被动等待 $p_1$ 来"匹配"它。但 $p_1$ 也不能简单地输出常数——因为 $z_2$（由共享编码器产生）随着训练不断变化，常数输出很快就不再匹配。

这形成了一个微妙的动态平衡：
$$
p_1 \xrightarrow{\text{梯度更新}} \text{尝试匹配 } z_2
$$
$$
z_2 \text{ 不接收梯度，} \text{但它由共享编码器产生，} \text{随 } f \text{ 的更新而变化}
$$

#### 2. SimSiam 是 EM 算法的一个实例

SimSiam 论文给出了一个优美的理论解释：stop-gradient 相当于隐式执行 **期望最大化（EM）算法**。

**EM 视角下**：
- **E-step（期望步）**：将 $z$（无梯度的分支输出）视为"隐变量的期望值"
- **M-step（最大化步）**：更新 $p$（有梯度的分支），使其尽量匹配隐变量

由于两个分支共享编码器 $f$，$z$ 也会随 $f$ 的更新而改变——这形成了隐式的交替优化。

#### 3. BYOL vs SimSiam vs 其他方法的组件对比

SimSiam 通过消融实验清晰地展示了各种方法的"最小必要组件"：

| 组件 | SimCLR | MoCo v2 | BYOL | SwAV | **SimSiam** |
|------|--------|---------|------|------|------------|
| 负例 | ✅ 需要 | ✅ 需要 | ❌ 不需要 | ✅ 需要 | ❌ 不需要 |
| 动量编码器 | ❌ | ✅ | ✅ | ❌ | ❌ |
| 大 batch | ✅ 需要 | ❌ | ❌ | ❌ | ❌ |
| 预测头 | ❌ | ❌ | ✅ | ❌ | ✅ |
| Stop-gradient | ❌ | ❌ | 隐式 | ❌ | **✅ 显式** |
| 聚类 | ❌ | ❌ | ❌ | ✅ | ❌ |

**结论**：SimSiam 证明了，如果你想用最简单的方式做非对比式自监督学习——只需要**预测头 + stop-gradient**即可。

### 消融实验的关键发现

#### 预测头的作用

| 配置 | Top-1 | 观察 |
|------|-------|------|
| 无预测头 | 坍塌 | 没有预测头，stop-gradient 不够 |
| 有预测头（随机初始化） | 67.7% | 即使随机初始化也可用 |
| 有预测头（正常训练） | 68.1% | 标准做法 |

预测头提供了一个"不对称"的变换——$p = h(z)$ 和 $z$ 不同，这让网络不能简单地输出常数。

#### Batch Size 的影响（令人惊讶的鲁棒性）

| Batch Size | Top-1 (100 epoch) |
|-----------|-------------------|
| 64 | 67.8% |
| 128 | 68.1% |
| 256 | 68.0% |
| 512 | 68.1% |
| 1024 | 67.9% |
| 2048 | 68.0% |
| 4096 | 67.6% |

**惊人的发现**：SimSiam 在 BS=64 和 BS=4096 上性能几乎相同！这与 SimCLR 对大 batch 的强依赖形成鲜明对比。原因在于：SimSiam 没有对比损失——不需要负例，因此也不需要大 batch 提供足够的负例。

#### 优化器选择

| 优化器 | Top-1 | 说明 |
|--------|-------|------|
| SGD | 68.1% | 论文默认 |
| SGD + momentum | 67.8% | 动量不影响 |
| Adam | 66.9% | 略微下降 |
| LARS | 67.6% | 不需要特殊优化器 |

SimSiam 可以用最普通的 SGD（甚至不需要 momentum），这进一步证明了其简洁性。

### 详细训练配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 编码器 | ResNet-50 | 标准残差网络 |
| 投影头 $g$ | MLP: 2048→2048→2048 | 三层，每层后 BN + ReLU |
| 预测头 $h$ | MLP: 2048→512→2048 | 两层，隐藏层有 BN + ReLU，输出层无 BN |
| 输出维度 | 投影=2048, 预测=2048 | 和 BYOL 不同（BYOL 输出 256） |
| 优化器 | SGD | **无 momentum**（lr=0.05） |
| 权重衰减 | 0.0001 | — |
| Batch Size | 512 | 不敏感，256-4096 均可 |
| 训练 Epoch | 100 / 200 / 400 | 更长训练持续提升 |
| 学习率 | 0.05 | 固定学习率（前 10 epoch warmup 除外） |
| 学习率调度 | 余弦衰减 | — |
| 数据增强 | SimCLR 风格 | 随机裁剪 + 色彩抖动 + 高斯模糊等 |

#### Stop-gradient 的 PyTorch 实现

SimSiam 的核心可以用极少的代码实现：

```python
# 前向传播
z1 = projector(encoder(x1))     # 分支1的投影
z2 = projector(encoder(x2))     # 分支2的投影
p1 = predictor(z1)              # 分支1的预测
p2 = predictor(z2)              # 分支2的预测

# 损失 (stop-gradient 通过 .detach() 实现)
loss = -0.5 * (cosine_sim(p1, z2.detach()) + cosine_sim(p2, z1.detach()))

# cosine_sim = (u·v) / (||u|| ||v||) — 归一化后再点积
```

|------|--------|-------|---------|
| SimSiam | ResNet-50 | 100 | 68.1% |
| SimSiam | ResNet-50 | 200 | 70.0% |

SimSiam 在短 epoch 下性能接近 SOTA 方法，但更长训练下逐渐被 BYOL 拉开差距。这说明 SimSiam 的简洁性有一定代价——它在"极限性能"上不如有动量编码器的方法。

### 迁移学习的实用价值

SimSiam 的核心贡献不在于 SOTA 性能，而在于**理论的简洁性**：

1. **最小组件的概念验证**：证明了只要 stop-gradient + 预测头就能防止坍塌，不需要复杂的动量编码器或大 batch
2. **理论解释**：将非对比学习与 EM 算法联系起来，提供了理解自监督学习的新视角
3. **教学价值**：SimSiam 仅需 ~100 行代码就能实现有效的自监督学习，是自监督学习的最佳入门方法
4. **工程友好**：对 batch size 不敏感 + 不需要特殊优化器 + 不需要动量更新，使得在资源受限的环境下也能训练
