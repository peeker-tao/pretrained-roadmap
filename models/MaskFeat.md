# MaskFeat (Masked Feature Prediction)

## 基本信息

- **论文**: [Masked Feature Prediction for Self-Supervised Visual Pre-Training](https://arxiv.org/abs/2112.09133)
- **作者**: Chen Wei, Haoqi Fan, Saining Xie, Chao-Yuan Wu, Alan Yuille, Christoph Feichtenhofer (Meta)
- **发表**: CVPR 2022

## 创新点

1. **HOG 特征重建**: 提出重建 HOG（方向梯度直方图）特征而非原始像素
2. **鲁棒性**: HOG 特征对光照变化、颜色变换等不敏感，提供更稳定的学习信号
3. **避免像素模糊性**: HOG 是局部梯度统计，比像素更接近语义信息

## 核心原理

### HOG 特征

HOG 特征计算图像局部区域的梯度方向直方图：
- 将图像分为小单元 (cells)
- 计算每个像素的梯度大小和方向
- 将方向量化为 9 个 bin，统计每个 cell 的梯度分布
- 优点：对光照变化不敏感，包含丰富的形状信息

### MaskFeat 框架

1. 随机掩码图像 patch（约 40%）
2. 编码器（ViT）处理可见 patch
3. 预测头输出每个掩码位置的 HOG 特征
4. 计算预测 HOG 与真实 HOG 之间的 Smooth L1 损失

### HOG vs. 像素重建

| 方面 | 像素重建 (MAE) | HOG 重建 (MaskFeat) |
|------|---------------|-------------------|
| 目标信号 | 原始 RGB 值 | 梯度方向直方图 |
| 语义层级 | 低层 | 中层 |
| 对光照鲁棒性 | 差 | 好 |
| 信息量 | 高但冗余 | 适中但更精炼 |

## 预训练方法

### 核心思想：不重建像素，重建"边缘直方图"

MAE 的直接像素重建有一个根本问题：**像素是模糊的学习信号**——遮住一只猫的尾巴，模型重建出"灰色模糊"也算错了，重建出精确的 RGB 值才算对了，但前者已经学到了"那里是灰色皮毛"的语义。MaskFeat 的核心洞察：**与其让模型预测"精确的颜色值"，不如让它预测更抽象的视觉特征**——HOG（Histogram of Oriented Gradients，方向梯度直方图）就是这样的特征。

> HOG 是计算机视觉中经典的局部特征描述子，它统计图像每个小区域内"边缘"的方向分布——不管颜色怎么变，只要形状在，HOG 就稳定。MaskFeat 就是让模型"画轮廓"而非"填颜色"。

### 训练流水线（Step by Step）

#### Step 1 — 计算 HOG 特征作为"目标"

对输入图像，预先计算每个 patch 的 HOG 特征：

1. 将图像分为 $4 \times 4$ 像素的小 cell（对应 ViT 中 $16 \times 16$ 的 patch 的 1/4 细分）
2. 计算每个 cell 中所有像素的梯度（Sobel 算子）
3. 将梯度方向量化为 9 个 bin（0°, 20°, 40°, ..., 160°）
4. 在每个 $2 \times 2$ 的 cell 块中统计梯度方向分布
5. 对每个 patch 得到一个 HOG 特征向量

**HOG 特征的数学形式**：

$$\text{HOG}_{i} = \sum_{p \in \text{cell}_i} \|\nabla I(p)\| \cdot \mathbb{1}[\text{orientation}(\nabla I(p)) \in \text{bin}_i]$$

直观理解：HOG 告诉你"在这个 $16 \times 16$ 的 block 里，有多少个像素的梯度是向左的、向右的、向上的……"——这不关心颜色，只关心形状和边缘。

#### Step 2 — 掩码策略

随机掩码约 40% 的图像 patch（与 BEiT 类似的掩码率）：
- 被掩码的 patch → 替换为 $[MASK]$ token（一个可学习的向量）
- 可见 patch → 保留原始 patch embedding

#### Step 3 — ViT 编码

完整的 patch 序列（包括 $[MASK]$ token）+ 位置编码 → ViT 编码器。

#### Step 4 — HOG 预测头

每个被掩码位置的 ViT 输出通过一个**轻量预测头**（MLP）输出预测的 HOG 特征。

$$\hat{h}_i = \text{MLP}_{\text{pred}}(z_i), \quad i \in \text{masked}$$

#### Step 5 — 损失计算：Smooth L1

计算预测 HOG 与真实 HOG 之间的 **Smooth L1 损失**：

$$\mathcal{L} = \frac{1}{|\text{masked}|} \sum_{i \in \text{masked}} \text{SmoothL1}(\hat{h}_i, h_i)$$

Smooth L1 的定义：

$$\text{SmoothL1}(x) = \begin{cases} 0.5x^2 & \text{if } |x| < 1 \\ |x| - 0.5 & \text{otherwise} \end{cases}$$

为什么用 Smooth L1 而不是 MSE？Smooth L1 对离群值不敏感——HOG 特征中有些梯度方向可能非常强，MSE 会让模型过分关注这些高梯度区域，而 Smooth L1 提供了更均衡的学习信号。

### 为什么 HOG 优于像素重建？

#### 像素重建的三大问题

| 问题 | 说明 | 示例 |
|------|------|------|
| 颜色歧义 | 同样形状可以是任何颜色 | 猫可以是黑猫/白猫/花猫 |
| 纹理歧义 | 像素级别的纹理变化包含大量冗余 | 毛发、木纹的细微波纹 |
| 光照敏感 | 阴影和亮度变化导致像素值剧烈变化 | 室外光照变化 |

#### HOG 怎么解决这些问题

| 维度 | 像素重建（MAE） | HOG 重建（MaskFeat） |
|------|---------------|-------------------|
| 信号目标 | 原始 RGB (3 通道) | 方向梯度直方图 (9-36 bin) |
| 语义层级 | 低层（颜色精确性） | **中层**（形状/纹理概要） |
| 颜色不变性 | 无 | **天然具备**（只关心梯度，不关心值） |
| 光照鲁棒性 | 差 | **好**（梯度相对光照变化稳定） |
| 冗余信号 | 高（相邻像素高度相关） | 低（HOG 已经做了降维汇总） |

#### 实验验证

MaskFeat 在多个任务上验证了 HOG 重建的优越性：

- **图像分类**：MaskFeat 达到了与 MAE 相当的水平
- **视频理解**：MaskFeat **显著优于** MAE——视频中的光照和颜色变化尤为剧烈，HOG 的稳定性发挥更大作用
- **目标检测**：MaskFeat 在低数据微调场景下表现优于 MAE

### 扩展到视频：HOG 特征的天然适配

MaskFeat 在视频预训练中的优势特别明显。视频帧之间的颜色和光照变化很大——同一场景下午和黄昏的光照完全不同，但形状和边缘保持稳定。HOG 特征天生适合处理这种情况，不需要额外的"颜色归一化"技巧。

视频中 MaskFeat 使用 **MViT（Multi-scale ViT）** 架构，在 Kinetics-400/600/700 等视频动作识别数据集上取得了非常有竞争力的结果。

### MaskFeat vs BEiT vs MAE vs data2vec

| 方法 | 重建目标 | 信号类型 | 特点 |
|------|---------|---------|------|
| MAE | RGB 像素 | 低层连续 | 最简单直接 |
| BEiT | 离散视觉 token | 语义离散 | 需要额外 dVAE 训练 |
| **MaskFeat** | **HOG 特征** | **中层连续** | 手工特征，但语义丰富 |
| data2vec | 教师网络输出 | 高层连续 | 多模态统一 |

> MaskFeat 代表的是一种"手工特征 + 深度学习"的折中方案——用经典计算机视觉的特征（HOG）作为学习目标，稳定且高效，不需要训练额外的网络来提供监督信号。

### 详细训练配置

| 参数 | 图像 (ViT-B) | 视频 (MViT) |
|------|------------|------------|
| 掩码率 | 40% | 90%（时间维度 + 空间维度） |
| HOG bin 数 | 9 | 9 |
| Cell 大小 | 4×4 | 4×4 |
| Block 大小 | 2×2 | 2×2 |
| 损失函数 | Smooth L1 | Smooth L1 |
| 优化器 | AdamW | AdamW |
| 学习率 | 2e-4 | — |
| 学习率调度 | 余弦衰减 | 余弦衰减 |
| Warmup | 10 epoch | — |
| Batch Size | 1024 | 512 |
| Epoch | 300 | — |
| Stochastic Depth | 0.1 | — |

### 预训练的迁移价值

1. **视频预训练的突破**：在视频理解任务上的表现优于 MAE，HOG 的时间稳定性在视频中特别有价值
2. **轻量高效**：HOG 特征可以在数据预处理阶段计算，不增加训练时的计算负担
3. **"经典特征+深度学习"的成功融合**：证明了传统计算机视觉特征在自监督预训练中仍有价值
4. **提供了一个重要洞见**：不是所有重建目标的效果都一样，**目标特征的"语义层级"**对预训练效果有决定性影响
