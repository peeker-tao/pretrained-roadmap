# Gemini

## 基本信息

- **论文**: [Gemini: A Family of Highly Capable Multimodal Models](https://arxiv.org/abs/2312.11805)
- **作者**: Google DeepMind
- **发表**: arXiv, 2023

## 创新点

1. **原生多模态 (Native Multimodal)**: 从训练开始就是多模态的，非拼接式
2. **多模态同时训练**: 文本、图像、音频、视频、代码同时训练
3. **MMLU 首次超人类专家**: Gemini Ultra 在 MMLU 上达到 90.0%

## 核心原理

### 原生多模态架构

Gemini 在模型设计之初就考虑了多模态输入，所有模态共享同一 Transformer 架构，使用统一的序列表示。

### 不同规模

| 版本 | 能力定位 |
|------|---------|
| Gemini Ultra | 最强能力 |
| Gemini Pro | 平衡性能与效率 |
| Gemini Nano | 设备端运行 |

## 预训练方法

### 核心思想："原生多模态"——从出生起就"看得到、听得到、读得到"

Gemini 是 Google DeepMind 在 2023 年底发布的旗舰多模态模型。它的核心设计理念是"原生多模态"（Native Multimodal）——与 LLaVA 这类"事后拼接"（先训练好 LLM，再接入视觉）的方法不同，Gemini 从预训练的第一天开始就同时处理文本、图像、音频、视频和代码。

> Gemini = Google 的"全模态总攻"。它不仅是语言模型、不仅是视觉-语言模型——它被设计为一个通用的多模态基础模型。

### 训练流水线（概览）

由于 Gemini 的技术报告未公开所有训练细节，以下基于公开信息和合理推断：

#### Step 1 — 多模态 Token 化

Gemini 将所有模态统一为"token 序列"：

| 模态 | Token 化方式 |
|------|------------|
| 文本 | SentencePiece 分词（32K/256K 词汇） |
| 图像 | Patch 分割 → 视觉编码器（类似 ViT） |
| 音频 | 16kHz 波形 → 音频编码器（类似 USM） |
| 视频 | 帧采样 → 每帧 patch + 时序编码 |
| 代码 | 与文本相同的 tokenizer，特殊标记 |

**交错的 token 序列**：

```text
[<img> img_patch_1 img_patch_2 ... </img>] [文本 token ...] [<audio> audio_token_1 ... </audio>] ...
```

#### Step 2 — 联合预训练目标

Gemini 可能使用了多种预训练目标的组合：

| 目标 | 说明 |
|------|------|
| **下一个 token 预测** | 标准的因果 LM 损失，覆盖所有 token |
| **多模态对比** | 可能的图像-文本、视频-文本对齐 |
| **掩码预测** | 可能的 MLM/MIM 变体 |

**核心损失**：

$$\mathcal{L} = -\sum_{t} \log P(x_t | x_{<t}, \text{多模态上下文})$$

其中多模态上下文包括所有在 $x_t$ 之前的 token（无论来自哪种模态）。

#### Step 3 — 训练基础设施

Gemini 的训练规模前所未有：

| 维度 | Gemini Ultra | 说明 |
|------|-------------|------|
| 训练硬件 | TPU v4/v5 Pods | 具体数量未公开 |
| 训练数据 | 多模态网页 + 书籍 + 代码 + 视频 + 音频 | 数万亿 token 级别 |
| 训练策略 | 多阶段（预训练 → 对齐 → 指令微调） | — |
| 上下文长度 | 32K（Pro 版本为 128K，1.5 Pro 为 1M+） | 外推能力 |

#### Step 4 — 对齐训练

预训练之后，Gemini 经历了类似 InstructGPT 的对齐训练：

1. **SFT**：多模态指令微调
2. **RLHF**：基于人类偏好的强化学习（可能包含 Constitutional AI 元素）
3. **安全训练**：针对安全/有害内容的额外训练

### "原生多模态" vs "拼接式多模态"

| 维度 | 拼接式（LLaVA/Flamingo） | 原生多模态（Gemini/Fuyu） |
|------|------------------------|------------------------|
| 预训练起点 | 纯文本 LLM | 从零开始的统一模型 |
| 视觉模块 | 冻结的预训练视觉编码器 | **同时训练**的视觉编码器 |
| 模态融合 | 后期注入（投影/交叉注意力） | **全程交织** |
| 跨模态理解 | 依赖于桥接模块 | **端到端学习** |
| 对新模态的适应 | 复杂（需要新桥接） | 较简单（统一框架） |
| 训练成本 | 低（复用预训练组件） | **极高** |

### Gemini 模型家族

| 版本 | 定位 | 上下文长度 | 关键能力 |
|------|------|-----------|---------|
| **Ultra** | 最强模型 | 32K | MMLU 90.0%, 多模态 SOTA |
| **Pro** | 平衡版 | 128K | API 主力 |
| **Nano-1/2** | 设备端 | 8K | 手机本地运行 |

### 里程碑性能

| 基准 | GPT-4V | Gemini Ultra |
|------|--------|-------------|
| MMLU | 86.4% | **90.0%**（首次超人类专家） |
| MMMU（多模态理解） | 56.8% | **59.4%** |
| MathVista | 49.9% | **53.0%** |
| 视频理解 | — | 强 |
| 音频理解 | — | 强 |

### 预训练的实用价值

1. **原生多模态范式**：验证了"从零开始统一训练"是可行的（虽然极贵）
2. **MMLU 90% 里程碑**：首次在 MMLU 上超越人类专家（89.8%）
3. **多模态评估新基准**：MMMU 成为衡量 VLM 综合能力的重要基准
4. **Google AI 生态的基石**：Gemini 驱动了 Google Search、Workspace、Android 等全线产品
5. **长上下文（1M+）的突破**：Gemini 1.5 Pro 的 100 万 token 上下文长度重新定义了 LLM 的"记忆"能力
