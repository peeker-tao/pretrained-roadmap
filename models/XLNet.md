# XLNet

## 基本信息

- **论文**: [XLNet: Generalized Autoregressive Pretraining for Language Understanding](https://arxiv.org/abs/1906.08237)
- **作者**: Zhilin Yang et al. (Google / CMU)
- **发表**: NeurIPS 2019

## 创新点

1. **排列语言建模 (Permutation Language Modeling, PLM)**: 使用排列组合实现自回归模型的双向上下文建模
2. **两段式自注意力 (Two-Stream Self-Attention)**: 解决排列语言建模中的位置信息冲突问题
3. **Transformer-XL 集成**: 将 Transformer-XL 的片段级循环机制和相对位置编码引入

## 核心原理

### 排列语言建模

对一个长度为 $T$ 的序列，随机选择 $T!$ 种排列中的一种，然后按照该排列顺序进行自回归预测。**模型看到的不是序列的物理顺序，而是排列后的因式分解顺序**。

$$\mathcal{L} = \mathbb{E}_{z \sim \mathcal{Z}_T} \left[ \sum_{t=1}^T \log P_\theta(x_{z_t} | x_{z_{<t}}) \right]$$

### 双流自注意力

传统自回归模型无法使用双向上下文的原因是位置编码问题。XLNet 引入两种注意力表示：
- **内容表示 (Content Stream)**: 编码 $x$ 和位置信息（类似标准注意力）
- **查询表示 (Query Stream)**: 仅编码位置信息（用于预测当前 token）

两段式自注意力巧妙地解决了"预测目标不能看到自己"但"其他 token 需要看到它"的矛盾。

## 预训练方法

### 核心思想：让自回归也能"双向看"

BERT 的核心优势是双向上下文，但它是通过"遮住词再猜"（MLM）来实现的——这种方式存在一个根本问题：**BERT 假设被遮住的词彼此独立**，但现实中句子中的词是互相依赖的。GPT 是自回归的（词之间存在依赖），但它只能单向看（从左到右）。

XLNet 的核心贡献：**通过排列语言模型（Permutation Language Modeling），让自回归模型也能捕捉双向上下文**——而且保留了自回归模型捕捉词间依赖的优势。

> XLNet = 自回归（GPT 的依赖捕捉）+ 双向上下文（BERT 的上下文利用）+ Transformer-XL（长程建模）。

### 训练流水线（Step by Step）

#### Step 1 — 排列语言建模（PLM）

给定一个长度为 $T$ 的序列 $x_1, x_2, ..., x_T$，XLNet 不是按物理顺序 $1 \to 2 \to ... \to T$ 做自回归预测，而是**对序列进行随机排列**，然后按排列后的顺序预测。

**排列因式分解**：

$$\mathcal{L} = \mathbb{E}_{z \sim \mathcal{Z}_T} \left[ \sum_{t=1}^T \log P_\theta(x_{z_t} | x_{z_{<t}}) \right]$$

其中 $\mathcal{Z}_T$ 是 $T!$ 种排列的集合。

**示例**：

原始序列：`[我, 爱, 北京, 天安门]`

排列 1: `[3, 2, 1, 4]` → 预测顺序：北京 → 爱 → 我 → 天安门

- `P(北京 | [START])` — 没有任何上下文，预测"北京"
- `P(爱 | 北京, [START])` — 看到"北京"，预测"爱"
- `P(我 | 北京, 爱, [START])` — 看到"北京→爱"，预测"我"
- `P(天安门 | 北京, 爱, 我, [START])` — 看到前面所有词

排列 2: `[1, 4, 2, 3]` → 预测顺序：我 → 天安门 → 爱 → 北京

注意：排序的是索引位置，不是词本身。每个位置 $z_t$ 能看到的都是原序列中排在它**之前**位置的词——按排列顺序。

> **关键洞察**：在排列 $[3, 2, 1, 4]$ 中，第 1 步预测位置 3（"北京"）时看不到任何东西；第 3 步预测位置 1（"我"）时能看到位置 3（"北京"）和位置 2（"爱"）——这就是"双向"的来源——"我"既能看到左边的"爱"（真实邻接），也能看到右边的"北京"（在排列中排在前面）。

**排列语言模型 vs BERT MLM**：

| 特性 | BERT MLM | XLNet PLM |
|------|---------|----------|
| 上下文 | 双向 | 双向（通过排列） |
| 词间依赖 | 独立假设（掩码词之间） | 保留（自回归） |
| 训练信号 | 仅 15% token | 全部 token（在期望上） |
| 预训练-微调差异 | 有（$[MASK]$ token） | 无 |

#### Step 2 — 双流自注意力（Two-Stream Self-Attention）

排列语言建模遇到了一个困境：**如何让模型知道"预测位置"但看不到"预测内容"？**

标准自注意力：

$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{Q K^T}{\sqrt{d}}\right)V$$

在自回归中，$x_t$ 的 Q、K、V 都是从 $x_t$ 的内容 + $t$ 的位置编码计算而来。但在排列中，当预测 $x_{z_t}$ 时，它的**内容**不能用于自身预测——但它的**位置信息**应该被后续的 token 看到。

**双流解决方案**：

| 表征类型 | 用途 | 包含位置？ | 包含内容？ |
|---------|------|----------|----------|
| **内容流** ($h$) | 作为其他位置的上下文 | ✓ | ✓ |
| **查询流** ($g$) | 用于预测当前自身 | ✓ | ✗（不能看自己的内容） |

```text
查询流 (Query Stream):  g_t = Attention(Q=位置_t, K,V=内容流中z_<t的内容)
内容流 (Content Stream): h_t = Attention(Q=内容_t+位置_t, K,V=所有z_≤t的内容)
```

**示例**：对于排列 $[3, 2, 1, 4]$，当预测到位置 1（"我"）时：
- 查询流 $g_1$：使用位置 1 的嵌入作为 Q，看到位置 3（"北京"）和位置 2（"爱"）的**内容流**作为 K,V
- 内容流 $h_1$：使用位置 1 的内容 + 位置嵌入作为 Q，看到所有之前位置的**内容流**作为 K,V

> 双流自注意力让 XLNet 既满足自回归的因果约束（预测时看不到内容），又能利用所有位置信息（位置编码在查询流中传递）。

#### Step 3 — Transformer-XL 的片段循环

XLNet 集成了 Transformer-XL 的 **Segment-Level Recurrence** 机制：

**问题**：标准 Transformer 的最大长度受限于显存（通常 512 token）。长文本被切分为固定长度的片段，片段之间没有信息流动。

**解决方案**：隐藏状态缓存 + 相对位置编码：

1. 处理当前片段时，缓存上一个片段的隐藏状态
2. 当前片段的注意力可以看向缓存的隐藏状态
3. 使用相对位置编码来处理跨片段的距离

$$\tilde{h}_{\tau+1}^{n-1} = [\text{SG}(h_\tau^{n-1}) \circ h_{\tau+1}^{n-1}]$$

其中 SG 是 stop-gradient，$h_\tau^{n-1}$ 是缓存的上一片段隐藏状态。

> Transformer-XL 机制让 XLNet 可以建模远超 512 token 的依赖关系——这对需要长距离语义理解的文档级 NLP 任务极为重要。

### 详细训练配置

| 参数 | XLNet-Base | XLNet-Large |
|------|-----------|------------|
| 层数 | 12 | 24 |
| 隐藏维度 | 768 | 1024 |
| 注意力头 | 12 | 16 |
| 参数量 | 110M | 340M |
| 序列长度 | 512 | 512 |
| 内存长度 | 384 | 384 |
| 优化器 | Adam | Adam |
| 学习率 | 1e-4 | 1e-4 |
| 学习率调度 | 线性衰减 | 线性衰减 |
| Warmup | 12.5K 步 | 12.5K 步 |
| Batch Size | 512 | 2048 |
| 训练步数 | 500K | 500K |
| 数据 | Wiki + Books + Giga5 + ClueWeb + CommonCrawl | 同 Base |

**数据集组合**（XLNet 的数据量约 126GB，介于 BERT 的 16GB 和 RoBERTa 的 160GB 之间）：

| 数据源 | 规模 |
|--------|------|
| BooksCorpus | ~5GB |
| English Wikipedia | ~13GB |
| Giga5 | ~16GB |
| ClueWeb 2012-B | ~19GB |
| Common Crawl | ~73GB |

### 预训练性能

| 任务 | BERT-Large | XLNet-Large |
|------|-----------|------------|
| SQuAD 2.0 (F1) | 80.2 | **87.9** |
| RACE (Acc) | 72.0 | **81.8** |
| GLUE | 82.1 | **87.9** |
| 文本分类 | 好 | 更好 |
| 文档排序 | 好 | 更好 |
| 文本生成 | 不支持 | 支持（自回归） |

XLNet 在发布时创造了几乎所有 NLP 基准的新纪录，特别是在需要长程依赖的任务（阅读理解 RACE）上大幅超越 BERT。

### 预训练的迁移价值

1. **弥合自回归和双向的鸿沟**：XLNet 证明了两者并非互斥
2. **排列语言模型的创新**：提供了一种优雅的方式让自回归获得双向上下文
3. **Transformer-XL 的价值验证**：片段循环机制在长文本 NLP 中至关重要
4. **为后续混合架构铺路**：启发了 XLNet-Large 和 ELECTRA、DeBERTa 等后续改进

> 不过 XLNet 的排列语言模型也带来了更高的计算成本——$T!$ 种排列只抽样一部分（实际训练中每步采样一个排列），且双流注意力需要两倍的内存和计算。这与后续 ELECTRA 的"更简单、更高效"形成了有趣的对比。
