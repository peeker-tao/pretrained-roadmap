
# SWAG (Swollen Vocabulary Adversarial Generation)

## 基本信息

- **论文**: [SWAG: A Large-Scale Adversarial Dataset for Grounded Commonsense Inference](https://arxiv.org/abs/1808.05326)
- **作者**: Rowan Zellers et al. (UW / Allen AI)
- **发表**: EMNLP 2018

## 创新点

1. **大规模常识推理数据集**: 11.3 万个视频片段，标注了 4 种可能的续集场景
2. **对抗性过滤**: 使用迭代对抗过滤生成高质量负例
3. **预训练评估基准**: 成为语言模型常识推理能力的标准测试集

## 核心原理

### 对抗性数据生成

1. 从视频中提取事件序列
2. 使用语言模型生成候选续集
3. 对抗性过滤：保留能"骗过"当前最佳模型的负例
4. 迭代多轮以提高难度

### 预训练评估

SWAG 常用于评估预训练语言模型的常识推理能力（不依赖外部知识）。

## 预训练相关（评估基准与数据策略）

### 核心思想：如果你给模型看一段视频"一个人走进厨房，打开冰箱"，模型应该能推断接下来最可能发生什么——"拿出食材"还是"开始跳舞"？SWAG 用对抗性过滤生成了 11.3 万个这样的常识推理题，成为检验 LLM 是否具有"常识"的金标准

SWAG 本身是一个**评估基准**而非预训练方法。但它的数据生成策略——对抗性过滤——对预训练数据筛选有深远影响。核心洞察：用当前最好的模型来"鉴别"哪些负例是真正困难的——然后用这些困难负例来测试下一轮模型。

> SWAG = 11.3 万视频场景 + 4 选 1 常识续集 + 对抗性过滤生成困难负例。评估 LLM 的常识推理能力。

### 数据生成流水线（Step by Step）

#### Step 1 — 视频场景提取

```text
数据来源: LSMDC (Large Scale Movie Description Challenge) 视频

提取: 从电影/电视剧中抽取场景
  → 视频片段 (10-30 秒)
  → 标注: 场景描述文本（"一个人走进厨房"）
```

#### Step 2 — 候选续集生成

```text
对于每个视频场景:

1. Ground Truth: 视频中真实发生的下一段 = 正确答案
   "打开冰箱，拿出蔬菜"

2. 语言模型生成候选:
   输入: "一个人走进厨房。接下来这个人..."
   LM (GPT-2 级别) 生成 N 个候选:
    候选 A: "打开冰箱" ✓ (正确答案)
    候选 B: "坐下看电视" ✗ (合理的错误)
    候选 C: "拿出扫把打扫" ✗ (部分相关但不对)
    候选 D: "开始跳舞" ✗ (虽然可能但离谱)
```

#### Step 3 — 对抗性过滤（核心创新）

SWAG 的"SW"代表 Swollen Vocabulary——但真正的创新在对抗性过滤：

```text
迭代 1:
  用初始模型 M_1 评估所有候选
  → 保留"骗过" M_1 的负例（M_1 选错的候选）
  → 这些是"对 M_1 来说困难的"

迭代 2:
  训练更强的模型 M_2
  → M_2 在新数据集上评估
  → 保留"骗过" M_2 的负例

迭代 N:
  重复直到负例质量足够高
```

| 属性 | 对抗性过滤 | 随机采样 |
|------|---------|---------|
| 负例质量 | **高（能骗过强模型）** | 低（对模型来说太简单） |
| 模型区分度 | **更好** | 差（天花板效应） |
| 迭代成本 | 高（需多轮过滤） | 低 |
| 数据集难度 | **持续升级** | 固定 |

#### Step 4 — SWAG 作为预训练评估

SWAG 常用于评估 LLM 预训练后的常识推理能力：

| 模型 | SWAG 准确率 | 训练方式 |
|------|----------|---------|
| 随机猜测 | 25.0% | — |
| BERT-base | 81.6% | MLM 预训练 |
| RoBERTa-large | 89.3% | 改进的 MLM |
| GPT-3 (175B) | 78.0% | AR 预训练 |
| LLaMA-7B | ~85% | AR 预训练 |
| **人类** | **88.0%** | 自然学习 |

> 人类的 SWAG 准确率约 88%——意味着这个"常识"任务对机器来说已经接近人类水平（但 GPT-3 175B 才 78%，说明规模不是唯一答案）。

#### Step 5 — 对抗性过滤对预训练数据策略的启发

| 预训练数据阶段 | SWAG 的启发 |
|-------------|------------|
| 数据筛选 | 用模型"困惑度"发现困难样本 → 加权采样 |
| 负例采样 | 简单负例价值低 → 需要"硬负例" |
| 数据去重 | 低质量重复 vs 高质量困难 → 保留后者 |
| 课程学习 | 先易后难 → 逐步引入对抗性样本 |
| 主动学习 | 用当前模型找出"最需要学"的样本 |

### SWAG 的评估配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 数据集规模 | 113K | 训练 73K, 验证 20K, 测试 20K |
| 选项数 | 4 | 4 选 1 |
| 视频来源 | LSMDC | 电影+电视剧 |
| 评估指标 | 准确率 | — |
| 随机基线 | 25% | — |

### 对预训练的实用价值

1. **对抗性数据生成的先驱**：用模型弱点生成困难样本→迭代提升数据集质量
2. **LLM 常识推理的金标准**：BERT、GPT、LLaMA 都用 SWAG 评估
3. **数据质量迭代范式**：数据筛选不是一次性工作→持续用模型 feedback 改进
4. **硬负例的价值**：证明了"区分易混淆选项"对预训练质量更重要
5. **对 DataComp/DCLM 的启发**：对抗性过滤的思想被数据筛选竞赛继承
