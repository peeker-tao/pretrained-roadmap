# CAN (Contrastive-Auxiliary Network)

## 基本信息

- **论文**: [CAN: Co-Training with Additive Normalization for Self-Supervised Learning](https://arxiv.org/abs/2212.00435)
- **作者**: Xiaoyi Dong et al.
- **发表**: arXiv, 2022

## 创新点

1. **对比 + 掩码协同训练**: 同时进行对比学习和掩码图像建模
2. **加性归一化**: 协调两个目标函数的梯度冲突
3. **互补表征学习**: 对比学习捕获全局语义，MIM 捕获局部细节

## 核心原理

### 协同训练框架

- **对比分支**: 使用 MoCo/SimCLR 风格的对比学习
- **MIM 分支**: 使用 MAE 风格的掩码重建
- **共享编码器**: 两个分支共享骨干网络

### 梯度协调

对比学习和 MIM 的梯度方向可能冲突，CAN 使用加性归一化机制协调两者的优化。

## 预训练方法

### 核心思想：对比学习擅长全局语义（"这是一只猫"），掩码重建擅长局部细节（"猫耳朵的纹理"）。CAN 的洞察是——两者不是"二选一"，而是互补的关系。让一个编码器同时做对比学习和掩码重建，"像用两只眼睛看世界"——一只看大图画（对比），一只看细节（重建）

CAN（Contrastive-Auxiliary Network）解决了自监督学习中一个重要但常被忽视的问题：不同预训练目标之间的梯度冲突。当对比学习想要"拉近同类、推远不同类"而掩码重建想要"还原像素"时，两个梯度方向可能打架。CAN 提出了加性归一化来协调两者。

> CAN = 共享编码器 + 两个分支（对比分支 + MIM 分支）+ 加性归一化（协调梯度冲突）。对比分支提供全局语义，MIM 分支提供局部细节，两者互补。

### 训练流水线（Step by Step）

#### Step 1 — 双分支协同设计

```text
输入图片 x
  ↓
共享编码器 (ViT / ResNet)
  ↓
┌─────────────────┬─────────────────┐
│  对比分支        │  MIM 分支        │
│  (全局语义)      │  (局部细节)      │
└─────────────────┴─────────────────┘
  ↓                  ↓
InfoNCE 损失       MSE 损失
  ↓                  ↓
┌───────────────────────────────────┐
│     加性归一化 (协调两路梯度)      │
└───────────────────────────────────┘
  ↓
总损失: L = L_contrast + L_mim
```

**两分支的分工**：

| 分支 | 方法 | 学到的特征 |
|------|------|----------|
| **对比分支** | SimCLR/MoCo 风格 | 全局语义、类别判别 |
| **MIM 分支** | MAE 风格 (掩码+重建) | 局部纹理、空间细节 |
| **互补效果** | 全局+局部 = 完整 | 下游任务表现更好 |

#### Step 2 — 对比分支

```text
对比分支 (标准 SimCLR/MoCo 风格):

图片 x → 两种增强 (x_1, x_2)
  → 编码器 → 投影头
  → 嵌入 1, 嵌入 2

正例: (x_1, x_2) — 同一图片的不同增强
负例: mini-batch 中其他图片

损失: InfoNCE
  L_contrast = -log(exp(sim(z_1,z_2)/τ) / Σ exp(sim(z_1,z_j)/τ))
```

#### Step 3 — MIM 分支（掩码重建）

```text
MIM 分支 (MAE 风格):

图片 → 随机 mask 75% patch
  → 编码器处理可见 patch
  → 解码器重建被遮 patch

损失: MSE (仅被遮区域)
  L_mim = Σ_{i∈masked} ||patch_pred_i - patch_true_i||²
```

#### Step 4 — 梯度冲突与加性归一化

**梯度冲突的问题**：

```text
情况: 图片中有一只猫

对比分支的梯度 ↓:
  "把猫的特征推到和其他动物不同方向"
  → 编码器倾向于: 紧凑、全局的判断

MIM 分支的梯度 ↓:
  "保留猫毛的细节纹理和空间布局"
  → 编码器倾向于: 保留所有细节

冲突: 两个梯度方向可能不完全一致!
```

**CAN 的加性归一化解决方案**：

```text
标准联合:
  总损失梯度 = ∇L_contrast + ∇L_mim
  问题: 两个梯度可能互相抵消或冲突

CAN 加性归一化:
  总损失 = L_contrast + α · L_mim
  
  α = 动态调整：
    α = ||∇L_contrast|| / ||∇L_mim||  (平衡大小)
  或
    α = 固定的 0.5-1.0

目的: 让两个分支的梯度规模相近，
     不会一个主导另一个
```

#### Step 5 — 完整训练配置

| 参数 | CAN | 说明 |
|------|-----|------|
| 预训练数据 | ImageNet (无标签) | — |
| 编码器 | ViT-B / ViT-L | 共享 |
| 对比分支 | MoCo v3 风格 | — |
| MIM 分支 | MAE 风格 | 75% mask |
| 对比损失 | InfoNCE | — |
| MIM 损失 | MSE (被遮区域) | — |
| 权重系数 α | 0.5-1.0 | 平衡两分支 |
| 优化器 | AdamW | — |
| 学习率 | 1.5e-4 | — |
| 训练 Epoch | 300-800 | — |

### CAN vs 纯对比学习 vs 纯 MIM

| 维度 | 纯对比 (MoCo) | 纯 MIM (MAE) | **CAN（两者结合）** |
|------|-------------|------------|-----------------|
| 全局语义 | ✓✓ | ✗ | **✓✓** |
| 局部细节 | ✗ | ✓✓ | **✓✓** |
| 分类任务 | 好 | 中 | **更好** |
| 分割/检测 | 一般 | 好 | **更好** |
| 梯度冲突 | 无 | 无 | **需协调** |
| 训练效率 | 中 | 高 | **中（双分支开销）** |

### 预训练的实用价值

1. **SSL 目标函数的融合范式**：对比+生成不是矛盾的，而是互补的
2. **梯度协调机制**：加性归一化为多目标训练提供了简单有效的方案
3. **统一表征学习的方向**：全图+细节 → 更好的下游任务迁移
4. **超越单一方法**：分类、检测、分割全面优于纯对比或纯 MIM
5. **对 iBOT/DINOv2 的启发**：后续的统一 SSL 方法都融合了对比+重建
