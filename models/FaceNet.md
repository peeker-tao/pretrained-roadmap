# FaceNet

## 基本信息

- **论文**: [FaceNet: A Unified Embedding for Face Recognition and Clustering](https://arxiv.org/abs/1503.03832)
- **作者**: Florian Schroff, Dmitry Kalenichenko, James Philbin (Google)
- **发表**: CVPR 2015

## 创新点

1. **Triplet Loss**: 提出三元组损失（Anchor + Positive + Negative），端到端学习人脸嵌入
2. **统一嵌入空间**: 直接学习从人脸图像到欧几里得空间的映射，距离即人脸相似度
3. **高精度**: 在 LFW 数据集上达到 99.63% 的准确率
4. **高效嵌入**: 128 维嵌入向量即可高效表示人脸

## 核心原理

### Triplet Loss

在嵌入空间中，希望同一人的不同照片（正例对）距离近，不同人的照片（负例对）距离远。Triplet Loss 定义为：

$$\mathcal{L} = \sum_i [||f(x_i^a) - f(x_i^p)||_2^2 - ||f(x_i^a) - f(x_i^n)||_2^2 + \alpha]_+$$

其中 $x^a$ 是锚点 (Anchor)，$x^p$ 是同一人的正例，$x^n$ 是不同人的负例，$\alpha$ 是边际 (margin)。

### 三元组选择

三元组选择对训练至关重要，有两种策略：
- **难例挖掘 (Hard Negative Mining)**: 选择距离锚点最近的负例
- **随机选择**: 随机选取三元组（效率低，需要大 batch）

FaceNet 采用 **在线难例挖掘**：在 mini-batch 内选择最难的负例。

## 预训练方法

### 核心思想：不要学习分类器——直接学一个"距离就是相似度"的嵌入空间：同一人的脸靠近，不同人的脸远离

FaceNet 是度量学习在人脸识别领域的开创性工作。它的核心范式变革：**不再把人脸识别建模为分类问题（N 个身份 = N 类 Softmax），而是建模为度量学习问题（学习一个嵌入空间，空间中欧氏距离直接代表人脸的相似度）**。

> FaceNet = Triplet Loss（锚点-正例-负例三元组）+ 在线难例挖掘 + 128 维嵌入。不再需要维护巨大的分类层——嵌入空间本身就能处理任意数量的身份。

### 训练流水线（Step by Step）

#### Step 1 — 三元组（Triplet）的定义

每个训练样本是一个三元组 $(x^a, x^p, x^n)$：

| 角色 | 含义 | 示例 |
|------|------|------|
| Anchor ($x^a$) | 锚点人脸 | 张三的照片1 |
| Positive ($x^p$) | 正例（同一个人） | 张三的照片2（不同角度/光照） |
| Negative ($x^n$) | 负例（不同人） | 李四的照片 |

#### Step 2 — Triplet Loss

$$\mathcal{L} = \sum_i \left[ \|f(x_i^a) - f(x_i^p)\|_2^2 - \|f(x_i^a) - f(x_i^n)\|_2^2 + \alpha \right]_+$$

其中 $[\cdot]_+ = \max(0, \cdot)$，$\alpha$ 是边际（margin）。

**Triplet Loss 的三个项**：

| 项 | 含义 | 优化方向 |
|----|------|---------|
| $\|f(x^a) - f(x^p)\|_2^2$ | Anchor-正例距离 | **越小越好**（同一人脸的特征要接近） |
| $\|f(x^a) - f(x^n)\|_2^2$ | Anchor-负例距离 | **越大越好**（不同人脸的特征要远离） |
| $\alpha$ | 边际 | **安全距离**——正例要比负例近至少 $\alpha$ |

**损失为 0 的条件**：

$$\|f(x^a) - f(x^n)\|_2^2 > \|f(x^a) - f(x^p)\|_2^2 + \alpha$$

即负例距离 > 正例距离 + 边际 → 完美分离 → 损失为零。

> 类比：Triplet Loss 像一个严格的训练教官：看到"张三的照片1"和"张三的照片2"→ "太远了！给我靠近！"；看到"张三"和"李四"→ "还是不够远！再拉开！"。要同时满足"自己人靠拢，外人大距离"。

#### Step 3 — 在线难例挖掘（Online Hard Negative Mining）

这是 FaceNet 训练成功的关键。随机选三元组的问题是：**大多数三元组太简单了**（正例很近、负例很远，损失为 0，梯度为 0）。

**难例挖掘策略**：

```text
在一个 mini-batch（B 个人，每人 K 张照片）中:
1. 计算所有嵌入对的距离
2. 对于每个 Anchor:
   a. 选择最难的正例: argmax ‖f(x^a) - f(x^p)‖² （最不像的同一人）
   b. 选择最难的负例: argmin ‖f(x^a) - f(x^n)‖² （最像的不同人）
3. 仅对难例计算 Triplet Loss
```

**为什么需要在线挖掘？**
- 离线挖掘：在固定数据集上预选三元组 → 很快过时（嵌入空间在变化）
- 在线挖掘：在每个 mini-batch 内动态选择 → 始终使用当前模型下最难的样本

#### Step 4 — Mini-batch 的构造

| 参数 | 典型值 | 说明 |
|------|--------|------|
| Batch 中的人数 | 40-50 | 选择中等数量的不同人 |
| 每人照片数 | 5-10 | 保证每个人有足够多正例对 |
| 总 batch 大小 | 1800 | 40人 × 45张（近似） |

**为什么用这种"身份-batch"结构？**
- 保证每个 batch 内正例和负例都充足
- 难例挖掘在 batch 内完成（不需要额外的队列或内存库）

#### Step 5 — 完整训练流程

```text
1. 人脸图像 → 人脸检测 + 对齐 → 220×220 RGB
2. 构建 mini-batch: 选 40 人，每人 ~45 张
3. 前向传播:
   → CNN（Zeiler&Fergus 或 Inception）→ L2 归一化 → 128 维嵌入
4. 计算 batch 内所有嵌入对的欧氏距离
5. 在线难例挖掘: 为每个 Anchor 选 hardest positive + hardest negative
6. 计算 Triplet Loss，L = [d(a,p) - d(a,n) + α]_+
7. 反向传播（梯度仅通过难例）
8. 异步训练: 多个 GPU worker 独立采样三元组
```

### FaceNet 对比 margin-based 方法

| 维度 | FaceNet (Triplet Loss) | ArcFace/CosFace |
|------|----------------------|-----------------|
| 损失类型 | 度量学习 | 分类损失变体 |
| 需要分类层？ | ✗（嵌入即结果） | ✓（需维护 N×d 权重矩阵） |
| 训练稳定性 | 依赖难例挖掘质量 | **更稳定** |
| 大规模身份 | 天然支持（无分类层） | 需要巨大分类层 |
| 样本效率 | 低（大部分三元组无梯度） | 高 |
| 精度 | ~99.6% LFW | **~99.8% LFW** |

> FaceNet 开创了嵌入空间的直接学习方法，但训练效率低（需要精心构造三元组）。ArcFace 后来通过 margin-based 分类损失获得了更好的性能和稳定性——但 FaceNet 的"距离即相似度"的哲学影响了整个度量学习领域。

### 详细训练配置

| 参数 | FaceNet 配置 | 说明 |
|------|------------|------|
| 数据集 | Google 内部 ~200M 人脸 | 大规模私有数据 |
| 输入尺寸 | 220×220 RGB | 人脸对齐后 |
| 架构 | NN2（基于 Inception） | 也尝试了 Zeiler&Fergus |
| 嵌入维度 | 128 | 紧凑嵌入 |
| 边际 $\alpha$ | 0.2 | 正例-负例最小距离差 |
| 优化器 | SGD + AdaGrad | — |
| Batch 结构 | 40 人 × 45 张 | 每 batch ~1800 张 |
| 难例挖掘 | 在线（batch 内） | hardest positive + hardest negative |
| 训练方式 | 多 GPU 异步 | 1000-2000 GPU workers |

### 预训练的实用价值

1. **度量学习的里程碑**：将人脸识别从"分类问题"重构为"嵌入学习问题"
2. **128 维紧凑嵌入**：一个 128 维向量 = 一个人脸的完整身份表征——存储和检索效率极高
3. **开放集识别的能力**：不需要预定义身份列表——新人脸只需计算嵌入距离
4. **Triplet Loss 的推广**：被用于行人重识别、图像检索、跨模态检索
5. **难例挖掘的方法论**：在线难例挖掘成为度量学习的标准技术

FaceNet 区别于传统分类预训练，它不学习具体类别的分类器，而是学习一个通用嵌入空间。这种 **度量学习预训练** 范式特别适合人脸识别、行人重识别等需要细粒度相似度比较的下游任务。
