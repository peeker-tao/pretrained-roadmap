# BEiT (BERT Pre-Training of Image Transformers)

## 基本信息

- **论文**: [BEiT: BERT Pre-Training of Image Transformers](https://arxiv.org/abs/2106.08254)
- **作者**: Hangbo Bao, Li Dong, Songhao Piao, Furu Wei (Microsoft)
- **发表**: ICLR 2022

## 创新点

1. **离散视觉编码预测**: 将 MLM 范式引入视觉——预测 DALL-E 的离散编码 (Visual Token)
2. **两阶段预训练**: VQ-VAE 量化 + BERT 式掩码预测
3. **对标 BERT**: 在视觉上实现了与 NLP 中 BERT 完全对等的预训练范式

## 核心原理

### 两阶段训练

**第一阶段: 学习视觉词汇表（VQ-VAE）**
- 训练一个 VQ-VAE 将图像 patch 量化为离散代码
- 代码本大小: 8192

**第二阶段: 掩码图像建模**
- 随机掩码 40% 的图像 patch
- 模型根据可见 patch 预测被掩码位置的离散代码
- 预测目标为多类分类（8192 类）

### BEiT vs. MAE

| 方面 | BEiT | MAE |
|------|------|-----|
| 重建目标 | 离散代码（VQ-VAE） | 原始像素 |
| 掩码率 | 40% | 75% |
| 架构 | 对称编码器 | 非对称编码器-解码器 |
| 复杂度 | 需要两阶段训练 | 端到端训练 |

## 预训练方法

### 核心思想：把图片变成"词"，然后做 BERT 式的完形填空

BEiT 的目标是让图像的预训练和 BERT 完全对等：BERT 预测被掩码的**离散词**，BEiT 预测被掩码的**离散视觉 token**。为了实现这个目标，BEiT 需要先"发明"一个视觉词汇表——这通过第一阶段的 VQ-VAE（Vector Quantized-Variational AutoEncoder）完成。

> 类比：想象你要教一个外国学生中文完形填空，但中文没有字母——你先得发明一套注音符号，然后用这些符号来做填空。BEiT 的两阶段训练就是"发明视觉字母 → 做视觉填空"。

### 训练流水线（Step by Step）

#### 第一阶段：学习视觉"词汇表"（dVAE / VQ-VAE）

**目的**：将连续的图像 patch 转换为离散的"视觉词"。

**Step 1 — 编码**：dVAE 编码器将图像 patch 编码为连续向量

**Step 2 — 量化**：将每个连续向量映射到"码本"中最接近的条目：

$$\text{token}_i = \arg\min_{k} \| z_i - e_k \|_2$$

其中 $e_k$ 是码本中第 $k$ 个可学习的向量（$k \in \{1, 2, ..., K\}$，$K=8192$）。

**Step 3 — 解码重建**：dVAE 解码器从量化后的离散 token 重建原始图像。损失函数是重建损失 + 码本学习损失的组合：

$$\mathcal{L}_{\text{dVAE}} = \|x - \hat{x}\|^2 + \|\text{sg}[z] - e\|_2^2 + \beta\|z - \text{sg}[e]\|_2^2$$

其中 $\text{sg}[\cdot]$ 是 stop-gradient 操作。第一项是重建损失，后两项分别让码本向量向编码器输出靠拢、编码器输出向码本向量靠拢。

> 训练完成后，dVAE 的码本（8192 个向量）就成为了"视觉词汇表"——图像的每个 patch 都可以被量化为 8192 个"词"中的一个。

#### 第二阶段：掩码图像建模（MIM）

**Step 1 — 图像分块 + 离散化**：

将输入图像分割为 $14 \times 14 = 196$ 个 $16 \times 16$ 的 patch，然后用第一阶段的 dVAE 将每个 patch 量化为离散 token $z_i \in \{1, ..., 8192\}$。

**Step 2 — 随机掩码**：

随机掩码 40% 的图像 patch（约 78 个），将这些 patch 替换为可学习的 $[MASK]$ token。

**Step 3 — ViT 编码**：

完整的 196 个 token（包含可见 patch embedding + 被掩码位置的 $[MASK]$ token）+ 位置编码，送入 ViT 编码器。

**Step 4 — 预测离散 token**：

对于每个被掩码的位置，ViT 的输出通过一个分类头，预测该位置的离散 token（8192 类）：

$$\mathcal{L}_{\text{MIM}} = -\sum_{i \in \text{masked}} \log P(z_i | x_{\backslash i})$$

> 这与 BERT 的 MLM 完全对等——BERT 预测被掩码的词（3 万类），BEiT 预测被掩码的视觉 token（8192 类）。

### 为什么这样设计——深入理解

#### 1. 为什么用离散 token 而不是像素回归？

这是 BEiT 与 MAE 的根本区别：

| 方法 | 重建目标 | 损失函数 | 信息粒度 |
|------|---------|---------|---------|
| **BEiT** | 离散视觉 token（8192 类） | 交叉熵 | 语义级 |
| **MAE** | 归一化像素值 | MSE | 像素级 |

**BEiT 的优势**：离散 token 迫使模型进行**语义级别的推理**——8192 个 token 类别对应 8192 种"视觉概念"，模型需要理解"这个被遮住的区域最可能是哪种视觉概念"。

**BEiT 的劣势**：需要额外的第一阶段 dVAE 训练（增加了工程复杂度），而且 dVAE 的码本质量直接影响预训练效果。

#### 2. 为什么掩码率是 40%？

BEiT 使用 40% 的掩码率（而不是 BERT 的 15% 或 MAE 的 75%），这是模态特性的折中：

| 掩码率 | 说明 |
|-------|------|
| 15% | 对于图像来说太简单，模型可以从周围像素"插值"恢复 |
| 40% | BEiT 的经验最优——足够挑战但不至于无解 |
| 75% | MAE 的掩码率，但 BEiT 处理全部 token（无效率优势），75% 掩码率会让计算很重 |

#### 3. BEiT vs MAE 的结构差异

| 维度 | BEiT | MAE |
|------|------|-----|
| 编码器输入 | 全部 196 个 token | 仅 49 个可见 token |
| $[MASK]$ token | 编码器处理（256 维可学习向量） | 仅解码器处理 |
| 重建方式 | 分类头预测离散 token | 解码器重建像素 |
| 计算量 | 高（编码器处理全部 token） | 低（编码器仅处理可见 token） |
| 工程复杂度 | 高（需要 dVAE 预训练） | 低（端到端） |
| 可扩展性 | 中等 | 好（轻量编码器） |

### 详细训练配置（第二阶段 MIM）

| 参数 | 值 | 说明 |
|------|-----|------|
| 编码器 | ViT-B/16 | 12层, 768维, 12头 |
| 掩码率 | 40% | 约 78 个/196 个 patch 被掩码 |
| 码本大小 $K$ | 8192 | 8192 个"视觉词" |
| 损失函数 | 交叉熵（仅掩码位置） | 8192 类分类 |
| 优化器 | AdamW | β₁=0.9, β₂=0.999 |
| 学习率 | 1.5e-3 | 余弦衰减 |
| 权重衰减 | 0.05 | — |
| Warmup | 10 epoch | — |
| Batch Size | 2048 | — |
| Epoch | 300 | — |
| 数据增强 | RandomResizedCrop + 水平翻转 | 基础增强 |

### 预训练性能

| 方法 | 编码器 | 线性探测 | 微调 |
|------|--------|---------|------|
| BEiT | ViT-B/16 | 37.6% | 83.2% |
| MAE | ViT-B/16 | 68.0% | 83.6% |
| DINO | ViT-B/16 | 78.2% | 82.8% |

BEiT 的线性探测性能非常低（37.6%），但微调性能很不错（83.2%）。原因是：离散 token 预测的特征信息被"压缩"到了各层中，需要微调来释放——而 DINO 的特征更"线性可分"，适合线性探测。

### 迁移学习的实用价值

1. **第一个真正对等 BERT 的视觉预训练**：在概念层面建立了 NLP 和 CV 预训练的统一框架
2. **启发了大量后续工作**：MaskFeat、PeCo、CAE、SimMIM 等都在不同程度上借鉴了 BEiT 的思路
3. **离散 token 预测的独特优势**：在某些需要语义理解的任务上（如 VQA），离散 token 预测可能优于像素回归
