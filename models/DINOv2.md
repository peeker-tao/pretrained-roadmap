# DINOv2

## 基本信息

- **论文**: [DINOv2: Learning Robust Visual Features without Supervision](https://arxiv.org/abs/2304.07193)
- **作者**: Maxime Oquab et al. (Meta)
- **发表**: arXiv, 2023

## 创新点

1. **大规模数据 + 蒸馏**: 在精心筛选的 1.42 亿张图片上训练
2. **Registers 机制**: 解决 ViT 特征不连续问题
3. **特征质量达到监督预训练水平**: 在多种下游任务上匹敌甚至超越监督预训练

## 核心原理

### 数据筛选

1. 从海量网络数据中筛选出 1.42 亿张高质量图片
2. 使用多种质量指标（美学分数、NSFW 过滤、去重）
3. 设计专用的数据清洗流程

### Registers

ViT 的 patch 特征在某些区域会出现**伪影**问题，DINOv2 引入额外的可学习寄存器 token 来吸收这些伪影。

### 联合训练策略

DINOv2 联合了多种自监督目标：
1. **DINO 损失**: 自蒸馏
2. **iBOT 损失**: 掩码图像建模
3. **SwiAV 损失**: 聚类对比

## 预训练方法

### 核心思想：做自监督学习的"GPT-3 时刻"

DINOv2 的目标是在自监督学习领域做一件类似 GPT-3 在 NLP 领域做的事：**用海量数据和超大模型证明规模的力量**。它的口号是"自监督学习也能达到甚至超越监督预训练的水平"——关键在于数据质量和数据规模的结合。

### 训练流水线（Step by Step）

#### Step 1 — 精心筛选的大规模数据：LVD-142M

DINOv2 最核心的工程贡献是其**数据处理流水线**。从海量网络图片出发，经过多轮筛选，最终得到 1.42 亿张高质量图片：

**筛选流程**：
1. **种子数据**：从约 25 个精选数据集中选取高质量图片
2. **扩展下载**：用种子图片在互联网上检索相似图片，爬取约 12 亿张
3. **去重过滤**：图像层面去重（SSIM/哈希）+ 跨源去重
4. **质量评分**：美学评分 + 内容安全过滤（NSFW）+ 模糊/过曝检测
5. **均衡采样**：确保概念多样性，避免某些类别过度主导

> 最终 LVD-142M（Large Visual Dataset - 142M）的图片质量远高于未经筛选的 LAION-2B 等数据集，这是 DINOv2 性能的关键基础。

#### Step 2 — 联合损失：三个自监督目标一起训练

DINOv2 的创新之一是将多种自监督目标**联合训练**：

**DINO 损失（自蒸馏）**：

$$\mathcal{L}_{\text{DINO}} = \sum_{x \in \{\text{globals}\}} \sum_{x' \in \{\text{locals}\}} H(P_t(x), P_s(x'))$$

学生网络（局部裁剪）匹配教师网络（全局裁剪）的输出。

**iBOT 损失（掩码图像建模）**：

对图像 patch 做随机掩码，用教师网络对被掩码位置的输出作为目标，学生网络预测：

$$\mathcal{L}_{\text{iBOT}} = \sum_{i \in \text{masked}} H(P_t^{\text{patch}}(x_i), P_s^{\text{patch}}(\text{mask}_i))$$

**联合损失**：

$$\mathcal{L}_{\text{DINOv2}} = \lambda_1 \mathcal{L}_{\text{DINO}} + \lambda_2 \mathcal{L}_{\text{iBOT}}$$

DINO 损失学习全局语义（图像级别），iBOT 损失学习局部细节（patch 级别）——两者互补，使得特征同时具备全局理解力和局部细节力。

#### Step 3 — Registers：解决 ViT 特征伪影

DINOv2 发现了一个重要问题：ViT 的 patch 特征在图像的某些区域会出现**非自然的伪影**（特征图中出现突然的"跳跃"）。

**解决方案：引入 Register Token**

在 patch embedding 之后、送入 Transformer 之前，插入 $R$ 个额外的、可学习的 "register" token（$R=4$）：

```text
输入序列 = [CLS] + [Patch1, Patch2, ..., PatchN] + [Reg1, Reg2, Reg3, Reg4]
```

这些 register token 不参与后续的图像语义分析——它们的角色是**吸收**那些不自然的表征信息，让 patch token 专注于"真正的"语义。

> 注册 token 的效果类似于"垃圾回收站"——把不需要的、杂乱的信号收集起来，不让它们污染正常的 patch 特征。这显著提升了特征的连续性和平滑性。

#### Step 4 — KoLeo 正则化器

DINOv2 引入了一个巧妙的数学技巧：KoLeo 正则化器（Kozachenko-Leonenko entropy estimator），确保所有特征在空间中均匀分布：

$$\mathcal{L}_{\text{KoLeo}} = -\frac{1}{n} \sum_{i=1}^n \log \min_{j \neq i} \| f(x_i) - f(x_j) \|$$

直觉解释：如果一些特征聚在一起、另一些特征稀疏分布，特征空间的利用率就不高。KoLeo 正则化**推开所有特征点**，让它们均匀地填充整个特征空间——这增强了特征的整体表达能力。

### 详细训练配置

#### 教师网络蒸馏

DINOv2 不直接在 ViT-g（10 亿参数）上做标准训练（计算量太大），而是使用**蒸馏策略**：

1. **教师模型**：在 LVD-142M 上预训练 ViT-g/14（1B 参数）
2. **学生模型**：各种尺寸的 ViT（ViT-S, ViT-B, ViT-L, ViT-g）
3. **蒸馏方式**：学生在教师预训练特征的基础上进一步训练，学习教师的输出

| 参数 | ViT-L（学生） | ViT-g（教师/学生） |
|------|-------------|-----------------|
| 层数 | 24 | 40 |
| 隐藏维度 | 1024 | 1536 |
| 注意力头 | 16 | 24 |
| Patch 大小 | 14×14 | 14×14 |
| 参数量 | 300M | 1.1B |
| 优化器 | AdamW | AdamW |
| 学习率 | 5e-4 | 5e-4 |
| 学习率调度 | 余弦衰减 | 余弦衰减 |
| 权重衰减 | 0.04 → 0.4 | 0.04 → 0.4 |
| Batch Size | 2048 | 2048 |
| Epoch | 取决于模型大小 | — |
| Warmup | 20 epoch | 20 epoch |
| 教师动量 | 0.994 | — |
| Register Token | $R=4$ | $R=4$ |
| iBOT 掩码率 | 50% | — |
| KoLeo 权重 | 0.1 | — |

### 为什么 DINOv2 能超越监督预训练？

DINOv2 的成功来自几个关键因素的叠加：

1. **数据质量 > 数据量**：精心筛选的 1.42 亿图片 > 粗制滥造的 10 亿图片
2. **联合学习的互补性**：DINO（全局语义）+ iBOT（局部细节）= 全面特征
3. **规模法则**：10 亿参数的 ViT-g + 大规模数据 = 涌现出监督学习没有的能力
4. **工程细节的极致打磨**：Register token、KoLeo 正则化、教师蒸馏等

### 预训练性能

| 方法 | 编码器 | ImageNet Top-1（线性探测） |
|------|--------|-------------------------|
| 监督 | ViT-H/14 | 82.9% |
| MAE | ViT-H/14 | 76.0% |
| DINO | ViT-B/8 | 80.1% |
| **DINOv2** | ViT-L/14 | **83.5%** |
| **DINOv2** | ViT-g/14 | **84.5%** |

除此之外，DINOv2 的特征在以下任务上表现出色：
- **深度估计**：只需在 DINOv2 特征上加一个小解码器即可做单目深度估计
- **语义匹配**：DINOv2 特征可以直接用于跨图像的语义对应（dense matching）
- **少样本分类**：在极少的标注样本下即可达到高精度

### 迁移学习的实用价值

1. **自监督学习的 GPT-3 时刻**：首次在多个维度上全面超越监督预训练
2. **通用特征提取器**：一个模型适配分类、分割、深度估计、检索等多种任务
3. **工业级可用性**：Meta 提供了开箱即用的预训练权重
4. **特征质量新高度**：Register token 的设计启发了后续 ViT 架构的改进
