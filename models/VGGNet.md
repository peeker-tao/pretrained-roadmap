# VGGNet

## 基本信息

- **论文**: [Very Deep Convolutional Networks for Large-Scale Image Recognition](https://arxiv.org/abs/1409.1556)
- **作者**: Karen Simonyan, Andrew Zisserman
- **发表**: ICLR 2015

## 创新点

1. **深度与性能的关系**: 系统性地证明了增加网络深度（16–19 层）可以显著提升性能
2. **统一的小卷积核**: 全部使用 3×3 卷积核，替代 AlexNet 的大卷积核（11×11, 5×5）
3. **简洁一致的设计哲学**: 整个网络使用相同的卷积配置（3×3 conv + 2×2 pooling），简化了设计
4. **感受野的堆叠**: 两层 3×3 卷积等效于一层 5×5 的感受野，但参数量更少且非线性更强

## 核心原理

VGGNet 的核心思想是：**通过堆叠多个小卷积核来代替大卷积核**，在保持相同感受野的同时增加网络深度和非线性表达能力。

- 两个 3×3 卷积堆叠 → 等效 5×5 感受野（参数量：2×9C² vs 25C²）
- 三个 3×3 卷积堆叠 → 等效 7×7 感受野（参数量：3×9C² vs 49C²）

VGG 有多个变体（VGG11, VGG13, VGG16, VGG19），其中最常用的是 VGG16 和 VGG19。

## 预训练方法

### 核心思想：更深的"3×3 堆叠"网络 = 更强大的视觉特征

VGGNet 的设计哲学是极简主义：**所有卷积层都用 3×3，所有池化层都用 2×2，然后拼命加深**。这种设计在同一时期与 GoogLeNet 的 Inception 模块形成鲜明对比——一个是"复杂模块"路线，一个是"简单重复加深度"路线。结果证明两者都有效。

> VGGNet 的预训练揭示了：在 ImageNet 尺度下，深度（16-19 层）与性能呈正相关——前提是有恰当的初始化和训练策略。两个 3×3 卷积等价于一个 5×5 卷积的感受野，但非线性更强、参数更少。

### 训练流水线（Step by Step）

#### Step 1 — 递增式初始化策略

训练 VGGNet 的核心困难：**太深了会梯度消失**。

VGG 的解决方案：**先训练浅层网络，再用浅层权重初始化深层网络**。

```text
Stage 1: 训练 VGG-11（11层，相对较浅，梯度尚可）
Stage 2: 用 VGG-11 的权重初始化 VGG-16 的前 11 层
         新增的 5 层用随机初始化
Stage 3: 继续训练 VGG-16
Stage 4: 用 VGG-16 初始化 VGG-19，训练最终模型
```

> 这种"渐进式训练"是早期的课程学习思想——让模型先解决简单问题，再逐步加难。现代训练中，BatchNorm 的普及使这种方法不再必要，但在 2014 年这是一个关键创新。

#### Step 2 — 多尺度训练（Scale Jittering）

VGGNet 使用了当时最先进的多尺度训练方法：

$$S \in [S_{\min}, S_{\max}], \quad \text{通常 } S_{\min}=256, S_{\max}=512$$

**训练过程**：
1. 对每张图，随机选择一个缩放尺寸 $S$
2. 将图像缩放到 $S \times S$
3. 从缩放后的图像中随机裁剪 224×224

**为什么有效？** 模型被迫学习尺度不变特征——同一只猫可以在 50 像素内，也可以在 200 像素内。

| 训练方案 | $S$ 范围 | 效果 |
|---------|---------|------|
| 固定尺度 | $S=256$ | 基准 |
| 单尺度抖动 | $S \in [256, 512]$ | **+0.5% top-1** |
| 多尺度（全范围） | $S \in [256, 512]$ 多尺度 | **最强** |

#### Step 3 — 测试时多裁剪（Multi-Crop Evaluation）

测试时的增强策略——Dense Evaluation：

1. 将测试图像缩放到多个尺度（如 256, 384, 512）
2. 在每个尺度上滑窗裁剪多个 224×224 的区域
3. 对所有裁剪的预测进行平均

$$\hat{y}_{\text{final}} = \frac{1}{N \times M} \sum_{s=1}^M \sum_{c=1}^N f(x_{s,c})$$

其中 $M$ 是尺度数，$N$ 是每尺度的裁剪数。

> 这本质上是一种"测试时集成"——同一模型在不同视角看同一张图，多数表决得到最终答案。代价是推理时间增加 50-100×。

#### Step 4 — 损失函数与优化

**损失**：标准多类交叉熵

$$\mathcal{L} = -\sum_{c=1}^{1000} y_c \log(p_c), \quad p_c = \frac{\exp(z_c)}{\sum_j \exp(z_j)}$$

**优化器**：SGD + Momentum (0.9)

| 参数 | VGG 配置 | 说明 |
|------|---------|------|
| 初始学习率 | 1e-2 | — |
| 学习率衰减 | 验证集准确率停滞时 ×0.1 | 手动监控衰减 |
| 总衰减次数 | 3 次 | 典型值 |
| Batch Size | 256 | 较大 batch 稳定训练 |
| 权重衰减 | 5e-4 | $L_2$ 正则化 |
| Dropout | 0.5（仅 FC 层） | 全连接层防过拟合 |
| 动量 | 0.9 | SGD 动量 |

#### Step 5 — 为什么全连接层需要 Dropout？

VGGNet 约 90% 的参数集中在三个全连接层中（FC 4096→4096→1000）。Dropout 在 4096 维的全连接层中至关重要：

- FC 层参数量 = 4096×4096 ≈ 16M（仅一层）
- 不带 Dropout，FC 层极易对训练集过拟合
- Dropout 在 4096 维空间做随机"特征遮蔽"，迫使网络学习冗余表征

### 为什么 3×3 堆叠优于大卷积核？

| 方案 | 感受野 | 参数量 | 非线性层数 |
|------|--------|--------|-----------|
| 1 层 5×5 | 5×5 | $25C^2$ | 1 |
| 2 层 3×3 | 5×5 | $18C^2$ | **2** |
| 1 层 7×7 | 7×7 | $49C^2$ | 1 |
| 3 层 3×3 | 7×7 | $27C^2$ | **3** |

> 两层 3×3 不仅参数量少 28%（18 vs 25），还多了一层 ReLU 非线性——这增加了网络的表达能力。预训练时，更多的非线性意味着更丰富的特征空间。

### 详细训练配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 数据集 | ImageNet ILSVRC 2012 | 130 万图 / 1000 类 |
| 输入尺寸 | 224×224×3 | 从缩放图像裁剪 |
| 架构变体 | VGG-11/13/16/19 | 仅深度不同 |
| 优化器 | SGD + Momentum (0.9) | — |
| 学习率 | 1e-2 → 1e-5（经 3 次衰减） | 手动监控衰减 |
| Batch Size | 256 | — |
| 权重衰减 | 5e-4 | — |
| Dropout | 0.5 | 仅 FC6/FC7 |
| 训练尺度 | $S \in [256, 512]$ | 多尺度 |
| 参数初始化 | 递进式（浅→深） | 防梯度消失 |
| Epoch | ~74 | 每次衰减约 18-20 epoch |

### 预训练性能

| 模型 | 深度 | ImageNet Top-5 错误率 | 参数量 |
|------|------|----------------------|--------|
| AlexNet | 8 | 15.3% | 60M |
| VGG-11 | 11 | 10.4% | 133M |
| VGG-13 | 13 | 9.9% | 133M |
| VGG-16 | 16 | **8.8%** | 138M |
| VGG-19 | 19 | **9.0%** | 144M |

> VGG-16 和 VGG-19 性能接近——说明在 ImageNet 尺度下，16-19 层已经接近深度增长的收益递减点。

### 预训练的实用价值

1. **"深度 = 性能"的实证**：首次系统性证明增加网络深度（16-19 层）能稳定提升性能
2. **标准化特征提取器**：VGG16/19 预训练权重成为 R-CNN、Fast R-CNN、FCN 等工作的标准 backbone
3. **风格迁移的标配**：Gatys 的风格迁移算法使用 VGG-19 的中间层特征，成为艺术风格迁移的经典范式
4. **简洁设计的持久影响力**：统一 3×3 + 2×2 pooling 的设计理念影响深远
5. **多尺度训练/测试的标杆**：尺度抖动和 Dense Evaluation 被后续工作广泛采用
