# VideoCLIP

## 基本信息

- **论文**: [VideoCLIP: Contrastive Pre-training for Zero-shot Video-Text Understanding](https://arxiv.org/abs/2109.14084)
- **作者**: Hu Xu et al. (Meta / CMU)
- **发表**: EMNLP 2021

## 创新点

1. **视频-文本对比学习**: 将 CLIP 的图文对比扩展到视频-文本领域
2. **时序对比**: 对比视频片段与对应文本片段
3. **零样本视频理解**: 无需微调即可进行视频分类和检索

## 核心原理

### 视频-文本对比框架

1. 视频编码器提取时序特征
2. 文本编码器提取文本特征
3. 对比损失对齐视频和文本的嵌入空间

### 时序对齐

VideoCLIP 不仅对齐全局视频和文本，还对视频片段和句子片段进行细粒度对齐。

## 预训练方法

### 核心思想：CLIP 证明了图文对比可以学到通用的视觉表征——VideoCLIP 将同样的逻辑扩展到视频领域：视频和它的描述文本互为正例，让模型理解"发生了什么"以及"这段视频在讲什么"

VideoCLIP 是 CLIP 范式在视频-文本领域的最早尝试之一。核心思路不变：**视频编码器 + 文本编码器 + 对比损失，将视频和对应文本的嵌入对齐到同一个空间**。额外的挑战是时序——视频有"过程"属性，需要对动作序列进行时序建模。

> VideoCLIP = 视频 Transformer 编码器 + BERT 文本编码器 + InfoNCE 对比损失 + 时序对齐技巧。预训练后，视频可以与任意文本进行零样本匹配。

### 训练流水线（Step by Step）

#### Step 1 — 视频-文本对数据集

VideoCLIP 使用 HowTo100M——一个包含 1 亿+ 教学视频及其字幕的大规模数据集：

| 数据源 | 规模 | 类型 |
|--------|------|------|
| HowTo100M | 1.36 亿视频-文本对 | 教学/教程视频 + 旁白 |
| YouTube 教学视频 | ~1M 小时 | 涵盖各种主题 |

与 CLIP 的图文对类似——但视频文本对的对齐更松散（一句话可能跨越几秒甚至几十秒的视频）。

#### Step 2 — 视频编码器

VideoCLIP 的视频编码器处理视频片段：

```text
视频片段 (T 帧 × H × W × 3):
  → 片时空切分（Tubelet）
  → Video Transformer / TimeSformer
  → 时序池化（可选）
  → 视频嵌入向量 (d 维)
```

**时序编码方式**：

| 方式 | 描述 |
|------|------|
| 平均池化 | 所有帧 token 的平均 → 全局视频特征 |
| CLS Token | 学习一个 [CLS] token 聚合全局信息 |
| 时序 Transformer | 额外 Transformer 对帧序列建模 |

#### Step 3 — 文本编码器

文本编码器处理视频的相关文本（标题、描述、旁白）：

```text
文本: "How to bake a chocolate cake"
  → Tokenization
  → BERT / Transformer 文本编码器
  → CLS Token 或 平均池化
  → 文本嵌入向量 (d 维)
```

#### Step 4 — 视频-文本对比损失

$$\mathcal{L} = -\frac{1}{2N} \sum_{i=1}^N \left[ \log \frac{\exp(v_i^T t_i / \tau)}{\sum_j \exp(v_i^T t_j / \tau)} + \log \frac{\exp(t_i^T v_i / \tau)}{\sum_j \exp(t_i^T v_j / \tau)} \right]$$

| 项 | 含义 |
|----|------|
| 第一项（视频 → 文本） | 给定视频找到正确的文本 |
| 第二项（文本 → 视频） | 给定文本找到正确的视频 |

与 CLIP 完全相同——对称的 InfoNCE 对比损失。

#### Step 5 — 时序细粒度对齐

VideoCLIP 不仅对齐全局视频和文本，还在片段级别进行对齐：

```text
视频:  [片1:准备材料] [片2:搅拌] [片3:烘焙] [片4:装盘]
文本:  "准备材料, 然后搅拌, 然后烘焙, 最后装盘"

全局对齐: 整个视频 ↔ 整段文本
时序对齐: 片1 ↔ "准备材料", 片2 ↔ "搅拌", ...
```

**时序对齐的方式**：
- 将文本按句子拆分
- 将视频按时间段切分
- 使用软对齐（如 Transformer 的 Cross-Attention 或动态时间规整）

> 时序对齐是 VideoCLIP 相比简单"视频-文本 CLIP"的核心增强——让模型学习到指令序列和视频动作序列之间的对应关系。

### VideoCLIP vs CLIP

| 维度 | CLIP | VideoCLIP |
|------|------|----------|
| 数据类型 | 图像-文本对 | 视频-文本对 |
| 数据规模 | 400M 对 | 136M 对 |
| 视觉编码器 | ViT / ResNet | Video Transformer / TimeSformer |
| 时序建模 | 无 | **Tubelet + 时序 Attention** |
| 对齐粒度 | 全局 | **全局 + 时序片段** |
| 零样本能力 | 图像分类 | 视频分类 + 视频检索 |

### 详细训练配置

| 参数 | VideoCLIP | 说明 |
|------|----------|------|
| 预训练数据集 | HowTo100M (136M 对) | 教学视频 + 旁白 |
| 视频帧数 | 32-64 帧 | 几秒钟的视频片段 |
| 视频编码器 | TimeSformer 或 VideoViT | — |
| 文本编码器 | BERT-base | — |
| 嵌入维度 | 512 | 视频和文本共享 |
| 温度 $\tau$ | 0.07 | — |
| 优化器 | AdamW | — |
| 学习率 | 1e-4 | — |
| Batch Size | 4096 | 大 batch |

### 预训练的实用价值

1. **零样本视频理解**：任意文本描述 → 找到对应视频 / 分类视频
2. **视频检索的变革**：自然语言搜索视频内容，无需预定义标签
3. **视频-文本对比范式的确立**：后续 VideoCoCa、InternVideo 沿袭此路线
4. **时序细粒度对齐**：片段级对齐是 VideoCLIP 对 CLIP 范式的独特增强
5. **教学视频的理解**：HowTo100M 上的预训练使模型特别适合理解"如何做某事"类视频
