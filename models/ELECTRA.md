# ELECTRA (Efficiently Learning an Encoder that Classifies Token Replacements Accurately)

## 基本信息

- **论文**: [ELECTRA: Pre-training Text Encoders as Discriminators Rather Than Generators](https://arxiv.org/abs/2003.10555)
- **作者**: Kevin Clark, Minh-Thang Luong, Quoc V. Le, Christopher D. Manning
- **发表**: ICLR 2020

## 创新点

1. **判别式预训练 (RTD)**: 替代 MLM 的生成式预训练——不是预测被掩码的词，而是判断每个 token 是否被替换
2. **全 token 训练**: 所有 token 都参与损失计算，比 MLM 仅 15% token 参与更高效
3. **生成器-判别器架构**: 小生成器 + 大判别器的对抗式训练

## 核心原理

### Replaced Token Detection (RTD)

1. **生成器 (Generator)**: 类似 BERT 的 MLM 模型，预测被掩码位置的 token
2. **判别器 (Discriminator)**: 对每个位置判断该 token 是原始的还是生成的
3. 训练完成后只保留判别器，丢弃生成器

### 效率优势

| 方法 | 参与训练的 token 比例 | 每个 token 的信息量 |
|------|---------------------|-------------------|
| MLM (BERT) | 15% | 高（需知道具体词） |
| RTD (ELECTRA) | 100% | 低（仅需二分类） |

实验证明，RTD 的训练效率远高于 MLM。

## 预训练方法

### 核心思想：与其"猜词"，不如"打假"

BERT 的 MLM 有一个隐含浪费：只有 15% 的被掩码 token 参与损失计算，剩余的 85% 被白白"看过"了。ELECTRA 的核心洞察：**将所有 token 都变成训练信号**——让模型判断每一个 token 是原始的还是被生成器"伪造"的。

> 类比：BERT 是"完形填空"（遮住一些词让你填），ELECTRA 是"打假"（有人偷偷把某些词换了，让你找出所有被换掉的词）。后者让每一个词都参与判断，学习效率自然更高。

### 训练流水线（Step by Step）

#### Step 1 — 生成器（Generator）预测被掩码的词

生成器本质上是一个**小型的 BERT**，负责 MLM 任务：

1. 随机掩码输入序列中 15% 的 token
2. 生成器预测被掩码位置的原词
3. 对每个被掩码位置，从 softmax 分布中**采样**一个词作为"伪造"结果
4. 用这些采样词替换被掩码位置的 $[MASK]$ token

```
原始文本:   the [MASK] chef cooked the meal
生成器预测:   the [great] chef cooked the meal  (正确!)
           the [chef] chef cooked the [dinner] (错误!)
```

> 关键细节：ELECTRA 使用的是**采样**而不是 argmax（取概率最高的词）——因为如果生成器总是选"正确"的词，判别器就没有假货可打了。适度的"假词"是判别器的训练盐。

#### Step 2 — 判别器（Discriminator）判断每个词的真假

生成器的输出文本被送入判别器：

1. 对序列中的**每个 token**（包括未被掩码的），判别器输出一个二分类判断
2. Token 被判断为 "original"（原始）或 "replaced"（被替换）
3. 所有 512 个 token 都参与损失计算

$$\mathcal{L}_{\text{RTD}} = -\sum_{i=1}^{N} \underbrace{[\mathbb{1}(\text{original}) \log D(x_i) + \mathbb{1}(\text{replaced}) \log (1-D(x_i))]}_{\text{每个位置的二分类交叉熵}}$$

其中 $N$ 是序列长度（如 512）。对比 BERT 的 MLM 只有 $0.15N \approx 77$ 个位置参与计算，ELECTRA 的**有效训练信号是 BERT 的约 6.7 倍**。

#### Step 3 — 联合训练生成器与判别器

两个模型**同时训练**，共享嵌入层（token embedding）：

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{MLM}} + \lambda \mathcal{L}_{\text{RTD}}$$

- $\mathcal{L}_{\text{MLM}}$ 用于训练生成器（仅作用于被掩码位置）
- $\mathcal{L}_{\text{RTD}}$ 用于训练判别器（作用于所有位置）
- $\lambda$ 是平衡系数（通常设为 50 左右，因为 RTD 损失值较小）

> 关键细节：判别器的梯度**不传回生成器**——如果生成器为了"欺骗"判别器而故意产生易识别的假词，会破坏生成器的 MLM 学习。通过 stop-gradient 断开两者，生成器专注于 MLM，判别器专注于 RTD。

#### Step 4 — 训练完成后只保留判别器

生成器在训练完成后被丢弃（和 GAN 相反！GAN 中判别器被丢弃，这里恰好反过来）。判别器的编码器就是下游任务要使用的预训练模型。

### 为什么这样设计——深入理解

#### 1. 为什么 RTD 比 MLM 更高效？

核心原因是**训练信号密度**：

| 预训练目标 | 每次迭代的监督信号 | 信号类型 |
|-----------|------------------|---------|
| MLM (BERT) | 15% × 512 = 77 个位置 | 30K 类多分类（繁重） |
| RTD (ELECTRA) | 100% × 512 = 512 个位置 | 2 类二分类（简单） |

虽然每个位置的任务更简单（二分类 vs 多分类），但 **6.7 倍的信号量 + 更简单的任务**使得整体学习效率显著提升。

ELECTRA 论文的实验表明：
- 在相同的计算量下，ELECTRA 的 GLUE 得分远高于 BERT
- BERT 需要 4× 的训练步数才能达到 ELECTRA 相同步数的性能

#### 2. 为什么生成器要比判别器小？

如果生成器和判别器一样大（都是 12 层），生成的"假词"太难以被判别——判别器也需要很大的模型才能有效判断。但如果生成器小（4 层），它产生的替代词的分布会更"集中"，判别器更容易从小型生成器的"可疑改写"中学习。

| 生成器大小 | 判别器大小 | 效果 |
|-----------|-----------|------|
| 12 层（同大小） | 12 层 | 效率降低，生成器的假词太难以辨别 |
| 4 层（1/3 大小） | 12 层 | **最优**——生成器的假词有明显的"可辨别"特征 |
| 2 层（1/6 大小） | 12 层 | 生成器太弱，假词太明显，缺乏难度 |

最终选择：生成器隐藏维度 = 判别器的 1/4，层数等比例缩小。

#### 3. 为什么使用采样而不是 argmax？

如果生成器总是取概率最高的词（argmax），大部分替换会与原文相同。以一个典型 BERT 生成器（MLM 准确率约 65%）来说：
- argmax 策略：65% 的掩码位置被正确还原 → 判别器实际需要判断的"假词"只有 35%
- 采样策略：生成器有意"犯错"，约 15% 的整个序列位置都被替换 → 判别器有更多的正/负平衡样本

**没有足够假词 → 判别器学不到东西 → 效率优势消失。**

#### 4. 为什么不用 GAN 的对抗训练？

ELECTRA 的生成器-判别器看起来很像是 GAN，但实际训练不是对抗的：
- GAN：生成器试图**欺骗**判别器（对抗性）
- ELECTRA：生成器和判别器用**分开的损失**训练，没有对抗性

原因是文本领域对抗训练非常困难：
- 文本的离散性使得梯度无法从判别器传回生成器（softmax 采样的不可微分）
- 即使使用 Gumbel-Softmax 等技术，训练也极不稳定

**用最大似然（MLM）训练生成器，用 RTD 训练判别器**——简单但有效，避免了对抗训练的所有麻烦。

### 预训练性能

| 模型 | GLUE | 训练 FLOPs | 效率比 |
|------|------|-----------|--------|
| BERT-Small | 75.1 | 1.0× | 基准 |
| ELECTRA-Small | **79.9** | 1.0× | 更好 |
| BERT-Base | 79.6 | 1.0× | 基准 |
| ELECTRA-Base | **85.1** | **0.25×** | **4× 更快** |
| RoBERTa | 88.5 | 1.0× | 基准 |
| ELECTRA-Large | **89.3** | **0.25×** | **4× 更快** |

ELECTRA 的核心成就：**用 1/4 的计算量达到超越 BERT/RoBERTa 的性能**。这在预训练计算量指数增长的时代，是一次重要的"效率革命"。

### 预训练的迁移价值

1. **判别式预训练的范示**：证明了"判断真假"可以比"填空猜词"更高效
2. **全 token 训练**：将有效训练信号增加了 6.7 倍
3. **计算效率的极致优化**：对预算有限的预训练场景极具价值
4. **为生成-判别框架铺路**：启发了后续的 DeBERTa、COCO-LM 等判别-生成混合预训练方法
