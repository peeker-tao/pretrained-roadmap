# CogVLM (Cognitive Visual Language Model)

## 基本信息

- **论文**: [CogVLM: Visual Expert for Pretrained Language Models](https://arxiv.org/abs/2311.03079)
- **作者**: Weihan Wang et al. (智源研究院 / 清华大学)
- **发表**: arXiv, 2023

## 创新点

1. **视觉专家 (Visual Expert)**: 在 LLM 每层加入视觉深度模块
2. **深度融合**: 将视觉信息深度注入 LLM 而非仅通过投影层
3. **高性能**: 在多项 VQA 任务上达到 SOTA

## 核心原理

### 核心思想：LLaVA 只是"把视觉贴在表面"，CogVLM 要让视觉"渗透到骨髓"

LLaVA 和 MiniGPT-4 等模型的视觉注入只在**输入层**——视觉特征通过投影层变成 LLM 的"前缀 token"，然后 LLM 各层就按常规方式处理。CogVLM 则认为这种方式不够：**视觉信息应该参与 LLM 每一层的计算**。

> CogVLM 的核心：在冻结的 LLM 的**每一层**中，并行插入一个"视觉专家模块"（Visual Expert）。每个 Transformer 层都包含了文本路径（原 LLM 的 Self-Attn + FFN）和视觉路径（新增的 Visual Attn + Visual FFN），两者通过门控机制融合。

### 训练流水线

#### Step 1 — 架构：视觉专家模块（Visual Expert）

CogVLM 的每一层都从标准 Transformer：

```text
标准层: x → [Self-Attn] → [FFN] → 输出
```

扩展为带有视觉专家的版本：

```text
                    ┌──────────────────────┐
文本分支:  x_text → │ Self-Attn (共享QKV)  │ → FFN_text → ┐
                    │       ↓              │               │  门控融合
视觉分支:  x_vis  → │ Visual Attn (Q:vis,  │ → FFN_vis  → ┤ → 输出
                    │  K,V: 文本+视觉)     │               │
                    └──────────────────────┘               ┘
```

**关键设计**：
- **共享 QKV**：Self-Attention 的 Q、K、V 来自文本和视觉 token 的拼接
- **Visual FFN**：仅在视觉 token 上操作的独立 FFN 层（与文本 FFN 并行）
- **门控融合**：可学习的门控参数控制视觉路径贡献的权重

$$\text{Output} = \text{FFN}_{\text{text}}(h_{\text{text}}) + \alpha \cdot \text{FFN}_{\text{vis}}(h_{\text{vis}})$$

#### Step 2 — 预训练数据

CogVLM 使用了大规模的多模态预训练数据：

| 数据类型 | 规模 | 用途 |
|---------|------|------|
| 图文描述 | 1.5B 样本 | 基础视觉-语言对齐 |
| 图文交错 | 数亿样本 | 多图对话能力 |
| 视觉问答 | — | 理解与推理 |

#### Step 3 — 预训练目标

**下一 token 预测**（Next Token Prediction），以视觉为条件：

$$\mathcal{L} = -\sum_{t} \log P(y_t | y_{<t}, \text{Visual}(I))$$

其中 $\text{Visual}(I)$ 是所有层中视觉专家对文本 token 的贡献总和。

#### Step 4 — 两阶段训练策略

| 阶段 | 内容 | 可训练 | 目标 |
|------|------|--------|------|
| 阶段 1 | 图文描述预训练 | 视觉专家 + 投影层 | 基础视觉理解 |
| 阶段 2 | 指令微调 | 视觉专家 + 部分 LLM | 多模态对话/推理 |

### 为什么 CogVLM 的深度注入有效？

#### LLaVA 的问题

在 LLaVA 中，视觉 token 在输入层进入 LLM，然后：
- 浅层：视觉 token 与文本 token 通过 Self-Attention 交互
- 深层：视觉 token 的贡献逐渐被"稀释"——深层更关注高级语义

这导致 LLM **只在浅层真正"看到"图像**，深层几乎完全依赖文本路径。

#### CogVLM 的解决方案

每一层都有独立的视觉专家——视觉信息在每一层都被"刷新"进文本路径：

| 层 | LLaVA 的视觉信息 | CogVLM 的视觉信息 |
|----|-----------------|------------------|
| 浅层 | ✓ 有（输入层） | ✓ 有（视觉专家） |
| 中层 | 逐渐减弱 | ✓ 有（视觉专家持续注入） |
| 深层 | 几乎消失 | ✓ 有（视觉专家持续注入） |

> CogVLM 在深层仍然保留强烈的视觉信号——这让它在需要精细视觉理解的复杂任务（如 OCR、细粒度识别、空间推理）上表现更好。

### CogVLM vs 其他 VLM 架构

| 方法 | 视觉注入方式 | 每层注入 | 视觉-语言交互深度 | 视觉编码器微调 |
|------|------------|---------|-----------------|-------------|
| LLaVA | 输入层投影 | ✗ | 浅 | ✗ |
| Flamingo | 门控交叉注意力 | ✓ | 中 | ✗ |
| BLIP-2 | Q-Former | ✗ | 浅 | ✗ |
| **CogVLM** | **每层视觉专家** | **✓** | **深** | **✓** |

### 预训练性能

CogVLM-17B 在多个 VQA 基准上达到当时 SOTA，特别是在需要**细粒度视觉理解**的任务（如 TextVQA、OCR-VQA）上表现突出——这正是深层视觉注入的优势所在。

### 预训练的实用价值

1. **深度视觉注入范式**：证明了每层注入视觉信息比仅输入层注入更有效
2. **视觉专家的模块化设计**：可以轻松插入任何现有的 LLM 中
3. **在 OCR/细粒度任务上的优势**：深层视觉持续注入对精细视觉理解至关重要
4. **总参数量-效率权衡**：CogVLM-17B 在多项任务上可匹配更大的模型
