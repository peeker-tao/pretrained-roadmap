## 创新点

### GPT-1
1. **生成式预训练 + 判别式微调**: 首次证明大规模无监督预训练然后微调的有效性
2. **单向 Transformer 解码器**: 使用因果注意力掩码 (Causal Attention Mask)

### GPT-2
1. **零样本迁移能力**: 展示语言模型无需微调即可执行多种任务
2. **更大规模**: 15 亿参数，验证了规模扩展的有效性

### GPT-3
1. **In-Context Learning (上下文学习)**: 仅通过输入示例即可执行任务，无需梯度更新
2. **Scaling Law**: 系统性地展示了模型规模与性能的对数线性关系
3. **1750 亿参数**: 当时最大的稠密语言模型

### GPT-4
1. **多模态输入**: 支持图像和文本输入
2. **人类水平表现**: 在各种专业考试中达到人类水平
3. **强化的推理能力**: 思维链、多步推理能力显著增强

## 核心原理

### 自回归语言建模

GPT 系列使用单向自回归目标，根据前面的 token 预测下一个 token：

$$\mathcal{L} = -\sum_{t=1}^T \log P_\theta(x_t | x_{<t})$$

### Transformer 解码器架构

- 掩码多头自注意力 (Masked Multi-Head Self-Attention)
- 逐位置前馈网络 (Feed-Forward Network)
- 层归一化 (Layer Normalization)
- 残差连接

## 预训练方法

### 核心思想：预测下一个词

GPT 的预训练极其简单纯粹：**给定前面的文字，预测下一个词**。这看起来是一个简单的任务，但当你在整个互联网规模的文本上做这件事时，模型被迫学会了一切——语法、事实、推理、甚至一些常识。

打个比方：如果你读了足够多的"前文"，而且每次都能准确猜出"下一个词"，那你一定已经深刻理解了这段文字在说什么。GPT 正是通过这个机制学习语言的。

### 训练流水线（Step by Step）

#### Step 1 — 构建训练序列

从大规模文本语料中采样一段长度 $T$ 的序列（如 1024 或 2048 个 token）。不需要做任何标注或预处理。

例如一段 4 个 token 的序列：`["我", "爱", "吃", "苹果"]`

#### Step 2 — 自回归生成

模型依次处理每个位置：
- 位置 1：输入 `["我"]`，预测 `"爱"`
- 位置 2：输入 `["我", "爱"]`，预测 `"吃"`
- 位置 3：输入 `["我", "爱", "吃"]`，预测 `"苹果"`
- 位置 4：输入 `["我", "爱", "吃", "苹果"]`，预测 `"。"`

这就是**自回归（Autoregressive）**的含义：当前的输出成为下一步的输入。

#### Step 3 — 因果注意力掩码

GPT 使用**因果注意力掩码（Causal Attention Mask）**确保每个 token 只能看到它**之前**的 token，不能偷看后面的内容。具体实现是在注意力分数矩阵上施加一个上三角掩码：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} \odot M_{\text{causal}}\right)V$$

其中 $M_{\text{causal}}$ 是一个下三角矩阵（位置 $i$ 只能关注位置 $0$ 到 $i$）。

这个掩码保证了训练和推理时的一致性——因为推理时模型确实只能看到已生成的文本。

#### Step 4 — 计算损失

对于每个位置，模型输出一个概率分布（通过 Softmax 层，维度 = 词表大小），用交叉熵损失衡量预测词与真实词之间的差距：

$$\mathcal{L}_{\text{LM}} = -\sum_{t=1}^{T} \log P(x_t | x_{<t})$$

**总损失是所有位置交叉熵的平均**。这意味着每个词都被用来做监督信号——这被称为"密集监督"（dense supervision），是语言模型预训练高效的重要原因。

### 为什么自回归预训练如此强大？

#### 1. 无成本的标签

自回归预训练最大的优势是：**标签就在数据本身**。对于任何文本，每个 token 既是输入的一部分，也是前面 token 的标签。这意味着你可以用**互联网上所有的文本**来做预训练，无需任何人工标注。

#### 2. 隐式多任务学习

GPT-2 的论文提出了一个重要洞察：当你在足够多的文本上做下一个词预测时，很多 NLP 任务被隐式地作为"子问题"学习了：

| 任务类型 | 在文本中如何出现 |
|---------|----------------|
| 翻译 | `"The English phrase X means Y in French"` |
| 问答 | `"Q: What is the capital of France? A: Paris"` |
| 摘要 | 文章 + TL;DR + 摘要 |
| 情感分析 | `"This movie was amazing. Overall sentiment: positive"` |

因此，一个纯语言模型在足够大的语料上训练后，可以**零样本地**（不需要微调）完成多种 NLP 任务——这就是 GPT-2 和 GPT-3 展示的 In-Context Learning 能力的基础。

#### 3. 完美的可扩展性

自回归预训练可以无限扩展——更大的模型 + 更多的数据 = 更好的性能。这直接催生了后来的 Scaling Law 研究（如 Kaplan et al., Chinchilla），证明了模型规模和性能之间存在可预测的对数线性关系。

#### GPT-1 预训练

| 配置 | 值 | 说明 |
|------|-----|------|
| 数据集 | BookCorpus | ~7000 本未出版书籍，约 800M 词 |
| 架构 | 12 层 Transformer 解码器 | 隐藏维度 768，12 个注意力头 |
| FFN 维度 | 3072 | 4× 隐藏维度 |
| 参数量 | 117M | — |
| 序列长度 | 512 | 使用学习的位置编码 |
| 优化器 | Adam | β₁=0.9, β₂=0.999 |
| 学习率 | 2.5e-4 | 初始值 |
| 学习率调度 | 线性衰减 | 从初始值线性衰减到 0 |
| Batch Size | 64 | 相对较小的 batch |
| 训练 Epoch | 100 | 全量 BooksCorpus 过 100 遍 |
| 正则化 | Dropout 0.1, L2 0.01 | — |
| 激活函数 | GELU | — |
| 硬件 | 8 个 GPU | 训练约 1 个月 |

#### GPT-2 预训练

GPT-2 的核心升级是**数据质量**和**模型规模**：

| 配置 | 值 | 说明 |
|------|-----|------|
| 数据集 | WebText | 爬取 Reddit 上获得 ≥3 karma 的外部链接，约 8M 网页、40GB 纯文本 |
| 数据策略 | **质量优先** | 只保留高质量的人类撰写文本，去除低质量内容 |
| 架构 | 48 层，1600 维 | 16 个注意力头 |
| 参数量 | 1.5B | 约 GPT-1 的 13 倍 |
| 序列长度 | 1024 | 是 GPT-1 的 2 倍 |
| 词表大小 | 50,257 | BPE（Byte Pair Encoding） |
| 优化器 | Adam | 修改版（梯度裁剪 1.0） |
| LayerNorm | 移到注意力/FFN 之前 | Pre-LN 风格的改进 |

#### GPT-3 预训练

GPT-3 将"规模"推向极致：

| 配置 | 值 | 说明 |
|------|-----|------|
| 数据集 | Common Crawl + WebText2 + Books + Wikipedia | 总计约 300B tokens（经质量过滤后） |
| 数据过滤 | 训练一个分类器过滤低质量 Common Crawl | 用高质量数据（WebText）正例、低质量 Common Crawl 负例 |
| 架构变体 | 8 种尺寸：125M 到 175B | 最小 12 层 / 768 维，最大 96 层 / 12288 维 |
| 最大参数量 | 1750 亿（175B） | 96 层 Transformer，96 头，12288 维 |
| 序列长度 | 2048 | — |
| 优化器 | Adam | β₁=0.9, β₂=0.95, ε=1e-8 |
| 学习率 | 6e-5（175B） | 小模型用更大的学习率 |
| 学习率调度 | 余弦衰减 | 衰减到初始值的 10% |
| Warmup | 前 3.75 亿 tokens | 约 375M tokens 的线性 warmup |
| Batch Size | 3.2M tokens（175B） | 动态增加——训练前期小 batch，后期大 batch |
| 梯度裁剪 | 1.0 | — |
| 总训练量 | 300B tokens | 注意模型未过完整个数据（Chinchilla 法则后来指出应该训练更久） |
| 训练成本 | 约 460 万美元（估算） | 使用微软 Azure 的 V100 GPU 集群 |
| 权重初始化 | 正态分布 N(0, 0.02/√2N) | 按层数缩放 |

> **Chinchilla 法则**（2022 年发现）：对于 175B 参数的模型，最优训练 token 数应为 175B × 20 = **3.5 万亿 tokens**，远大于 GPT-3 实际训练的 300B。这说明 GPT-3 实际上是**欠训练（undertrained）**的——它的规模主要花在了模型容量上，而训练数据量没有匹配。

#### GPT-4 预训练

GPT-4 的具体训练细节大部分未公开，已知信息：

- **多模态训练**：同时处理图像和文本输入（但仅输出文本）
- **RLHF 对齐**：在预训练后使用**人类反馈强化学习（RLHF）**调整模型行为，使其更安全、更有帮助、更符合人类意图
- **规模**：参数量和训练数据量未公开，但据说总训练计算量远超 GPT-3
- **评估**：在各种专业考试（律师资格考试、GRE、SAT 等）上达到人类顶尖水平

### 预训练范式演进

GPT 系列的预训练范式经历了清晰的演进路径：

```
GPT-1:  预训练 → 微调（每任务）
GPT-2:  预训练 → 零样本（无需微调，直接使用）
GPT-3:  预训练 → 少样本 In-Context Learning（给几个示例）
GPT-4:  预训练 → RLHF 对齐 → 多模态应用
```

每一步演进的核心驱动力都是**更大的规模**——模型越大，涌现出的能力越多：

| 能力 | 何时出现 |
|------|---------|
| 基本语言理解 | GPT-1（100M 参数） |
| 零样本任务执行 | GPT-2（1.5B 参数） |
| 少样本上下文学习 | GPT-3（175B 参数） |
| 思维链推理 | GPT-3 规模+特殊提示 |
| 多模态理解 | GPT-4 |
| 复杂多步推理 | GPT-4 |

### 预训练的工程挑战

自回归预训练看似简单，但在百亿/千亿参数规模下面临严峻的工程挑战：

1. **内存**：175B 参数的 GPT-3 需要约 350GB 显存（FP16），需要**模型并行**（Tensor + Pipeline Parallelism）
2. **通信**：跨 GPU 的通信成为瓶颈，需要高效的 All-Reduce 实现
3. **稳定性**：大规模训练中 loss spike（损失突然飙升）是常见问题，需要梯度裁剪和混合精度训练
4. **数据**：收集和清洗五千亿 token 的高质量文本是极其庞大的工程任务
5. **成本**：训练一次 GPT-3 级别模型的电费可达数百万美元
