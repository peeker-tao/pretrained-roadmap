# RT-2 (Robotics Transformer 2)

## 基本信息

- **论文**: [RT-2: Vision-Language-Action Models Transfer Web Knowledge to Robotic Control](https://arxiv.org/abs/2307.15818)
- **作者**: Anthony Brohan et al. (Google DeepMind)
- **发表**: arXiv, 2023

## 创新点

1. **视觉-语言-动作 (VLA) 模型**: 将互联网知识迁移到机器人控制
2. **动作 Tokenization**: 将机器人动作离散化为 token，与文本 token 统一表示
3. **涌现泛化能力**: 对未见过的物体和场景进行泛化操控

## 核心原理

### VLA 框架

1. **视觉**: 从机器人摄像头图像提取视觉特征
2. **语言**: 自然语言指令
3. **动作**: 将连续动作（末端执行器位置、旋转、夹持器）离散化为 token
4. **统一序列**: [图像, 文本指令, 动作 token]

### 训练流程

1. 在互联网图文数据上预训练（PaLI-X / PaLM-E）
2. 在机器人演示数据上微调（动作 token 预测）
3. 推理时：图像 + 指令 → 生成动作序列

### 泛化能力

| 能力 | RT-1 | RT-2 |
|------|------|------|
| 已知物体 | ✅ | ✅ |
| 未见物体 | ❌ | ✅ |
| 推理能力 | ❌ | ✅ |

## 预训练方法

### 核心思想：你在网上读过无数关于"如何拿杯子""门把手怎么用"的文字描述。如果你把这些知识注入到一个机器人里，它能不能在没有见过的杯子上、没有训练过的门把手上完成任务？RT-2 的答案是：把互联网知识预训练 + 机器人动作微调 = 涌现的泛化能力

RT-2（Robotics Transformer 2）是 Google DeepMind 的视觉-语言-动作（VLA）模型。它的核心洞察：将互联网上预训练的 VLM（视觉语言模型）直接微调为机器人策略——把"动作"当作一种新的"语言 token"，让模型从"看图说话"变成"看图行动"。

> RT-2 = 互联网预训练 VLM（PaLI-X / PaLM-E）+ 机器人演示微调（动作 token 预测）+ 链式思维推理。未见过的物体、场景、指令 → 涌现泛化。

### 训练流水线（Step by Step）

#### Step 1 — 动作 Tokenization：把物理动作变成"语言"

机器人动作通常是连续值——末端执行器的 6-DoF 位姿变化 + 夹持器开合。RT-2 的巧妙之处在于将连续动作**离散化为 token**：

```text
机器人动作:
  原始连续: (Δx, Δy, Δz, Δroll, Δpitch, Δyaw, gripper)
  
离散化:
  每个维度分为 256 个 bin
  (Δx, Δy, Δz): 位置变化 [-1cm, 1cm] → 256 个离散值
  (Δroll, Δpitch, Δyaw): 旋转变化 [-5°, 5°] → 256 个离散值
  gripper: 0 或 1 → 2 个值

结果: 7 个 token = 1 个完整动作
```

| 属性 | 值 | 说明 |
|------|-----|------|
| 动作维度 | 7 | 位置(3) + 旋转(3) + 夹持器(1) |
| 每维 bin 数 | 256 | 8-bit 量化 |
| Token 序列 | 7 个 token/动作 | 类似"一个动作 = 一句话" |
| 词汇量 | 256 + 文本词汇 | 动作 token 混入文本词汇表 |

#### Step 2 — 互联网预训练（视觉-语言阶段）

RT-2 基于预训练的大规模 VLM——标准的视觉-语言预训练：

```text
阶段 1: 互联网图文预训练

基础模型: PaLI-X (55B) 或 PaLM-E (562B)
  视觉: ViT-22B
  语言: UL2 / T5 Encoder-Decoder

预训练数据: 
  图文对: 数十亿（类似 CLIP/ALIGN）
  纯文本: 网络文本、书籍、代码
  多模态: VQA、Captioning、OCR

任务:
  图文匹配 ✓
  图像描述 ✓  
  视觉问答 ✓
  纯语言 ✓
  代码 ✓

训练: 标准 VLM 预训练 (交叉熵)
```

**关键**：这个阶段模型学会了大量的常识——"杯子是用来喝水的""拉门把手可以开门""物体放在盘子里"等等。这些知识来自互联网文本和图片。

#### Step 3 — 机器人微调（视觉-语言-动作阶段）

```text
阶段 2: 机器人微调

数据: 机器人演示数据
  场景: 桌面操作（抓取、放置、推动）
  数量: ~130K 条真实机器人演示
  格式: (图片序列, 文本指令, 动作序列)

微调:
  输入: [img_1, img_2, ..., img_6] + "pick up the apple"
  → 模型生成: [动作 token 序列]
  
  损失: 仅在动作 token 上的交叉熵
  L = -log P(action_tokens | images, text_instruction)

特殊技巧: 链式思维 (Chain-of-Thought)
  让模型先输出"推理"—再输出动作:
  "The apple is on the left side of the table. 
   I need to move the gripper to (0.3, -0.2, 0.1)...
   [动作 token 序列]"
```

**CoT 推理示例**（来自论文）：

```text
任务: "把可乐罐移到 Taylor Swift 的照片旁边"

模型输出 (CoT):
  "我看到一个红色可乐罐在桌子左侧。
   Taylor Swift 的照片在右侧。
   我需要:
   1. 抓住可乐罐
   2. 移到右侧
   3. 放在照片旁边"
  [动作序列]
```

#### Step 4 — 涌现泛化能力

RT-2 最令人兴奋的特性：对**未见过的场景**的泛化：

| 泛化类型 | 示例 | RT-1（无互联网预训练） | RT-2 |
|---------|------|---------------------|------|
| 未见物体 | 没见过的杯子/水果 | 0-20% | **60-80%** |
| 未见背景 | 新的桌面/环境 | 10-30% | **50-70%** |
| 语义理解 | "把能吃的放在左边" | 0% | **~50%** |
| 推理 | "推到和红色物体一组" | 0% | **~40%** |

**为什么能泛化？**

| 知识来源 | 互联网预训练 | 机器人微调 |
|---------|----------|---------|
| "什么是可乐罐" | ✓ (见过图片+描述) | 只有训练中的那些 |
| "Taylor Swift 是谁" | ✓ (见过图片+文本) | 完全没有 |
| 精确抓取动作 | ✗ | ✓ |

> 互联网预训练提供了"世界知识"，机器人微调提供了"行动能力"。两者结合 → 从未见过的任务也能完成。

#### Step 5 — 完整训练配置

| 参数 | RT-2 (PaLI-X) | RT-2 (PaLM-E) |
|------|-------------|------------|
| 基础模型 | PaLI-X 55B | PaLM-E 562B |
| 视觉编码器 | ViT-22B | ViT-22B |
| 语言模型 | T5 Encoder-Decoder | PaLM Decoder |
| 预训练数据 | 图文对 + 纯文本 | 图文对 + 纯文本 |
| 机器人数据 | ~130K 演示 | ~130K 演示 |
| 动作 token 词汇 | 256 × 7 = 1792 | 256 × 7 = 1792 |
| 上下文长度 | 512 tokens | 512 tokens |
| 图像帧数 | 6 | 6 |
| 推理 | CoT 可选 | CoT 可选 |

### RT-2 的预训练意义

1. **VLA (Vision-Language-Action) 范式**：动作 = 语言 → 统一 token 空间
2. **互联网知识 → 物理世界**：证明了互联网预训练可以迁移到机器人控制
3. **CoT 在具身智能**：让机器人像人类一样"先想再做"
4. **数据效率**：仅 130K 机器人演示就能驱动 55B 模型
5. **泛化的涌现**：模型越大，泛化越强 → 规模律在具身智能也成立
