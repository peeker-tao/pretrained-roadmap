# BERT (Bidirectional Encoder Representations from Transformers)

## 基本信息

- **论文**: [BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding](https://arxiv.org/abs/1810.04805)
- **作者**: Jacob Devlin, Ming-Wei Chang, Kenton Lee, Kristina Toutanova (Google)
- **发表**: NAACL 2019

## 创新点

1. **双向上下文建模**: 使用掩码语言建模 (MLM) 实现真正的双向上下文理解，突破传统单向语言模型的限制
2. **掩码语言建模 (MLM)**: 随机掩码 15% 的 token，利用双向上下文预测被掩码的 token
3. **下一句预测 (NSP)**: 额外训练句对关系理解，提升问答和推理任务表现
4. **在 11 项 NLP 任务上刷新 SOTA**: 包括 GLUE、SQuAD、SWAG 等

## 核心原理

### 双向 Transformer 编码器

BERT 使用 Transformer 编码器架构，关键特点是：
- **双向自注意力**: 每个 token 可关注所有其他 token（无单向掩码）
- **深层次堆叠**: BERT-Base 12 层，BERT-Large 24 层

### 输入表示

输入由三部分嵌入相加构成：
1. Token Embeddings: WordPiece 分词（30k 词汇量）
2. Segment Embeddings: 区分句子 A 和 B
3. Position Embeddings: 学习的位置编码

特殊 token：[CLS]（分类标记）和 [SEP]（句子分隔符）

## 预训练方法

### 核心思想：完形填空 + 句子配对

BERT 的预训练可以类比为两个语文练习题：

1. **完形填空（MLM）**：把文章中 15% 的词挖掉，让模型根据上下文猜测被挖掉的词
2. **句子配对（NSP）**：给两个句子，判断它们是否在原文中前后相邻

这两个任务让模型同时学会了**理解词义**和**理解句间关系**。

### 训练流水线（Step by Step）

#### 任务一：掩码语言建模（MLM）

**Step 1 — 选择掩码位置**：随机选取输入序列中 15% 的 token 作为"候选掩码位"。

**Step 2 — 掩码策略（关键技巧）**：对于这 15% 的候选 token，不是全部替换为 [MASK]，而是：

| 处理方式 | 比例 | 目的 |
|---------|------|------|
| 替换为 `[MASK]` | 80% | 告诉模型"这里需要猜" |
| 替换为**随机词** | 10% | 防止模型只学会"查字典"而忽略了上下文 |
| **保持原词不变** | 10% | 缓解微调时没有 [MASK] 导致的不一致 |

举个例子，原句为 `"我 喜欢 吃 苹果"`，假设"苹果"被选中：

- 80% 概率变成：`"我 喜欢 吃 [MASK]"` → 模型需要从上下文推理出是"苹果"
- 10% 概率变成：`"我 喜欢 吃 火车"` → 模型需要利用上下文纠正这个错误
- 10% 概率保持：`"我 喜欢 吃 苹果"` → 模型需要"确认"这个词是对的

**Step 3 — 预测**：BERT 的双向 Transformer 让被掩码位置的输出表征融合了左右两侧的完整上下文，然后用一个分类头预测该位置的原始词。

**Step 4 — 损失计算**：仅对被掩码的 15% 位置计算交叉熵损失，其余 85% 不参与损失计算（否则模型会学得"偷懒"）。

$$\mathcal{L}_{\text{MLM}} = -\frac{1}{|M|}\sum_{m \in M} \log P(x_m | x_{\backslash M})$$

其中 $M$ 是被掩码位置的集合，$x_{\backslash M}$ 表示去掉被掩码位置后的序列。

#### 任务二：下一句预测（NSP）

**Step 1 — 构建句子对**：对于每个训练样本，随机选取两个句子 A 和 B：

- **50% 正例**：B 确实是 A 在原文中的下一句（标签 = `IsNext`）
- **50% 负例**：B 是语料中随机抽取的另一句话（标签 = `NotNext`）

**Step 2 — 编码**：将句子对拼接为 `[CLS] A [SEP] B [SEP]`，其中 `[CLS]` 的输出用于做句子关系判断。

**Step 3 — 预测**：用 `[CLS]` 位置的输出表征过一个二分类头，判断 B 是否是 A 的下一句。

$$\mathcal{L}_{\text{NSP}} = -y\log P(\text{IsNext}|A,B) - (1-y)\log P(\text{NotNext}|A,B)$$

**总损失** = $\mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{NSP}}$，两个任务联合训练。

### 为什么这样设计——深入理解每个选择

#### 1. 为什么是 15% 的掩码率？

这来自 NLP 领域的重要经验：掩码率太低 → 学习信号不足，太高 → 上下文被破坏太严重，模型无法有效推理。

| 掩码率 | 效果 |
|-------|------|
| 5% | 太简单，模型几乎不需要推理 |
| 15% | **最优** — 足够挑战但上下文仍可恢复 |
| 30% | 损失太多信息，性能下降 |
| 50% | 上下文严重不足，模型崩溃 |

#### 2. 为什么需要 80-10-10 的掩码策略？

纯 `[MASK]` 的问题在于：微调阶段没有 `[MASK]` token，模型在预训练时见到的 `[MASK]` 和微调时的实际输入不匹配（distribution shift）。

80-10-10 策略的巧妙之处：
- 80% `[MASK]` 确保模型学会"填空"
- 10% 随机替换强迫模型**利用上下文纠正错误**（而不是盲目信任输入）
- 10% 保持原词让模型知道"有些位置不需要改"

#### 3. 为什么需要双向注意力？

传统语言模型（如 GPT）使用单向注意力——每个词只能看到它前面的词。BERT 使用双向注意力——每个词可以看到前后所有词。

这一差异至关重要：
- **单向**："我 喜欢 吃 ___" → 只能根据前面猜，可能是"苹果"、"香蕉"……
- **双向**："我 喜欢 吃 ___，因为 它 很 甜" → 同时利用前后文，可以更确定是某种甜的食物

双向注意力让 BERT 的表征包含了**完整的上下文信息**，这就是为什么 BERT 在理解类任务（分类、问答、推理）上远超 GPT。

#### 4. NSP 到底有没有用？

这是一个有争议的问题。后续研究发现，RoBERTa 去掉 NSP 后性能反而提升。原因在于：
- NSP 是一个**相对简单**的任务——判断"两句是否相邻"不需要深层的语义理解
- 去掉 NSP 后可以用更长的连续文本训练（更长的上下文）
- 但 NSP 在问答（QA）和推理（NLI）等需要句间关系的任务上有细微帮助

### 详细训练配置

#### 数据准备

| 数据来源 | 规模 | 内容特点 |
|---------|------|---------|
| BooksCorpus | 8 亿词 | 长篇小说，连续文本，语义连贯 |
| English Wikipedia | 25 亿词 | 百科文章，结构化信息 |

> 总计约 33 亿词（3.3B words），约 2.5GB 文本。对于 2018 年来说规模很大，但按今天的标准来看是一个"小模型+小数据"的组合。

#### 输入处理

- **分词器**：WordPiece（30,000 词汇量），子词级别的分词
- **序列长度**：最大 512 个 token
- **输入构建**：`Token Embedding + Segment Embedding + Position Embedding`
  - Token Embedding：每个子词的可学习向量
  - Segment Embedding：区分是句子 A（0）还是句子 B（1）
  - Position Embedding：可学习的位置编码（不是 Sinusoidal）

#### 超参数

| 配置 | BERT-Base | BERT-Large | 设计考虑 |
|------|----------|-----------|---------|
| Transformer 层数 | 12 | 24 | 深层 = 更强的表征能力 |
| 隐藏维度 | 768 | 1024 | 每层升维增强表达 |
| 注意力头数 | 12 | 16 | 多头从不同子空间建模 |
| 总参数量 | 110M | 340M | — |
| FFN 维度 | 3072 | 4096 | 4× 隐藏维度 |
| 优化器 | Adam | Adam | $\beta_1=0.9, \beta_2=0.999, \epsilon=1e-8$ |
| 学习率 | 1e-4 | 1e-4 | 经验最佳值 |
| Warmup | 10,000 步 | 10,000 步 | 避免初期训练不稳定 |
| 学习率衰减 | 线性衰减 | 线性衰减 | 从 1e-4 线性降到 0 |
| Batch Size | 256（序列） | 256（序列） | — |
| 训练步数 | 1,000,000 步 | 1,000,000 步 | 约 40 epoch |
| Dropout | 0.1（所有层） | 0.1（所有层） | 防止过拟合 |
| 激活函数 | GELU | GELU | 平滑的 ReLU 变体 |
| 梯度裁剪 | 1.0 | 1.0 | 防止梯度爆炸 |

#### 训练硬件

- **BERT-Base**：4 个 Cloud TPU（16 个 TPU 芯片），训练约 4 天
- **BERT-Large**：16 个 Cloud TPU（64 个 TPU 芯片），训练约 4 天

### 预训练的深远影响

BERT 的 MLM 预训练范式彻底改变了 NLP —— 以至于后续 3-4 年内几乎所有主流 NLP 模型都建立在类似的双向预训练框架上。它的核心贡献在于：

1. **证明了双向上下文预训练的巨大价值** — 理解一个词需要同时看它的前后
2. **确立了"预训练→微调"范式** — 先在大规模无标签数据上学通用知识，再针对具体任务微调
3. **催生了整个"BERT 家族"** — RoBERTa、ALBERT、ELECTRA、SpanBERT、DeBERTa……
