# UniPi (Universal Policy)

## 基本信息

- **论文**: [UniPi: Learning Universal Policies via Text-to-Video Generation](https://arxiv.org/abs/2310.03074)
- **作者**: Tim Brooks et al. (Google / MIT)
- **发表**: NeurIPS 2023

## 创新点

1. **文本指令 → 视频 → 动作**: 用视频生成作为规划和策略的中间表示
2. **视频作为世界模型**: 生成未来状态的视频，再从中提取动作
3. **零样本泛化**: 利用视频扩散模型的泛化能力迁移到新任务

## 核心原理

### 两阶段框架

**阶段 1: 视频规划**
- 给定初始图像 + 文本指令
- 使用视频扩散模型生成未来状态的视频

**阶段 2: 动作提取**
- 从生成的视频中提取动作（逆动力学模型）
- 使用轨迹优化将视频帧映射到机器人动作

### 为什么视频作为中间表示？

- 视频包含丰富的世界动态信息
- 预训练的视频模型具有强泛化能力
- 视觉监督比动作监督更丰富

## 预训练关系

### 核心思想：RT-2 是让大模型直接输出动作——"看图 → 行动"。UniPi 走了一条完全不同的路——让大模型先输出"未来会发生什么的视频"，再从视频中提取动作。这就像你下棋前先在脑中"播放"几步棋的画面，然后从画面中推导出落子位置

UniPi（Universal Policy）代表了具身智能的另一种范式：视频生成作为通用规划器。它的核心洞察是——视频包含的信息远多于动作序列，预训练的视频扩散模型天然具有强大的物理世界理解能力。

> UniPi = 文本指令 + 初始图像 → 视频扩散模型（生成未来状态视频）→ 逆动力学模型（从视频帧提取动作）。"世界模型 + 规划"而非"端到端策略"。

### 训练流水线（Step by Step）

#### Step 1 — 为什么用视频作为中间表示？

| 对比 | 端到端策略（RT-2） | 视频中间表示（UniPi） |
|------|---------------|------------------|
| 输入→输出 | 图像 → 动作 | 图像 → 视频 → 动作 |
| 监督信号 | 动作标签（稀疏） | **视频帧（密集）** |
| 预训练数据 | 互联网图文+动作 | **互联网视频（海量）** |
| 可解释性 | 低（黑箱） | **高（可以看到"计划"）** |
| 泛化来源 | VLM 的知识 | **视频扩散模型的物理理解** |

> 视频包含了"物体如何运动""碰撞后会发生什么""重力如何作用"等物理直觉——这些都是动作序列无法直接编码的。

#### Step 2 — 阶段 1：利用预训练的视频扩散模型

```text
UniPi 依赖的核心预训练模型:
  视频扩散模型（类似 Sora / VideoLDM）

预训练方式（标准扩散模型）:
  数据: 大规模视频数据集
    → 互联网视频（人类动作、物体交互、自然现象）
    → 模拟器数据（机器人仿真）
  
  训练:
    前向过程: 逐步加噪 → 破坏视频
    反向过程: 学习从噪声中恢复视频帧
    条件: 文本描述 / 初始帧
  
  预训练目标: 从初始帧 + 文本生成合理的未来视频
```

| 参数 | 预训练视频扩散模型 | 说明 |
|------|--------------|------|
| 数据 | 互联网视频 + 仿真 | 通用视频理解 |
| 帧数 | 16-32 帧 | 预测 1-3 秒 |
| 分辨率 | 256×256 | — |
| 条件 | 文本 + 初始帧 | 可控生成 |
| 训练 | 标准扩散损失 | 去噪得分匹配 |

#### Step 3 — 阶段 2：从生成的视频中提取动作

```text
给定:
  初始图像 I_0
  文本指令: "把杯子推到桌子中间"

步骤 1: 视频生成（规划）
  输入: [I_0] + "push the cup to the center of the table"
  → 视频扩散模型
  输出: 未来 16 帧视频 [Î_1, Î_2, ..., Î_16]

步骤 2: 动作提取（执行）
  对于每一对连续帧 (Î_t, Î_{t+1}):
    逆动力学模型: a_t = f(Î_t, Î_{t+1})
    → 输出动作（抓取/移动/放置）

步骤 3: 执行
  机器人执行动作序列 [a_0, a_1, ..., a_15]
```

**逆动力学模型（Inverse Dynamics Model）**：

$$\hat{a}_t = g(\hat{I}_t, \hat{I}_{t+1})$$

给定当前帧和下一帧，逆动力学模型预测"从当前帧到下一帧需要什么动作"。

| 组件 | 说明 |
|------|------|
| 训练数据 | 机器人演示（帧+动作） |
| 输入 | 连续两帧 |
| 输出 | 动作（6-DoF + gripper） |
| 规模 | 轻量（几百万参数） |

#### Step 4 — UniPi vs RT-2 的范式对比

```text
RT-2 (端到端):
  图像 + 文本 → [LLM] → 动作 token
  优点: 简洁、快速
  缺点: 黑箱、泛化依赖 VLM 知识

UniPi (视频规划):
  图像 + 文本 → [视频扩散模型] → 视频 → 动作
  优点: 可解释、泛化依赖视频预训练、更自然的物理理解
  缺点: 慢（需要生成视频）、两阶段
```

| 维度 | RT-2 | UniPi |
|------|------|-------|
| 核心预训练 | VLM (图文) | **视频扩散模型** |
| 推理方式 | 端到端 | **先规划（视频）→ 再执行** |
| 可解释性 | 低 | **高** |
| 速度 | 快 | 慢（视频生成） |
| 物理理解 | 隐式 | **显式（视频中的物理）** |
| 代表方向 | "大模型 = 策略" | **"大模型 = 世界模拟器"** |

#### Step 5 — UniPi 的预训练启示

| 启示 | 说明 |
|------|------|
| **世界模型的重要性** | 好的视频生成模型 = 好的物理世界理解 |
| **中间表示的价值** | 视频作为"通用语言"连接感知和行动 |
| **预训练的角色** | 视频扩散模型预训练 → 泛化到新任务 |
| **分离规划与执行** | 规划（视频）和执行（动作为独立的模块） |

### UniPi 的预训练配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 视频扩散模型 | 预训练（互联网视频） | 核心依赖 |
| 规划帧数 | 16-32 | 1-3 秒 |
| 分辨率 | 256×256 | — |
| 逆动力学模型 | 轻量 MLP/CNN | 动作提取 |
| 任务 | 桌面操作 | Push/Pick/Place |

### 预训练的实用价值

1. **视频作为通用规划器**：大模型先"想象"未来 → 再执行
2. **预训练视频模型的新用途**：不只是生成视频 → 更是物理世界模拟器
3. **具身智能的两种路径**：VLA（RT-2）vs 视频规划（UniPi）→ 互补而非对立
4. **可解释性突破**：用户可以"看到"机器人的计划
5. **对 Sora/世界模型的影响**：视频生成 → 世界模型 → 通用规划的进化路线
