# BYOL (Bootstrap Your Own Latent)

## 基本信息

- **论文**: [Bootstrap Your Own Latent: A New Approach to Self-Supervised Learning](https://arxiv.org/abs/2006.07733)
- **作者**: Jean-Bastien Grill et al. (DeepMind)
- **发表**: NeurIPS 2020

## 创新点

1. **无需负例**: 仅使用正例对训练即可学习有效表征
2. **不对称架构**: 在线网络 (Online) + 动量目标网络 (Target) + 预测头 (Predictor)
3. **隐式正则化**: 预测头 + 动量更新防止表征坍塌

## 核心原理

### BYOL 架构

1. **在线网络**: 对视图 1 编码 → 投影 → 预测
2. **目标网络**: 对视图 2 编码 → 投影（动量更新）
3. **预测头**: 附加在在线网络上，预测目标网络的投影输出
4. **损失**: 预测与目标之间的均方误差

### 动量更新

$$\theta_{\text{target}} \leftarrow \tau\theta_{\text{target}} + (1-\tau)\theta_{\text{online}}$$

其中 $\tau$ 通常为 0.996。

### 防止坍塌的解释

BYOL 为什么不需要负例也不坍塌？后续理论分析提出：
- 预测头起到了**隐式正则化**的作用
- 动量目标网络引入了一种**慢变**的学习信号
- 目标网络与在线网络之间的交互构成了一种**自蒸馏**

## 预训练方法

### 核心思想：自己教自己

BYOL 的精髓可以用一句话概括：**让在线网络去"猜测"目标网络会怎么看待同一张图片的另一面**。这个"猜测-对答案"的过程，就是"自蒸馏"（self-distillation）——在线网络是学生，目标网络是老师，但老师其实是从学生"慢慢演变"来的。

### 训练流水线（Step by Step）

假设我们有一张猫的图片，BYOL 的训练流程如下：

**第一步：创建两个"视角"**

对同一张猫的图片做两次**不同的**数据增强（随机裁剪、颜色抖动、高斯模糊等），得到视图 $x_1$ 和视图 $x_2$。$x_1$ 可能只拍到猫的脸，$x_2$ 可能只拍到猫的身体——两者看起来完全不同，但本质上是同一只猫。

**第二步：在线网络处理视图1**

视图 $x_1$ 进入 **在线网络（Online Network）**：
$$x_1 \xrightarrow{\text{编码器 } f_\theta} \text{特征} \xrightarrow{\text{投影头 } g_\theta} z_1 \xrightarrow{\text{预测头 } q_\theta} q_\theta(z_1)$$

编码器（如 ResNet-50）提取图像特征，投影头将特征映射到低维空间，预测头再进一步变换得到一个**预测向量**。

**第三步：目标网络处理视图2**

视图 $x_2$ 进入 **目标网络（Target Network）**：
$$x_2 \xrightarrow{\text{编码器 } f_\xi} \text{特征} \xrightarrow{\text{投影头 } g_\xi} z'_2$$

目标网络的结构和在线网络完全相同，但参数 $\xi$ 不与在线网络 $\theta$ 共享。而且目标网络**不接收梯度**——它是"固定的参考标准"。

**第四步：计算损失并反向传播**

计算在线网络的预测 $q_\theta(z_1)$ 和目标网络的输出 $z'_2$ 之间的余弦相似度（取负号变成损失）：

$$\mathcal{L}_{\text{BYOL}} = -\frac{q_\theta(z_1)}{\|q_\theta(z_1)\|_2} \cdot \frac{z'_2}{\|z'_2\|_2}$$

然后对在线网络 $\theta$ 做梯度下降，**目标网络 $\xi$ 不更新**。

**第五步：交换角色，对称训练**

把 $x_1$ 和 $x_2$ 的角色互换——$x_1$ 送目标网络，$x_2$ 送在线网络，再算一次损失。总损失是两者的平均：

$$\mathcal{L} = \mathcal{L}_{\text{BYOL}}(x_1, x_2) + \mathcal{L}_{\text{BYOL}}(x_2, x_1)$$

**第六步：动量更新目标网络**

每次（或每隔几步）用**指数移动平均（EMA）**更新目标网络：

$$\xi \leftarrow \tau \cdot \xi + (1 - \tau) \cdot \theta$$

其中 $\tau = 0.996$（论文默认值），意味着目标网络每次只"向在线网络靠近一点点"（$0.4\%$）。

### 为什么这样设计——深入理解每个组件

#### 1. 预测头（Predictor）的作用

预测头 $q_\theta$ 是 BYOL 不对称架构的核心。它的存在让在线网络不能简单地"复制"输入——它必须学会**预测**不同视角下的表征。想象一下：

- 没有预测头：在线网络可以直接输出和目标网络一样的向量，网络瞬间坍塌（输出常数）
- 有预测头：在线网络必须"加工"一下自己的表征去匹配目标网络的表征，这个加工过程强制网络学习有意义的模式

#### 2. 动量更新的作用

目标网络的**缓慢更新**是防止坍塌的另一个关键。如果目标网络也做梯度更新，那么两个网络会"互相追逐"，最终一起收敛到平凡解。动量更新让目标网络像一个"缓慢漂移的锚点"——它变化很慢，给在线网络提供了稳定的学习目标。

直观类比：目标网络就像"昨天的在线网络"，在线的任务是预测"昨天的自己"会怎么看待这张图。因为目标网络更新极慢（$\tau=0.996$），它的输出在一段时间内基本恒定，这就形成了稳定的监督信号。

#### 3. 为什么不需要负例？

对比学习（SimCLR、MoCo）需要大量负例来"推开"不同样本的表征，防止所有表征坍缩到同一个点。BYOL 通过**预测头 + 动量更新**的隐式机制实现了同样的效果：目标网络的缓慢更新避免了"师生同谋"，而预测头的不对称性让在线网络不能简单复制输出。

后续理论分析（如 DirectPred 等）进一步揭示：BYOL 的动量更新实际上在隐式地对特征协方差矩阵做某种正则化，迫使表征保持信息丰富性。

### 详细的训练配置

#### 网络结构

| 组件 | 架构 | 输出维度 |
|------|------|---------|
| 编码器 $f$ | ResNet-50（去除分类头） | 2048 |
| 投影头 $g$ | MLP: 2048→4096→256（每层后跟 BN + ReLU） | 256 |
| 预测头 $q$ | MLP: 256→4096→256（仅首层有 BN + ReLU） | 256 |

注意：投影头和预测头在训练后被**丢弃**，只保留编码器 $f_\theta$ 用于下游任务。

#### 数据增强

BYOL 沿用 SimCLR 的增强组合，每个样本经过随机的：

1. **随机裁剪 + 缩放（RandomResizedCrop）**：将图像裁成 224×224，随机区域占比 0.08~1.0
2. **随机水平翻转**：50% 概率
3. **颜色抖动（ColorJitter）**：亮度/对比度/饱和度/色调，强度 0.8/0.8/0.8/0.2
4. **灰度化**：20% 概率
5. **高斯模糊**：$\sigma \in [0.1, 2.0]$
6. **可选的 Solarization**：BYOL v2 中引入

#### 超参数调度

| 参数 | 值 | 说明 |
|------|-----|------|
| 优化器 | LARS | 专为大 batch 设计的自适应学习率优化器 |
| 基础学习率 | 3.2（Linear Scaling: $lr = 0.2 \times \text{BatchSize}/256$） | 学习率与 batch size 线性缩放 |
| 权重衰减 | 1.5e-6 | 极小的权重衰减，避免过度正则化 |
| Batch Size | 4096 | 分布在 512 个 Cloud TPU v3 核心上 |
| 训练 Epoch | 300（v1）/ 1000（v2） | 更长训练持续提升性能 |
| Warmup | 前 10 个 epoch | 学习率从 0 线性增长到基础值 |
| 学习率衰减 | 余弦退火（Cosine Decay） | 平滑衰减到 0 |
| 动量 $\tau$ | 从 0.996 余弦调度到 1.0 | 训练后期让目标网络几乎冻结 |

#### 训练中的关键现象

- **初期训练不稳定**：前几个 epoch 损失可能波动较大，这是正常的——预测头正在学习如何"预测"目标网络
- **动量系数敏感**：$\tau$ 太大会导致目标网络更新太慢（学习信号弱），太小会导致坍塌（锚点漂移太快）。0.99~0.999 是经验最优区间
- **对称损失不可或缺**：如果只算一个方向（$x_1$→$x_2$），性能下降约 1-2 个百分点

### 预训练性能

| 评估方式 | BYOL (ResNet-50, 300ep) | 对比基准 |
|---------|------------------------|---------|
| ImageNet 线性探测 | **74.3%** | SimCLR 69.3%, MoCo v2 71.1% |
| ImageNet 半监督 (1%) | 53.2% | — |
| ImageNet 半监督 (10%) | 62.2% | — |
| VOC 检测 (AP⁵⁰) | **81.4%** | 监督预训练 81.3% |
| COCO 检测 (AP) | 40.4% | 监督预训练 40.8% |
| Places365 分类 | 56.0% | 监督预训练 55.0% |

BYOL (ResNet-200, 2×) 在 ImageNet 线性探测上达到 **79.6%**，超越了当时所有自监督方法。

### 迁移学习的实用价值

BYOL 证明了**自监督学习不需要负例也能超越监督预训练**，这一发现极为重要：
1. **简化工程**：不需要维护负例队列（MoCo）或大 batch 的负例（SimCLR），降低了实现复杂度
2. **对 batch size 不敏感**：BYOL 在 BS=256 时也能稳定训练（vs SimCLR 需要 4096+）
3. **通用性强**：BYOL 的表征在检测、分割、分类等多种下游任务上均优于或媲美监督预训练
4. **后续启发**：BYOL 的不对称设计启发了 SimSiam（显式 stop-gradient）、DINO（自蒸馏）、Barlow Twins 等一系列工作
