# KAN (Kolmogorov-Arnold Networks)

## 基本信息

- **论文**: [KAN: Kolmogorov-Arnold Networks](https://arxiv.org/abs/2404.19756)
- **作者**: Ziming Liu et al. (MIT / Caltech)
- **发表**: arXiv, 2024

## 创新点

1. **Kolmogorov-Arnold 定理**: 将激活函数放在边上而非节点上（MLP 是节点激活）
2. **可学习的激活函数**: 使用 B-spline 参数化每条边的激活函数
3. **更好的可解释性**: 激活函数可视化带来直观理解

## 核心原理

### MLP vs. KAN

| 特征 | MLP | KAN |
|------|-----|-----|
| 理论基础 | 通用逼近定理 | Kolmogorov-Arnold 定理 |
| 激活位置 | 节点（固定） | 边（可学习） |
| 参数效率 | 较低 | 较高（更少参数达同等性能） |
| 可解释性 | 差 | 较好 |

### 架构

$$\\text{KAN}(x) = \\sum_{q=1}^{2n+1} \\Phi_q\\left(\\sum_{p=1}^n \\phi_{q,p}(x_p)\\right)$$

其中 $\\phi_{q,p}$ 是边上可学习的 B-spline 激活函数。

## 预训练展望与训练方法

### 核心思想：MLP 的智慧是"节点上做非线性变换"——每个节点一个激活函数，边只做线性加权。KAN 反转了这个设计——**把激活函数放在边上**，每条边都是一个可学习的 B-spline 函数。这就像从"在路口设卡检查"变成了"在每条道路上设可调节的限速"

KAN（Kolmogorov-Arnold Networks）基于柯尔莫哥洛夫-阿诺德表示定理——任何多元连续函数都可以分解为一元函数的组合。KAN 将这个数学定理转化为神经网络架构，目前在小规模任务上展示了比 MLP 更高的参数效率和可解释性。

> KAN = 边可学习的激活函数（B-spline）+ 节点仅做求和。理论基础：柯尔莫哥洛夫-阿诺德定理。与 MLP 的对比：MLP = 固定激活在节点 + 可学习权重在边；KAN = 可学习激活在边 + 固定求和（Σ）在节点。

### 训练流水线（Step by Step）

#### Step 1 — 理论基础：Kolmogorov-Arnold 定理

定理陈述：任意连续多元函数可以表示为有限个一元函数的两层嵌套：

$$f(x_1, x_2, ..., x_n) = \sum_{q=1}^{2n+1} \Phi_q\left(\sum_{p=1}^n \phi_{q,p}(x_p)\right)$$

其中 $\phi_{q,p}$ 和 $\Phi_q$ 都是一元连续函数。

| 定理 | 神经网络的对应 | 激活位置 |
|------|-------------|---------|
| 通用逼近定理（MLP） | $f(x) = W_2 \cdot \sigma(W_1 \cdot x)$ | **节点（$\sigma$ 固定）** |
| Kolmogorov-Arnold（KAN） | $f(x) = \sum \Phi(\sum \phi(x))$ | **边（$\phi$ 可学习）** |

#### Step 2 — KAN 的前向传播

与 MLP 的逐层对比：

**MLP 层**：

```text
输入 x₁, x₂, x₃

层 1 → 层 2:
  节点 1: σ(w₁₁·x₁ + w₂₁·x₂ + w₃₁·x₃)    ← 激活在节点
  节点 2: σ(w₁₂·x₁ + w₂₂·x₂ + w₃₂·x₃)    ← σ 固定 (ReLU/GELU)
  节点 3: σ(w₁₃·x₁ + w₂₃·x₂ + w₃₃·x₃)
```

**KAN 层**：

```text
输入 x₁, x₂, x₃

层 1 → 层 2:
  节点 1: Σ(φ₁₁(x₁) + φ₂₁(x₂) + φ₃₁(x₃))  ← 每条边有自己的 φ
  节点 2: Σ(φ₁₂(x₁) + φ₂₂(x₂) + φ₃₂(x₃))  ← φ 是可学习的 B-spline
  节点 3: Σ(φ₁₃(x₁) + φ₂₃(x₂) + φ₃₃(x₃))
  节点无激活函数，只做 Σ
```

**边的 B-spline 函数**：

每个 $\phi(x)$ 由 B-spline 基函数的加权组合定义：

$$\phi(x) = w_b \cdot \text{SiLU}(x) + w_s \cdot \sum_i c_i B_i(x)$$

| 组成 | 公式 | 作用 |
|------|------|------|
| 基函数 | $\text{SiLU}(x) = x \cdot \sigma(x)$ | 提供非线性基础 |
| B-spline | $\sum_i c_i B_i(x)$ | 精细调整函数形状 |
| 系数 $c_i$ | 可学习参数 | 控制 B-spline 曲线的形态 |

> 每条边的 $\phi(x)$ 是一个完整个的一元函数——类似"边上的小神经网络"。KAN 的一个隐藏层有 $n_{\text{in}} \times n_{\text{out}}$ 个这样的"边函数"。

#### Step 3 — KAN 的训练

KAN 的训练比 MLP 更复杂——因为每步前向传播需要计算 B-spline 基函数：

```text
初始化:
  - B-spline 系数 c_i: N(0, σ²) 随机初始化
  - 基函数网格: 均匀分布在输入范围内

训练步骤:
  1. 输入 x → 每条边计算 φ(x)（B-spline 基 + SiLU）
  2. 节点求和 Σ φ(x)
  3. 逐层前向传播
  4. 计算损失
  5. 反向传播 → 更新 B-spline 系数 c_i
  6. 动态更新 B-spline 网格（适应输入分布变化）
```

| 参数 | KAN（小规模） | 说明 |
|------|-----------|------|
| 网络宽度 | [2, 5, 1] | 输入 2 维 → 5 个中间节点 → 1 维输出 |
| B-spline 阶数 | 3 | 三次 B-spline |
| 网格点数 | 5-8 | 每个 φ 的基函数数 |
| 优化器 | L-BFGS | 二阶优化器（KAN 参数少） |
| 学习率 | 1e-2 | — |
| 数据 | 小型函数拟合 | sin, exp, 多项式等 |

#### Step 4 — KAN 的可解释性优势

KAN 最吸引人的特性之一：**每条边的 $\phi(x)$ 可以直接可视化**：

```text
训练后可视化:
  φ₁₁: 近似 sin(x) 形状 → 模型学到了正弦函数
  φ₂₁: 近似 x² 形状 → 模型学到了平方函数
  φ₃₂: 近似 exp(x) 形状 → 模型学到了指数函数

→ 我们可以直接"读出"模型学到了哪些数学运算
```

这与 MLP 形成鲜明对比——MLP 的可解释性极差（权重矩阵只是一堆数字）。

#### Step 5 — KAN 与大规模预训练的关系

**当前状态**：KAN 尚未在大规模预训练中实践。

| 方面 | 现状 | 挑战 |
|------|------|------|
| 任务规模 | 小型（函数拟合、偏微分方程） | 扩展到大模型 |
| 参数量 | <1M | 扩展到 B 级别 |
| 训练速度 | 慢（B-spline 计算开销） | 比 MLP 慢 ~10× |
| GPU 友好度 | 低（逐边计算，难以并行） | 需要专门优化 |
| 预训练范式 | 无 | 需开发 KAN-friendly 的预训练任务 |

**为什么 KAN 被认为是 Transformer 的潜在替代者？**

| 维度 | MLP (Transformer 的 FFN) | KAN |
|------|--------------------------|-----|
| 参数效率 | 标准 | **更高（更少参数可达同等性能）** |
| 可解释性 | 差 | **好（边函数可视化）** |
| 复杂度 | $O(n_{\text{in}} \cdot n_{\text{out}})$ | 同阶（但常数更大） |
| 理论基础 | 通用逼近定理 | **Kolmogorov-Arnold 定理** |

### KAN vs MLP

| 维度 | MLP | KAN |
|------|-----|-----|
| 激活位置 | 节点（固定激活函数） | **边（可学习 B-spline）** |
| 理论基础 | 通用逼近定理 | **Kolmogorov-Arnold 定理** |
| 每个激活函数 | 所有输入共享（如 ReLU） | **每条边有自己独特的函数** |
| 函数复杂度 | 单次非线性 | **B-spline 基函数组合** |
| 参数效率 | 较低 | 较高 |
| 可解释性 | 差 | 好 |

### 训练的实用价值

1. **新神经网络范式的提出**：MLP 统治了 40 年——KAN 提出了全新的可能性
2. **可解释 AI 的突破**：边函数的可视化使理解网络变得可能
3. **符号回归的新工具**：KAN 可以自动发现数据中的数学公式
4. **小样本学习的潜力**：参数效率更高 → 少数据场景的优势
5. **对 Transformer FFN 的替代潜力**：如果 KAN 解决了规模问题，Transformer 的 FFN 层可以用更可解释的 KAN 层替换
