# DAE (Denoising Autoencoder)

## 基本信息

- **论文**: [Extracting and Composing Robust Features with Denoising Autoencoders](https://www.cs.toronto.edu/~larocheh/publications/icml-2008-denoising-autoencoders.pdf)
- **作者**: Pascal Vincent et al. (Université de Montréal)
- **发表**: ICML 2008

## 创新点

1. **去噪预训练**: 从加噪输入中重建原始数据，学习鲁棒的表征
2. **Dropout 的前身**: 输入噪声相当于一种正则化
3. **自监督学习的早期探索**: 无需标签即可学习有意义表征

## 核心原理

### 去噪自编码器

给定干净输入 $x$：
1. 通过随机损坏过程 $C(\\tilde{x}|x)$ 生成噪声输入 $\\tilde{x}$
2. 编码器将 $\\tilde{x}$ 映射到隐空间 $h$
3. 解码器从 $h$ 重建原始 $x$
4. 损失: $\\mathcal{L} = \\| \\hat{x} - x \\|^2$

### 为什么有效？

DAE 迫使模型学习数据流形的结构，而非简单地复制输入。

## 训练方法

### 核心思想：给你一张被泼了墨水的照片，你能还原出原来的样子吗？如果你的回答是"能"，那是因为你理解了照片中的结构（人脸有两只眼睛、一个鼻子，位置是固定的）。DAE（Denoising Autoencoder）正是利用这个洞察——故意给输入加噪，让网络从噪声中还原原始信号。为了成功，网络**必须学会数据的底层结构**

DAE 是 2008 年提出的，但它的思想影响了整整 15 年后的所有模型——BERT 的 MLM、MAE 的 Patch Masking、扩散模型，本质上都是 DAE 的"孙辈"。

> DAE = 输入加噪 → 编码器压缩 → 解码器重建干净输入 → MSE/交叉熵。核心理念：从损坏中重建的能力 = 对数据结构的深层理解。

### 训练流水线（Step by Step）

#### Step 1 — 去噪自编码器的基本流程

```text
干净输入: x
  ↓ 损坏过程 C(·|x)
噪声输入: x̃ (加噪版本)
  ↓ 编码器 h = f(x̃)
潜表示: h
  ↓ 解码器 x̂ = g(h)
重建: x̂

损失: L = ||x̂ - x||²  (重建的是干净 x，不是 x̃!)
```

**关键细节**：损失是**重建干净输入 $x$**，而不是重建噪声输入 $\tilde{x}$——这意味着网络必须"看穿"噪声。

#### Step 2 — 噪声类型与损坏过程

| 噪声类型 | 描述 | 应用场景 |
|---------|------|---------|
| **加性高斯噪声** | $\tilde{x} = x + \mathcal{N}(0, \sigma^2)$ | 连续数据（图像像素） |
| **椒盐噪声** | 随机像素置为 0 或 1 | 图像降噪 |
| **掩码噪声 (Dropout Noise)** | 随机将输入元素置零 | **BERT 的灵感来源！** |
| **遮盖噪声** | 随机遮挡连续区域 | **MAE 的灵感来源！** |
| **加性拉普拉斯噪声** | 重尾噪声 | 对异常值更鲁棒 |

**损坏程度的权衡**：

| 损坏程度 | 效果 |
|---------|------|
| 太低（<10%） | 网络只需要简单插值 → 学不到结构 |
| 中等（30-50%） | 最佳 → 必须理解底层结构才能重建 |
| 太高（>80%） | 信息损失太大 → 无法重建 |

#### Step 3 — 编码-解码（Bottleneck 设计）

```text
DAE 的 Bottleneck 结构:

输入 (784 维, MNIST 28×28)
  ↓ 损坏: 随机 30% 置零
噪声输入 (784 维)
  ↓ Encoder: 784 → 256 → 128 → 64
潜表示 (64 维!)  ← 关键: 必须用压缩维度!
  ↓ Decoder: 64 → 128 → 256 → 784
重建输出 (784 维)

损失: MSE(输出, 干净输入)
```

**为什么必须要 Bottleneck（低维潜表示）？**

| 有 Bottleneck | 无 Bottleneck |
|-------------|-------------|
| 被迫压缩学到的特征 | 简单复制输入（identity mapping） |
| 学到有意义的流形 | 学不到任何东西 |
| 类似"理解" | 类似"背诵" |

#### Step 4 — 堆叠 DAE（逐层预训练）

DAE 诞生于深度学习早期——当时训练深层网络很困难。堆叠 DAE 因此提出了逐层预训练的范式：

```text
阶段 1: 训练第一层
  输入: 原始数据 x
  加噪 → 单层 DAE → 重建
  训练后: 冻结 W_1, b_1

阶段 2: 训练第二层
  输入: 第一层的潜在表示 h_1 = σ(W_1·x + b_1)
  加噪 → 单层 DAE → 重建 h_1
  训练后: 冻结 W_2, b_2

阶段 3: ... 重复 ...

阶段 N: 全局微调
  将所有层堆叠 → 在分类任务上微调全部参数
```

| 参数 | 堆叠 DAE | 说明 |
|------|---------|------|
| 层数 | 3-5 | 逐层训练 |
| 编码维度 | 784→256→128→64→32 | 逐步压缩 |
| 噪声率 | 30% | 逐层相同 |
| 激活 | Sigmoid / Tanh | 当时标准 |
| 每层训练 Epoch | 50-100 | — |
| 微调 Epoch | 20-50 | 有标签 |

#### Step 5 — DAE 的思想谱系

```text
DAE (2008)
  → 机器学习: dropout 理论
  → BERT (2019): 随机 token 掩码 = "文本 DAE"
  → MAE (2022): 随机 patch 掩码 = "图像 DAE"
  → Diffusion Models (2015→2022): 逐步加噪→逐步去噪
  → Stable Diffusion (2022): 潜空间中的 DAE
```

**DAE → BERT 的关系**：

| 维度 | DAE | BERT |
|------|-----|------|
| 噪声类型 | 输入置零 | Token 替换为 [MASK] |
| 重建目标 | 原始像素 | 原始 token |
| Bottleneck | 低维潜空间 | Transformer Encoder |
| 年份 | 2008 | 2019 |

> BERT 本质上就是一个**堆叠 DAE，把高斯噪声换成了 [MASK] token**——DAE 的思想 11 年后统治了 NLP 和 CV。

| Epoch | 100 | —

### 训练的实用价值

1. **现代掩码预训练的祖父**：BERT、MAE、BEiT 都是 DAE 的后代
2. **噪声 = 正则化**：为 Dropout 提供了理论解释
3. **逐层预训练范式**：解决了当时深层网络训练困难的瓶颈
4. **流形学习视角**：DAE 学习的是数据流形的结构
5. **扩散模型的前身**：逐步加噪→逐步去噪的思想源于 DAE
