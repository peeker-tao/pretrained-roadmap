# VideoMAE

## 基本信息

- **论文**: [VideoMAE: Masked Autoencoders are Data-Efficient Learners for Self-Supervised Video Pre-Training](https://arxiv.org/abs/2203.12602)
- **作者**: Zhan Tong et al.
- **发表**: NeurIPS 2022

## 创新点

1. **视频时空掩码**: 将 MAE 扩展到视频的时空域
2. **极高掩码率 (90-95%)**: 利用视频的时间冗余
3. **数据高效**: 在较少的视频数据上有效预训练

## 核心原理

### 时空掩码策略

- 在时间维度上随机掩码整个帧或帧片段
- 在空间维度上随机掩码 patch
- 掩码率远高于图像 MAE（90-95% vs 75%）

## 预训练方法

### 核心思想：MAE 在图像上用 75% 掩码就取得了惊人的效果——视频有天然的时间冗余，可以遮得更多！90-95% 的掩码率下，模型必须真正"理解"场景的时空结构才能重建出被遮的帧

VideoMAE 是 MAE 在视频领域的直接扩展。视频比图像多了时间维度，这意味着：**相邻帧高度相似，时间冗余极高**。VideoMAE 利用这一点——将掩码率从图像的 75% 提升到视频的 90-95%，迫使模型学习更深层的时空语义。

> VideoMAE = 极高掩码率（90-95%）+ 时空掩码策略 + VideoViT Encoder + 轻量 Decoder + 像素回归。视频的高冗余性使得"几乎全遮"成为可能——模型需要在 5-10% 的可见信息中推断被遮的 90%。

### 训练流水线（Step by Step）

#### Step 1 — 视频 Tokenization

视频被切分为时空立方体：

```text
输入: T 帧 × H × W × 3 (如 16×224×224×3)

切分（Tubelet Embedding）:
  时间维度: 每 2 帧一组
  空间维度: 16×16 patch
  → 8 × 14 × 14 = 1568 个时空立方体（token）
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 帧数 | 16 | 标准输入 |
| Patch 大小 | 2×16×16 | 时间×空间 |
| Token 数 | 8 × 14 × 14 | 1568 个 |

相比图像（196 个 token），视频有 1568 个 token——多 8× 的信息量。

#### Step 2 — 极高掩码率（90-95%）

这是 VideoMAE 最核心的设计选择：

| 方法 | 掩码率 | 可见 token | 理由 |
|------|--------|----------|------|
| MAE (图像) | 75% | 49 / 196 | 图像信息密度低 |
| **VideoMAE** | **90%** | **157 / 1568** | **视频时间冗余极高** |
| **VideoMAE (极难)** | **95%** | **78 / 1568** | **最大化学习难度** |

**为什么视频可以遮这么多？**

图像中相邻空间 patch 之间的信息重叠有，但有限。视频中相邻**时间帧**之间的信息重叠极高——相邻两帧几乎一模一样（除非有快速运动）。

> 类比：图像遮住 75% 就像把照片撕掉 3/4 来猜内容；视频遮住 95% 就像给你一本连环画的 5% 页面来猜整个故事——你需要理解"运动"和"场景"才能补全。

#### Step 3 — 时空掩码策略

VideoMAE 不是随机散布掩码，而是有策略地遮住视频块：

| 策略 | 描述 | 效果 |
|------|------|------|
| **随机管状掩码** | 在时间维度上连续遮住同一空间位置的 patch | 遮挡了一个"时空管道" |
| **帧级掩码** | 随机遮住整帧 | 最难的挑战 |
| **空间块掩码** | 类似 MAE 的空间大块掩码 | 中等难度 |

**管状掩码（Tube Masking）的优势**：
- 如果只在空间上随机散布，相邻帧的同位置 patch 会泄露信息（时间泄漏）
- 管状掩码确保同一时空位置在所有帧中都被遮 → 防止时间泄露

```text
帧1: ░░░░░░░░██████████  ░=可见 █=遮住
帧2: ░░░░░░░░██████████  ← 同一空间位置在时间上也被遮
帧3: ░░░░░░░░██████████
帧4: ░░░░░░░░██████████
```

#### Step 4 — Encoder + Decoder（非对称设计）

**Encoder**（仅处理可见 token）：

```text
可见 token (157 个, 90%掩码时) → VideoViT Encoder
  → 每层 Self-Attention 仅在 157 个 token 间计算
  → 输出: 增强的可见 token 特征
```

**Decoder**（处理所有 token，重建被遮部分）：

```text
Encoder 输出 + [MASK] token (1411 个)
  → 轻量 VideoViT Decoder (4 层)
  → 每个 token → 预测对应时空立方体的 RGB 像素
```

**Encoder-Decoder 分离的意义**：
- Encoder 只处理 10% 的 token → 计算极快（与 MAE 同配方）
- Decoder 处理 100% token 但层数很少（4 层 vs Encoder 的 16 层）
- 总计算量 ≈ 原始 ViT 的 ~20%

#### Step 5 — 完整训练流程

```text
1. 视频 (16×224×224) → Tubelet Embedding → 1568 tokens
2. 随机管状掩码 → 90-95% token 被 mask
3. 仅 157 (10%) 或 78 (5%) 个可见 token 进入 Encoder
4. Encoder 输出 + [MASK] tokens → Decoder
5. 仅对被遮区域计算 MSE（像素值重建）
6. 反向传播
```

### VideoMAE vs MAE

| 维度 | MAE (图像) | VideoMAE |
|------|----------|---------|
| 输入维度 | 2D (H×W) | 3D (T×H×W) |
| Token 数 | 196 | 1568（8×） |
| 掩码率 | 75% | **90-95%** |
| 掩码策略 | 空间随机块 | **时空管状掩码** |
| 时间冗余 | 无 | **极高（相邻帧几乎相同）** |
| Encoder 计算 | ~25% of full | ~10% of full |

### 详细训练配置

| 参数 | VideoMAE (ViT-B) | VideoMAE (ViT-L) |
|------|-----------------|-----------------|
| 数据集 | Kinetics-400 | Kinetics-700 |
| 帧数 | 16 | 16 |
| 空间尺寸 | 224×224 | 224×224 |
| 管状 Patch | 2×16×16 | 2×16×16 |
| 掩码率 | 90% | 90% |
| Encoder 层数 | 12 | 24 |
| Decoder 层数 | 4 | 4 |
| 优化器 | AdamW | AdamW |
| 学习率 | 1.5e-4 | 1e-4 |
| Batch Size | 512 | 256 |
| Epoch | 800 | 800 |

### 预训练的实用价值

1. **视频自监督的新范式**：极高掩码率 + 轻量 Decoder → 极高效的视频预训练
2. **数据效率的突破**：视频标注成本极高（标注 1 秒视频 = 标注 30 帧），VideoMAE 的自监督预训练大幅提升了标注效率
3. **时间冗余的利用**：将视频的"缺点"（冗余）转化为"优势"（允许更高掩码率）
4. **动作识别的最强基座**：Kinetics-400 上 VideoMAE 达到 87.4% Top-1（ViT-L）
5. **视频理解任务的通用 backbone**：动作识别、时序检测、视频问答
