# 第 9 章：预训练的关键技术要素

预训练方法的成功不仅取决于目标函数的设计，还依赖于**模型架构、数据策略、优化方法和训练基础设施**等一系列技术要素的选择。

---

## 9.1 模型架构选择

### CNN 架构

- **ResNet** (He et al., 2016) — [[论文]](https://arxiv.org/abs/1512.03385)
  - 残差连接，简单高效的骨干网络
- **ConvNeXt** (Liu et al., 2022) — [[论文]](https://arxiv.org/abs/2201.03545)
  - 将 Swin Transformer 的设计灵感现代化 CNN
  - 使用 GELU 激活、LayerNorm、大卷积核

### Transformer 架构

- **ViT (Vision Transformer)** (Dosovitskiy et al., 2021) — [[论文]](https://arxiv.org/abs/2010.11929)
  - 将图像分割为 16×16 的固定大小 patch，直接应用 Transformer 编码器
  - 在大规模数据上超过 CNN，在小数据集上不如 CNN
  - **自监督训练对 ViT 尤为重要**（DINO, MAE）
- **Swin Transformer** (Liu et al., 2021) — [[论文]](https://arxiv.org/abs/2103.14030)
  - 层次化特征图（类似 CNN）
  - 移位窗口（Shifted Window）注意力
  - 适合检测、分割等稠密预测任务

### 混合架构

- **ConViT** (d'Ascoli et al., 2021) — [[论文]](https://arxiv.org/abs/2103.10656)
  - 在 Transformer 中引入卷积归纳偏置
- **CoAtNet** (Dai et al., 2021) — [[论文]](https://arxiv.org/abs/2106.04803)
  - 深度混合卷积 + 注意力

### 状态空间模型 (SSM)

- **Mamba** (Gu & Dao, 2023) — [[论文]](https://arxiv.org/abs/2312.00752)
  - 选择性状态空间模型
  - 线性复杂度的序列建模
  - 在长序列任务上匹敌甚至超越 Transformer
- **Vision Mamba** (Zhu et al., 2024) — [[论文]](https://arxiv.org/abs/2401.09417)
  - Mamba 在视觉领域的改编
- **Jamba** (Lieber et al., 2024) — [[论文]](https://arxiv.org/abs/2403.19887)
  - Mamba + Transformer 混合：吸收两者优势

### 架构对比

| 架构 | 计算复杂度 | 归纳偏置 | 长序列 | 典型预训练方法 |
|------|-----------|---------|--------|-------------|
| CNN | $O(N)$ | 强（平移不变性） | 差 | 监督、对比 |
| Transformer | $O(N^2)$ | 弱 | 好 | MAE, BERT, CLIP |
| Swin | $O(N \\log N)$ | 中（窗口偏置） | 好 | SimMIM, BYOL |
| Mamba | $O(N)$ | 弱 | 极好 | 新兴方向 |

## 9.2 数据策略

### 数据规模

预训练数据的规模经历了爆炸式增长：

| 年份 | 数据集 | 规模 | 说明 |
|------|--------|------|------|
| 2012 | ImageNet-1K | 1.3M 图片 | 手工标注 |
| 2018 | JFT-300M | 3 亿图片 | 弱标签（Google） |
| 2020 | CLIP WIT | 4 亿图文对 | 互联网采集 |
| 2021 | ALIGN | 10 亿图文对 | 噪声图文对 |
| 2021 | LAION-5B | 50 亿图文对 | 开源（目前已下线） |
| 2022 | Instagram-3.5B | 35 亿图片 | 弱标签（Meta） |
| 2024 | DCLM | 4T tokens | 文本（Apple） |

### 数据质量

**"数据质量比数据规模更重要"** 正成为共识：

- **过滤**：去除低质量、重复、不当内容
- **去重**：精确去重 + 模糊去重（SemDeDup, MinHash）
- **质量评分**：利用模型（如 CLIP 分数）评估数据质量
- **DataComp (2023)** — [[论文]](https://arxiv.org/abs/2304.14108)：
  - 系统性地研究了数据筛选策略
  - 发现在数据筛选上投入算力比增加模型参数量回报更高

### 数据增强

SimCLR 最重要的发现之一是：**数据增强的决定性作用**。

| 增强方法 | 适用场景 | 重要性 |
|---------|---------|--------|
| 随机裁剪 + 缩放 | 通用视觉 | ⭐⭐⭐ |
| 色彩抖动（Color Jitter） | 通用视觉 | ⭐⭐⭐ |
| 高斯模糊 | 对比学习 | ⭐⭐ |
| 旋转 / 翻转 | 通用视觉 | ⭐⭐ |
| CutMix / MixUp | 监督学习 | ⭐⭐ |
| RandAugment | 通用增强 | ⭐⭐ |

### 课程学习

根据训练阶段动态调整数据难度：

- 先易后难的训练策略
- 数据难度与训练步数的匹配

### 合成数据

利用生成模型产生预训练数据成为新兴方向：

- **DALL·E** 生成对比训练数据
- **LLM 合成文本**用于多模态训练
- **数据孵化（Data Incubation）**：用强模型生成数据来训练弱模型

## 9.3 优化与训练策略

### 优化器演进

| 优化器 | 特点 | 适用场景 |
|--------|------|---------|
| SGD + Momentum | 简单、稳定 | 监督预训练 |
| Adam / AdamW | 自适应学习率 | 自监督 / Transformer |
| LAMB | 大 batch 训练 | 大规模预训练 |
| Lion (2023) | 内存效率高 | 大规模训练 |

- **AdamW** (Loshchilov & Hutter, 2019) — [[论文]](https://arxiv.org/abs/1711.05101)
  - 解耦权重衰减（Weight Decay）和学习率
  - Transformer 训练的标准选择
- **LAMB** (You et al., 2020) — [[论文]](https://arxiv.org/abs/1904.00962)
  - 支持 batch size 扩展到 65536 而不损失精度

### 学习率调度

- **Cosine Decay**：余弦退火，平滑降低学习率
- **Warmup**：训练初期线性升温，稳定梯度
- **Linear Scaling Rule**：batch size 翻倍时，学习率也大致翻倍

### 批次大小

| 规模 | Batch Size | 代表模型 |
|------|-----------|---------|
| 小 | 256–1024 | 早期 CNN |
| 中 | 4096 | SimCLR |
| 大 | 16384–32768 | CLIP, MAE |
| 超大 | 65536 | LAMB 训练 |

**Scaling Rule**: 大 batch 需要配合更高学习率和更长 warmup。

### 混合精度训练

| 精度 | 特点 | 适用范围 |
|------|------|---------|
| FP32 | 数值稳定，速度慢 | 小模型 |
| FP16 | 速度快，需要 loss scaling | A100 前 |
| BF16 | 数值范围广，无需 scaling | A100 起 |
| FP8 | 最新，效率最高 | H100 起 |

### 分布式训练框架

| 策略 | 描述 | 代表框架 |
|------|------|---------|
| 数据并行（DDP） | 复制模型，切分数据 | PyTorch DDP |
| ZeRO 系列 | 分阶段切分优化器/梯度/参数 | DeepSpeed |
| FSDP | 全分片数据并行 | PyTorch FSDP |
| 模型并行 | 切分模型层 | Megatron-LM |
| 流水线并行 | 多层流水线 | GPipe |
| 张量并行 | 切分单层矩阵 | Megatron-LM |

### Scaling Laws

#### Kaplan Scaling (2020)

- **论文**：[Scaling Laws for Neural Language Models](https://arxiv.org/abs/2001.08361)
- **作者**：Jared Kaplan et al. (OpenAI)
- **核心发现**：
  - 模型参数量 × 数据量 × 计算量的幂律关系
  - 计算量翻倍时，参数量和数据量应等比例放大

#### Chinchilla Scaling (2022)

- **论文**：[Training Compute-Optimal Large Language Models](https://arxiv.org/abs/2203.15556)
- **作者**：Jordan Hoffmann et al. (DeepMind)
- **核心发现**：
  - 对于给定的计算预算，**模型参数和数据量比例有最优值**
  - 许多模型（GPT-3）参数过多而数据不足
  - Chinchilla: 70B 参数、1.4T tokens，性能超 GPT-3 (175B)

## 9.4 正则化与稳定性

### 权重衰减 (Weight Decay)

- 等效于 L2 正则化
- 在自监督学习中尤为重要（防止表征坍塌）
- 通常取值：0.1–0.0001

### Dropout / Stochastic Depth / DropPath

- **Dropout** (Srivastava et al., 2014)：随机丢弃神经元
- **Stochastic Depth** (Huang et al., 2016)：随机跳过网络层
- **DropPath**：Stochastic Depth 在 ViT 中的实现

### 归一化层

| 方法 | 归一化维度 | 适用架构 |
|------|-----------|---------|
| BatchNorm (BN) | 通道维度 | CNN |
| LayerNorm (LN) | 特征维度 | Transformer |
| RMSNorm | LN 的简化版 | LLaMA 系列 |
| GroupNorm (GN) | 分组 | 小 batch |

### Gradient Clipping

- 将梯度范数截断到给定阈值
- 防止梯度爆炸
- 典型阈值：1.0

### 学习率 Warmup

- 防止训练初期的不稳定
- 线性 warmup + Cosine Decay 是标准配置
