## 📋 概览

预训练（Pre-training）已成为现代深度学习的标准范式，但方法演进极快：从早期的 ImageNet 监督预训练，到 NLP 领域的 GPT/BERT，再到 CV 领域的 MAE 与对比学习，以及当下的多模态基础模型。不同领域的技术往往独立发展，但底层逻辑高度相通。

本文按**预训练目标函数**与**监督信号类型**重新组织内容，将经典模型归入其对应的技术路径，呈现一幅清晰的"预训练方法地图"。

## 1. 引言

### 1.1 背景与动机

- 深度学习对大规模标注数据的依赖
- 预训练—微调范式的诞生与演进
- 为什么预训练有效：表征学习、正则化、知识的迁移

### 1.2 核心问题

- 什么样的预训练目标函数能学到最好的表征？
- 不同监督信号（标签、文本、跨模态对齐）如何影响表征质量？
- 预训练方法是否存在统一的底层原理？

### 1.3 本文范围与结构

- 涵盖 CV、NLP、多模态领域的代表性预训练方法
- 按**目标函数类型**而非应用领域组织
- 重点分析 2015–2025 年间的方法演进

## 2. 预训练的范式演进

### 2.1 监督预训练时代（2012–2017）

- **关键事件**：AlexNet (2012) → ImageNet 预训练成为 CV 标准范式
- **核心思想**：在大规模标注数据上训练，然后迁移到下游任务
- **局限**：依赖人工标注，泛化边界受限于标签空间

### 2.2 自监督学习革命（2018–2021）

- **NLP 突破**：
  - GPT (2018)：自回归语言建模
  - BERT (2019)：掩码语言建模（MLM）
  - RoBERTa、XLNet、T5 等改进
- **CV 突破**：
  - SimCLR、MoCo (2020)：对比学习
  - MAE (2021)：掩码自编码器
  - BYOL、SimSiam：无负例对比学习
- **核心驱动力**：无需人工标注，可扩展至海量数据

### 2.3 基础模型时代（2021–至今）

- **大规模多模态**：CLIP、DALL·E、GPT-4V、Gemini
- **统一架构**：Transformer 成为跨模态通用骨干
- **涌现能力**：规模扩展带来的零样本、少样本、思维链等能力
- **新范式**：Foundation Model → 提示/指令微调 → 对齐

## 4. 监督预训练

### 4.1 ImageNet 监督预训练

- **方法概述**：在大规模标注数据集上训练分类任务
- **关键模型**：AlexNet, VGG, GoogLeNet, ResNet, EfficientNet
- **核心贡献**：
  - 层级化特征的自动学习
  - 迁移学习的标准化 pipeline
- **优势**：稳定、成熟、广泛验证
- **局限**：
  - 依赖大规模人工标注
  - 表征受限于标签空间的粒度
  - 对域偏移（domain shift）敏感

### 4.2 弱监督预训练

- **方法**：利用噪声标签、Webly-supervised 数据
- **代表**：Noisy Student (EfficientNet-L2), JFT-3B (BiT)
- **核心思想**：用更大规模的低质量数据补偿标签质量

### 4.3 度量学习预训练

- **目标**：学习判别性嵌入而非分类边界
- **方法**：Triplet Loss, ArcFace, CosFace, Margin Loss
- **应用**：人脸识别、图像检索、行人重识别

## 5. 生成式自监督预训练

### 5.1 自回归语言建模

- **核心思想**：预测序列中的下一个 token
- **关键模型**：
  - GPT-1/2/3/4：扩大模型与数据规模
  - LLaMA 系列：高质量数据 + 高效训练
  - PaLM, Chinchilla：Scaling Laws 指导下的优化
- **技术要点**：
  - Causal Attention Mask
  - 单向上下文建模
  - Prompting / In-Context Learning
- **优势**：强大的生成能力，涌现推理能力
- **局限**：仅利用单向上下文

### 5.2 掩码语言建模 (Masked Language Modeling)

- **核心思想**：随机掩码部分 token，根据双向上下文重建
- **关键模型**：
  - BERT：MLM + NSP
  - RoBERTa：去除 NSP，动态掩码
  - T5：Span Corruption 的统一文本到文本框架
  - ALBERT：参数共享 + SOP
  - ELECTRA：判别式预训练（Replaced Token Detection）
- **技术要点**：
  - 双向注意力
  - 掩码策略（静态/动态/span/whole word）
  - 训练-推理不匹配（masking 只在预训练）
- **优势**：充分利用双向上下文
- **局限**：掩码带来的训练-推理 gap

### 5.3 掩码图像建模 (Masked Image Modeling)

- **核心思想**：掩码图像块，根据可见部分重建
- **关键模型**：
  - MAE (He et al., 2021)：非对称编码器-解码器，高掩码率 (75%)
  - SimMIM (Xie et al., 2021)：简单掩码策略
  - MaskFeat (Wei et al., 2022)：预测 HOG 特征而非像素
  - BEiT (Bao et al., 2021)：预测 DALL-E 的离散编码
  - ConvNeXt-V2 (Woo et al., 2023)：全卷积掩码自编码器
- **技术要点**：
  - 掩码策略（随机/块状/语义）
  - 重建目标（像素/特征/离散 code）
  - 编码器-解码器结构设计
- **与 MLM 的对比**：视觉信号的连续性与语言信号的离散性

### 5.4 去噪自编码器 (Denoising Autoencoders)

- **核心思想**：从加噪的输入中重建原始数据
- **代表工作**：DAE, Stacked DAE, Noise-Contrastive AE
- **与其他方法的关系**：MAE 可视作 DAE 在视觉领域的特化

### 5.5 生成式 vs. 对比式的比较

| 维度 | 生成式 (MAE/MIM) | 对比式 (SimCLR/MoCo) |
|------|-----------------|---------------------|
| 目标函数 | 像素/特征重建 | 实例判别 |
| 归纳偏置 | 局部相关性 | 全局不变性 |
| 下游性能 | 分类 + 分割优 | 线性探测优 |
| 训练效率 | 计算效率高 | 需要大量负例 |
| 模态适用 | 通用 | 更适合语义任务 |

### 5.6 生成式与对比式的统一框架尝试

- **iBOT**：结合 MIM + 对比学习
- **MaskFeat**：预测 HOG 特征
- **data2vec**：自回归 + 掩码的统一框架
- **CAN**：对比 + 掩码的协同训练

## 6. 对比式自监督预训练

### 6.1 实例判别 (Instance Discrimination)

- **核心思想**：将每个样本视为独立类别，拉近增强视图、推远不同样本
- **关键模型**：
  - **SimCLR (2020)**：
    - 核心组件：数据增强、投影头、InfoNCE Loss
    - 大 batch size 是关键 (4096+)
  - **MoCo (2020–2021)**：
    - 动量编码器 + 队列机制
    - 解耦 batch size 与负例数量
    - MoCo v1 → v2 → v3 演进
  - **NPID / CPC / AMDIM**：早期对比学习探索
- **InfoNCE Loss** 的数学形式与变体
- **关键设计选择**：
  - 正负例定义
  - 数据增强策略（Color jitter, 裁剪, 高斯模糊）
  - 投影头 vs. 预测头
  - 负例数量与质量

### 6.2 聚类对比 (Clustering-based Contrastive)

- **核心思想**：在对比学习框架中引入聚类作为隐式监督
- **关键模型**：
  - **SwAV**：在线聚类 + 对比学习，Sinkhorn-Knopp 算法
  - **DeepCluster**：交替聚类与特征学习
  - **SeLa**：自标注 + 对比学习
- **优势**：利用数据分布的结构信息
- **局限**：聚类质量的稳定性问题

### 6.3 跨模态对比学习

- **核心思想**：利用另一个模态的语义作为监督信号
- **关键模型**：
  - **CLIP (2021)**：4 亿图文对，对比预训练
  - **ALIGN (2021)**：10 亿噪声图文对
  - **SigLIP (2023)**：Sigmoid Loss 替代 Softmax
  - **ImageBind (2023)**：6 模态统一嵌入空间
- **核心贡献**：零样本迁移能力，连接视觉与语言
- **技术要点**：
  - 双塔架构 (dual-encoder)
  - 对比损失函数设计
  - 数据规模与质量的影响

## 7. 非对比式／联合嵌入预训练

### 7.1 自蒸馏方法

- **核心思想**：不依赖负例，通过不对称网络防止坍塌
- **关键模型**：
  - **BYOL (2020)**：
    - 在线网络 + 目标网络（动量更新）
    - 预测头 (predictor) 防止坍塌
    - 隐式正则化的理论分析
  - **DINO / DINOv2 (2021–2023)**：
    - 自蒸馏 + 中心化 + 锐化
    - ViT 特性：自注意力图蕴含语义分割信息
    - DINOv2：大规模数据 + 蒸馏 + 增强策略
  - **SimSiam (2021)**：
    - 极简框架：只需 stop-gradient 即可防止坍塌
    - 理论解释：EM 算法的隐式实现

### 7.2 协方差正则化方法

- **核心思想**：约束嵌入向量的协方差矩阵接近单位阵
- **关键模型**：
  - **Barlow Twins (2021)**：
    - 跨视图互相关矩阵 → 单位阵
    - 冗余减少 (Redundancy Reduction) 原则
  - **VICReg (2022)**：
    - Variance + Invariance + Covariance Regularization
    - 解耦了不需要负例的机制
  - **W-MSE**：白化 MSE

### 7.3 非对比方法的统一视角

- Stop-gradient 的作用
- 不对称网络设计
- 隐式正则化 vs. 显式正则化
- 对比式 vs. 非对比式的理论关系

## 8. 多模态预训练

### 8.1 视觉-语言对比预训练

- **模型**：CLIP, ALIGN, SigLIP, BLIP, CoCa
- **核心设计维度**：
  - 单塔 vs. 双塔
  - 对比损失 vs. 匹配损失 vs. 生成损失
  - 数据策略：Web-scale 数据清洗

### 8.2 视觉-语言生成预训练

- **模型**：
  - **Flamingo (2022)**：冻结预训练模型，插入 gated cross-attention
  - **BLIP-2 (2023)**：Q-Former 连接视觉编码器与 LLM
  - **LLaVA (2023–2024)**：简单投影层连接 CLIP + Vicuna
  - **CogVLM, MiniGPT-4, InstructBLIP**
- **核心问题**：如何高效桥接视觉与语言编码器

### 8.3 统一多模态基础模型

- **GPT-4V / GPT-4o**：多模态输入 + 端到端训练
- **Gemini**：原生多模态（文本+图像+音频+视频）
- **Fuyu (Adept)**：简化多模态架构
- **Next-Gen 趋势**：原生多模态（非拼接式）

### 8.4 视频与 3D 预训练

- **视频**：VideoMAE, TimeSformer, VideoCLIP
- **3D 点云**：Point-BERT, PointContrast, 3DETR
- **音频**：AudioMAE, wav2vec 2.0, HuBERT

## 9. 预训练的关键技术要素

### 9.1 模型架构选择

- **CNN**：2D/3D 卷积，归纳偏置强
- **Transformer**：ViT, Swin, ConvNeXt
- **混合架构**：ConViT, CoAtNet, MaxViT
- **状态空间模型**：Mamba-2, Jamba, Vision Mamba

### 9.2 数据策略

- **数据规模**：从 ImageNet (1.3M) 到 LAION-5B (50亿)
- **数据质量**：过滤、去重、质量评分（例如 DataComp 实验）
- **数据增强**：SimCLR 见解 → 数据增强的决定性作用
- **课程学习**：数据难度与训练阶段的匹配
- **合成数据**：利用生成模型产生预训练数据

### 9.3 优化与训练策略

- **优化器**：SGD → Adam/AdamW → LAMB → Lion
- **学习率调度**：Cosine decay, Warmup, Linear scaling rule
- **批次大小**：从 256 到 65536 的扩展
- **训练时长**：Epoch 数 vs. 数据重复利用
- **混合精度训练**：FP16 → BF16 → FP8
- **分布式训练**：数据并行, 模型并行, ZeRO, FSDP
- **Scaling Law**：Kaplan Scaling, Chinchilla Scaling

### 9.4 正则化与稳定性

- Weight Decay 的作用
- Dropout / Stochastic Depth / DropPath
- LayerNorm / BatchNorm / RMSNorm
- Gradient clipping

## 10. 预训练模型的应用与迁移

### 10.1 下游任务适配方法

| 方法 | 描述 | 适用场景 |
|------|------|----------|
| 全量微调 (Full Fine-tuning) | 更新所有参数 | 下游数据充足 |
| 线性探测 (Linear Probing) | 仅训练分类头 | 快速评估表征质量 |
| 部分微调 | 只调最后几层 | 中等规模数据 |
| Adapter / Houlsby | 插入小网络 | 参数高效，多任务 |
| Prefix Tuning / Prompt Tuning | 学习连续提示 | 语言模型适配 |
| LoRA / QLoRA | 低秩适配矩阵 | 大模型微调的主流方案 |
| 视觉提示 (Visual Prompt) | 输入空间改适 | 免参数修改 |

### 10.2 领域迁移分析

- 同域迁移：预训练与下游数据分布一致
- 跨域迁移：自然图像 → 医学图像、卫星图像
- 跨任务迁移：分类 → 检测 → 分割 → 生成
- 迁移效率：多少数据足够？

### 10.3 零样本与少样本学习

- **零样本**：CLIP 的分类能力，GPT 的推理能力
- **少样本**：In-Context Learning, MAML 系列
- **涌现能力**：模型规模与能力的非连续关系

## 11. 评估基准与数据集

### 11.1 视觉评估

- **线性探测**：ImageNet Top-1, Places205, iNaturalist
- **半监督**：1% / 10% 标签下的 ImageNet 准确率
- **迁移学习**：VTAB (Visual Task Adaptation Benchmark)
- **检测/分割**：COCO, ADE20K, Cityscapes

### 11.2 NLP 评估

- **语言理解**：GLUE, SuperGLUE, SQuAD
- **推理**：MMLU, BIG-bench, GSM8K
- **生成**：BLEU, ROUGE, BERTScore

### 11.3 多模态评估

- **零样本分类**：ImageNet zero-shot
- **图文检索**：MSCOCO, Flickr30K
- **视觉问答**：VQAv2, GQA, VizWiz
- **多模态推理**：MMBench, MMMU, MME

## 12. 挑战与未来方向

### 12.1 当前的开放挑战

- **预训练-微调 gap**：预训练目标与下游目标的错配
- **数据瓶颈**：高质量数据的消耗即将见顶
- **计算成本**：万亿参数模型的训练与推理效率
- **评估不完整**：线性探测能否反映表征的真实质量？
- **理论基础薄弱**：对比学习的坍塌原因、MIM 的有效性来源尚不清晰
- **多模态对齐**：异构数据的统一表征仍然困难

### 12.2 未来方向

- **超越 Transformer**：状态空间模型、KAN、物理启发架构
- **数据高效的预训练**：合成数据、主动学习、课程学习
- **持续预训练**：参数更新下的知识保持
- **具身与世界模型**：预训练不只是"看"，更要"做"
- **科学领域预训练**：蛋白质、材料、气候等科学数据
- **安全保障**：对齐、越狱防护、幻觉缓解
- **预训练理论统一**：是否存在一个通用的预训练框架？

---

## 13. 结论

- 预训练方法的演进脉络：从监督到自监督，从单模态到多模态，从特定任务到通用基础模型
- 不同目标函数产生的表征性质不同，决定了下游任务的适配策略
- 预训练的终极目标：通用的、可迁移的、安全的智能表征
