# GoogLeNet (Inception-v1)

## 基本信息

- **论文**: [Going Deeper with Convolutions](https://arxiv.org/abs/1409.4842)
- **作者**: Christian Szegedy et al. (Google)
- **发表**: CVPR 2015

## 创新点

1. **Inception 模块**: 在同一层中使用多尺度卷积核（1×1, 3×3, 5×5）并行提取特征并进行通道拼接
2. **1×1 卷积降维**: 在 3×3 和 5×5 卷积前使用 1×1 卷积大幅降低计算量
3. **辅助分类器**: 在网络中间层添加辅助分类器，缓解梯度消失并起正则化作用
4. **参数效率极高**: 参数量仅为 AlexNet 的 1/12（约 500 万 vs 6000 万），但性能显著更好

## 核心原理

### Inception 模块

Inception 模块的核心思路是：**"让网络自己选择最合适的卷积核大小"**。每个模块包含四个并行分支：

1. 1×1 卷积（捕捉单点特征）
2. 1×1 卷积降维 → 3×3 卷积
3. 1×1 卷积降维 → 5×5 卷积
4. 3×3 最大池化 → 1×1 卷积

四个分支的输出在通道维度拼接，作为下一层的输入。

### 网络结构

- 22 层（含池化层），但参数量远少于 AlexNet（8 层）
- 使用全局平均池化替代最后的全连接层
- 网络末端使用 Dropout 防止过拟合

## 预训练方法

### 核心思想：不要选卷积核大小——让网络在每个位置同时用多种卷积核，然后自己学"用哪个"

VGGNet 走的是"更深更简单"路线，GoogLeNet 走的是"更宽更聪明"路线。Inception 模块的核心洞察：**在同一个位置上，有的特征需要用 1×1 捕捉（局部细节），有的需要用 5×5 捕捉（更大范围的模式）——为什么不把所有这些卷积核并联，让网络自己选择呢？**

> GoogLeNet 的预训练证明了：宽度（多尺度并行）和稀疏连接（1×1 降维）同样能提升性能，且参数效率远高于 VGGNet。500 万参数就达到了 60M 的 AlexNet 三倍以上的性能。

### 训练流水线（Step by Step）

#### Step 1 — Inception 模块的逐层计算

Inception 模块（原始版本）的四个并行分支：

```text
输入: H×W×C

分支 1: ─── 1×1 Conv (C/8) ────→ 输出 1
分支 2: ─── 1×1 Conv (C/16) → 3×3 Conv (C/8) ──→ 输出 2
分支 3: ─── 1×1 Conv (C/16) → 5×5 Conv (C/8) ──→ 输出 3
分支 4: ─── 3×3 MaxPool → 1×1 Conv (C/8) ──→ 输出 4

最终输出: Concat(输出1, 输出2, 输出3, 输出4) → 通道数 = 4×C/8 = C/2
```

**1×1 卷积降维的关键**：

以分支 2 为例：
- **不用降维**：直接做 3×3 卷积 = $C \times 3 \times 3 \times (C/8)$ = $1.125 C^2$ FLOPs
- **用降维**：1×1 计算量 $C \times 1 \times 1 \times (C/16)$ + 3×3 计算量 $(C/16) \times 3 \times 3 \times (C/8)$ ≈ $0.133 C^2$ FLOPs

**节省约 8.5× 计算量**。这种"先压再扩"的策略是 Inception 能在 22 层内保持高效的关键。

> 类比：1×1 卷积就像一个"信息压缩员"——先把 C 个通道的冗余信息压缩到 C/16 个最有用的维度，然后再用 3×3 处理这个压缩后的信息。压缩本身几乎不丢信息，但计算量大减。

#### Step 2 — 辅助分类器：给梯度的"加速通道"

GoogLeNet 有 22 层深（含池化）。在 2014 年，这么深的网络存在梯度消失问题。解决方案：

**两个辅助分类器**（放在 Inception-4a 和 Inception-4d 之后）：

```text
中间层特征 → AvgPool (5×5, stride 3) → 1×1 Conv → FC 1024 → FC 1000 → Softmax
```

**总损失**：

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{main}} + 0.3 \times \mathcal{L}_{\text{aux1}} + 0.3 \times \mathcal{L}_{\text{aux2}}$$

**为什么 0.3 的权重？**
- 辅助分类器的目的是"帮助底层获得梯度"，不是"主分类"
- 权重太高 → 模型会过度适应辅助分类器，偏离主任务
- 权重太低 → 梯度信号太弱，不起作用
- 0.3 是经验最优

> 辅助分类器在推理时被完全移除。它们的唯一目的是"训练时给浅层神经元提供额外的梯度信号"——就像给长跑选手在中途设置补给站，比赛结束后补给站就撤掉了。

#### Step 3 — 全局平均池化替代全连接层

AlexNet 和 VGGNet 在末端使用巨大的全连接层（4096 维），参数量庞大。GoogLeNet 的创新：

**传统**：Conv → Flatten → FC 4096 → FC 4096 → FC 1000

**GoogLeNet**：Conv → **Global Average Pooling** → FC 1000

- Global Average Pooling 将每个通道的 $H \times W$ 特征图平均为一个标量
- 输出是 $1 \times 1 \times C$ 的向量，直接接 FC 1000
- **零额外参数**（vs FC 层数百万参数）

$$y_c = \frac{1}{H \times W} \sum_{i=1}^H \sum_{j=1}^W x_{c,i,j}$$

这个设计不仅减少参数，还具有天然的**空间不变性**——不管物体在图像中偏移了多少位置，平均值不变。

#### Step 4 — 损失函数与优化

**损失**：多类交叉熵（主分类器） + 辅助分类器损失

$$\mathcal{L} = -\sum_{c} y_c \log(p_c^{\text{main}}) - 0.3\sum_{c} y_c \log(p_c^{\text{aux1}}) - 0.3\sum_{c} y_c \log(p_c^{\text{aux2}})$$

**优化器**：SGD + Momentum (0.9)

| 参数 | 值 | 说明 |
|------|-----|------|
| 初始学习率 | — | — |
| 学习率衰减 | 每 8 epoch ×0.96 | 指数衰减 |
| Batch Size | 32 | 分布式训练 |
| 权重衰减 | 4e-5 | — |
| 动量 | 0.9 | — |

**分布式训练**：GoogLeNet 在训练时使用了多 GPU 分布式并行（Data Parallel），每个 GPU 处理 32 张图，这种配置在当时是领先的。

### 为什么 Inception 模块有效？

| 设计选择 | 效果 |
|---------|------|
| 多尺度卷积并行 | 网络可以在不同感受野间自动选择——不需要人工决定"这一层应该用 3×3 还是 5×5" |
| 1×1 降维 | 计算量降低 5-10×，使深层 Inception 模块可行 |
| 分支结构 | 每个 Inception 模块就像一个小型"专家委员会"——不同分支看不同尺度的特征 |
| 全局平均池化 | 消除过参数化（FC 层），降低过拟合风险 |

| 参数 | 值 | 说明 |

| 数据集 | ImageNet ILSVRC 2014 | 120 万图 / 1000 类 |

| Batch Size | 32 | — |
| 权重衰减 | 4e-5 | — |

### 预训练性能

| 模型 | Top-5 错误率 | 参数量 | 深度 |
|------|------------|--------|------|
| AlexNet | 15.3% | 60M | 8 |
| VGGNet (VGG-16) | 8.8% | 138M | 16 |
| **GoogLeNet (单模型)** | **9.2%** | **5M** | 22 |
| GoogLeNet (7 模型集成) | **6.7%** | 35M | 22 |

> GoogLeNet 用 AlexNet 1/12 的参数达到远优于 AlexNet 的性能。宽度（多尺度）策略在不增加参数量的前提下，极大地增强了每层的表达能力。

### 预训练的实用价值

1. **多尺度特征融合的范式开创**：Inception 模块启发了 ResNeXt、SKNet、Res2Net 等多分支设计
2. **1×1 卷积的"降维"技巧**：成为后续 CNN 设计的标准工具
3. **稀疏连接的思想**：通过 1×1 降维实现"计算高效的稀疏连接"
4. **参数效率的终极体现**：5M 参数 vs 60M（AlexNet）vs 138M（VGG）
5. **辅助分类器的训练技巧**：影响后续深度监督、中间监督等训练策略
- **Inception-v3**: 分解 5×5 卷积为两个 3×3 卷积，引入 Label Smoothing
- **Inception-v4 / Inception-ResNet**: 结合残差连接，进一步加深网络
