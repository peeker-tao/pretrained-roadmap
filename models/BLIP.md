# BLIP (Bootstrapping Language-Image Pre-training)

## 基本信息

- **论文**: [BLIP: Bootstrapping Language-Image Pre-training for Unified Vision-Language Understanding and Generation](https://arxiv.org/abs/2201.12086)
- **作者**: Junnan Li et al. (Salesforce)
- **发表**: ICML 2022

## 创新点

1. **统一理解与生成**: 编码器-解码器架构同时支持理解和生成任务
2. **CapFilt**: 利用生成和过滤机制提高数据质量
3. **多目标联合训练**: 对比损失 + 匹配损失 + 语言模型损失

## 核心原理

### 多任务架构

BLIP 使用编码器-解码器架构，支持三种训练模式：
1. **对比学习 (ITC)**: 图文对比，对齐视觉和语言表征
2. **图文匹配 (ITM)**: 二分类任务，判断图文是否匹配
3. **语言模型 (LM)**: 以图像为条件的文本生成

### CapFilt

1. **Captioner (生成器)**: 为噪声图文对重新生成描述
2. **Filter (过滤器)**: 判断生成/原始描述是否与图像匹配
3. 保留匹配的描述作为高质量训练数据

## 预训练方法

### 核心思想：三个任务一起训练，理解与生成兼得

之前的视觉-语言模型（如 CLIP）往往"只理解不生成"（只能判断图文是否匹配），或者"只生成不理解"。BLIP 的核心创新是：在同一个框架下**联合训练三个互相补充的目标**——对比学习（对齐图文）、图文匹配（细粒度判断）、语言建模（以图为条件的文本生成）。三者合力，使得 BLIP 既能理解图文关系，又能生成描述。

### 训练流水线（Step by Step）

#### 架构总览

BLIP 的核心是**多模态混合编码器-解码器（MED）**，它包含四个组件：

| 组件 | 功能 | 使用场景 |
|------|------|---------|
| 视觉编码器（ViT） | 将图像编码为潜表征 | 所有任务 |
| 文本编码器（BERT 前 6 层） | 将文本编码为潜表征 | ITC, ITM |
| 图像引导的文本编码器（跨注意力层） | 在多模态融合表示文本 | ITM |
| 图像引导的文本解码器（因果自注意力+跨注意力） | 自回归生成文本 | LM |

共享的参数使得这些任务相互促进——视觉编码器同时服务于理解（ITC/ITM）和生成（LM）。

#### Task 1 — 图文对比学习（ITC: Image-Text Contrastive）

与 CLIP 的思路一致：拉近匹配的图文对，推开不匹配的。

- 对图像和文本分别编码
- 计算对比损失 $\mathcal{L}_{\text{ITC}}$（InfoNCE 的对称形式）
- **动量编码器**：为文本和图像各维护一个动量更新的编码器，用于生成更稳定的负例特征并扩大负例队列

> ITC 负责**粗粒度的图文对齐**——"这张图是不是和这段文字描述的是同一个东西？"

#### Task 2 — 图文匹配（ITM: Image-Text Matching）

ITC 是粗粒度的"是不是一对"，ITM 是细粒度的"哪里匹配、哪里不匹配"。

- 将图像特征和文本特征通过**跨注意力层**融合
- 输出一个二分类判断：这对图文是否匹配
- 使用**困难负例挖掘**：选取 ITC 分数高但实际不匹配的样本作为负例

> ITM 负责**细粒度的图文匹配**——"文字说'一只白猫坐在红沙发上'，但图中沙发是蓝色的——不匹配。"

#### Task 3 — 语言建模（LM: Language Modeling）

以图像为条件，自回归生成文本描述——这是 BLIP 的"生成"能力来源。

- 解码器以图像特征为交叉注意力条件
- 自回归预测文本的下一个 token
- 交叉熵损失 $\mathcal{L}_{\text{LM}}$

> LM 让 BLIP 学会**看图说话**——不仅理解图文关系，还能生成流畅的描述。

#### 联合训练

三个任务交替训练（不是并行），不同 batch 使用不同的训练目标。

### CapFilt：用模型自己提升数据质量

预训练图文数据通常来自网络爬取，包含大量**噪声**——文字与图像不完全匹配。BLIP 提出了 **CapFilt（Captioning and Filtering）** 方法，用模型自身来"清理"训练数据：

**步骤 1 — Captioner 生成**：
用预训练后的 BLIP 为网络图片生成新的文字描述。

**步骤 2 — Filter 过滤**：
用 BLIP 的 ITM 头判断原始描述和生成描述是否与图片匹配，移除不匹配的。

**步骤 3 — 重新训练**：
用清洗后的数据重新训练 BLIP。

> 这个"自举"过程类似于教师-学生蒸馏的变种——模型用自己的判断来净化自己的训练数据，在噪声数据上实现了更好的预训练。

### 为什么 BLIP 的设计有效？

#### CLIP 的局限性

CLIP 只能做对比检索，不能做生成——因为它的语言模型是浅层编码而非解码。BLIP 填补了这一空白。

#### 三任务互补

| 任务 | 学什么 | 互补性 |
|------|--------|--------|
| ITC | 全局图文对齐 | 粗粒度检索 |
| ITM | 细粒度匹配 | 补充 ITC 的模糊判断 |
| LM | 文本生成 | 让模型具备"说"的能力 |

三者共同训练使得视觉编码器既有助于理解（ITC+ITM）又有助于生成（LM），实现了**统一的视觉-语言表征**。

### 详细训练配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 视觉编码器 | ViT-B/16 或 ViT-L/16 | 初始化自 ImageNet 预训练权重 |
| 文本编码器/解码器 | BERT-base（123M） | 前 6 层编码 + 后 6 层解码 |
| 跨注意力层 | 随机初始化 | 6 层 for ITM, 6 层(因果) for LM |
| 优化器 | AdamW | β₁=0.9, β₂=0.98 |
| 学习率 | 1e-4（图像）/ 5e-5（文本） | 图像编码器解冻更早 |
| 学习率调度 | 余弦衰减 | — |
| Warmup | 2000 步 | — |
| 权重衰减 | 0.05 | — |
| Batch Size | 2048（ITC）/ 512（ITM+LM） | — |
| Epoch | 20 | — |
| 图像分辨率 | 224×224（预训练）/ 384×384（微调） | — |
| ITC 温度 τ | 0.07 | 可学习 |
| 动量编码器 τ_m | 0.995 | — |
| 动量队列大小 | 57600 | 负例队列 |
| CapFilt 迭代 | 1 轮 | 清洗后重新训练 |

### 预训练性能

| 任务 | BLIP ViT-B | BLIP ViT-L | 当时 SOTA |
|------|-----------|-----------|-----------|
| 图文检索（Flickr30K, R@1） | 88.0% | 90.0% | 87.5% (ALBEF) |
| 图像描述（COCO, CIDEr） | 133.3 | 136.7 | 134.5 (LEMON) |
| VQA | 77.5% | 78.3% | 76.4% (SimVLM) |
| NLVR² | 82.3% | 84.0% | 79.9% (VinVL) |

### BLIP-2：更进一步的轻量化

BLIP-2 使用了一个关键的简化技巧——**Q-Former（Querying Transformer）**：

- 不直接做端到端的多模态融合
- 用一个轻量的 Q-Former 将冻结的视觉编码器和冻结的 LLM 连接起来
- 计算量大幅减少，同时支持更强的 LLM（如 FlanT5、OPT）

### 预训练的迁移价值

1. **统一理解与生成**：第一个真正统一的视觉-语言预训练框架
2. **CapFilt**：噪声数据自净化的有效方法，适用于任何需要从噪声数据中预训练的场景
3. **多任务互补训练**：ITC + ITM + LM 的联合训练启发了后续的许多多模态工作
4. **BLIP-2 的 Q-Former** 为解决"视觉模型太大 + LLM 太大"的融合问题提供了优雅方案
