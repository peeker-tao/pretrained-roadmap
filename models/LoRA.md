# LoRA (Low-Rank Adaptation)

## 基本信息

- **论文**: [LoRA: Low-Rank Adaptation of Large Language Models](https://arxiv.org/abs/2106.09685)
- **作者**: Edward Hu et al. (Microsoft)
- **发表**: ICLR 2022

## 创新点

1. **低秩分解微调**: 将权重的更新量分解为两个低秩矩阵的乘积
2. **冻结原模型**: 仅训练低秩矩阵，原始权重保持不变
3. **参数极其高效**: 仅需 0.1-1% 的参数量即可达到全量微调性能

## 核心原理

### Low-Rank Decomposition

$$W' = W + \Delta W = W + BA$$

其中 $W \in \mathbb{R}^{d \times k}$, $B \in \mathbb{R}^{d \times r}$, $A \in \mathbb{R}^{r \times k}$, $r \ll \min(d, k)$

- $W$: 预训练的原始权重（冻结）
- $BA$: 低秩更新矩阵（仅训练）
- $r$: 秩（通常 1-64）

### 应用方式

LoRA 通常应用于注意力层的 Query 和 Value 投影矩阵。

## 预训练适配方法

### 核心思想：预训练模型的权重矩阵是"过参数化"的——微调只需要在低秩子空间中操作

LoRA（Low-Rank Adaptation）本身不是预训练方法，但它彻底改变了**预训练模型的使用方式**。它的核心洞察是：当一个大模型适配到下游任务时，权重变化的"有效维度"远低于参数总量——换句话说，$\Delta W$ 是**低秩**的。

> LoRA 的数学直觉：预训练模型在任务适配时，不需要修改所有参数。你只需要在特定的低维子空间中"推一推"，就足以让模型适应新任务。这就像一辆方向已经调好的车（预训练），你只需要微调方向盘（低秩适配）就能到达目的地——不需要重建整辆车。

### 低秩适配的数学原理

#### 权重更新分解

传统微调：

$$h = W x \quad \rightarrow \quad h = (W + \Delta W) x$$

LoRA 微调：

$$h = W x + BA x = W x + B A x$$

其中：
- $W \in \mathbb{R}^{d \times k}$：预训练权重（**冻结**）
- $B \in \mathbb{R}^{d \times r}$：低秩矩阵（**可训练**，随机初始化）
- $A \in \mathbb{R}^{r \times k}$：低秩矩阵（**可训练**，零初始化 / Kaiming 初始化）
- $r \ll \min(d, k)$：秩（通常 4-64）

#### 为什么零初始化 B？

如果初始时 $BA = 0$（即 $\Delta W = 0$），则 LoRA 适配器在开始时不会干扰预训练模型的行为——然后逐步学习任务特定的偏移量。

通常 $A$ 使用 Kaiming 初始化，$B$ 使用零初始化（或反向）。

#### 为什么低秩就够？

预训练模型已经包含了丰富的通用知识。任务适配只需要在**低秩子空间**中进行微调——这个子空间对应的是"如何调整通用知识以适应具体任务"的少数自由度。

经验上，$r=8$ 甚至 $r=4$ 就足以达到与全量微调相当的性能——这意味着 $\Delta W$ 的有效秩确实非常低。

### LoRA 训练流程

#### Step 1 — 确定目标模块

LoRA 最常应用于：

| 模块 | 作用 |
|------|------|
| Q 投影 ($W_q$) | 问题/查询相关的适配 |
| V 投影 ($W_v$) | 信息传递相关的适配 |
| K 投影 ($W_k$) | 注意力焦点相关的适配（可选） |
| O 投影 ($W_o$) | 输出相关的适配（可选） |

> 实验发现：仅对 $W_q$ 和 $W_v$ 应用 LoRA 通常就足够——这两个矩阵控制着注意力的"问什么"和"传什么"。

#### Step 2 — 正向传播

```text
输入 x → 原始权重 W (冻结) → 原始输出
      → LoRA B (r dims) → LoRA A (原dims) → LoRA 输出
                                              ↓
                                  最终输出 = 原始 + α/r · LoRA
```

**缩放因子** $\alpha/r$：
- $r$ 越大 → 每个维度的贡献越小（归一化效果）
- $\alpha$ 可以作为超参数调整适配"强度"
- 常见设置：$r=8, \alpha=16$（即 ×2 缩放）

#### Step 3 — 反向传播

仅计算 $B$ 和 $A$ 的梯度，$W$ 不参与反向传播——这极大减少了显存消耗。

#### Step 4 — 推理时合并

推理时可以将 LoRA 权重合并到原始权重中：

$$W_{\text{merged}} = W + BA$$

这样推理时没有任何额外计算开销——合并后的权重与原始模型完全相同，但行为已经适配。

### LoRA 的参数效率

| 模型 | 全量微调参数量 | LoRA (r=8, Q+V) | 压缩比 |
|------|-------------|-----------------|--------|
| GPT-3 175B | 175B | 4.7M | **37,000×** |
| LLaMA-7B | 7B | 4.2M | **1,667×** |
| LLaMA-65B | 65B | 16.2M | **4,012×** |

> 每个任务的 LoRA 权重仅 10-20MB；全量微调需要存储整个 7-65B 模型（14-130GB）。

### LoRA 的训练配置

| 参数 | 典型值 | 说明 |
|------|--------|------|
| 秩 $r$ | 4-64 | $r=8$ 为默认推荐 |
| $\alpha$ | 16-32 | 适配强度 |
| 目标模块 | q_proj, v_proj | 最常用 |
| Dropout | 0-0.1 | 防止过拟合 |
| 优化器 | AdamW | — |
| 学习率 | 1e-4 ~ 5e-4 | 通常高于全量微调 |
| 学习率调度 | 线性/余弦衰减 | — |
| Batch Size | 取决于任务 | — |

### 预训练的实用价值

1. **微调成本革命**：全量微调 GPT-3 175B 需要数月和大量 GPU；LoRA 在数小时、单卡完成
2. **多任务部署**：每个任务只保存 10MB LoRA 权重——无需为每个任务存储完整的 175B 模型
3. **无推理延迟**：合并后的权重与原模型相同，推理速度不受影响
4. **成为 PEFT 标准**：几乎所有大模型微调框架（HuggingFace PEFT, vLLM 等）都支持 LoRA
5. **启发了 QLoRA、DoRA、LoRA+、LongLoRA 等数十种变体**
