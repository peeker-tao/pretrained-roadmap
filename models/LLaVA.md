# LLaVA (Large Language and Vision Assistant)

## 基本信息

- **论文**: [LLaVA: Visual Instruction Tuning](https://arxiv.org/abs/2304.08485)
- **作者**: Haotian Liu et al. (UW-Madison)
- **发表**: NeurIPS 2023

## 创新点

1. **极简架构**: CLIP 视觉编码器 + 线性投影层 + Vicuna/LLaMA 语言模型
2. **Visual Instruction Tuning**: 生成多模态指令跟随数据
3. **LLaVA-1.5**: MLP 投影 + 多尺度输入 + 改进训练策略

## 核心原理

### LLaVA 架构

1. **视觉编码器**: 预训练的 CLIP ViT-L/14
2. **投影层**: 简单的线性投影（LLaVA-1.5 改为 MLP 投影）
3. **语言模型**: Vicuna / LLaMA

### Visual Instruction Tuning

1. 使用 GPT-4 或人工标注生成图文指令数据
2. 输入格式：`"Human: <image>\n{question}\nAssistant: {answer}"`
3. 训练时仅微调投影层和语言模型

## 预训练方法

### 核心思想：用"胶水层"粘合两个冻结的专家

LLaVA 是多模态大模型的"极简主义宣言"——它的架构优雅到只有三个组件：一个**冷冻的 CLIP 视觉编码器**（负责"看"）、一个**可训练的投影层**（负责"翻译"视觉信息给语言模型）、一个**微调中的 Vicuna/LLaMA 语言模型**（负责"说"）。

> LLaVA 的设计哲学：视觉和语言各自有最好的预训练模型（CLIP 和 Vicuna），不需要重新训练它们——只需要一个"适配器"让视觉信息能被语言模型理解。这个适配器在 LLaVA-1.0 中只是一个线性层，在 LLaVA-1.5 中升级为两层 MLP。

### 两阶段训练流水线

#### 第一阶段：特征对齐预训练（Projection Alignment）

**目标**：让投影层学会把 CLIP 的视觉特征"翻译"成 LLM 的 token 嵌入空间中的向量。

**训练设置**：
- **数据**：CC3M（Conceptual Captions 3M，约 300 万图文对）
- **训练内容**：仅训练投影层，CLIP ViT 和 LLM **完全冻结**
- **任务**：给定图像，生成对应的描述文本

**为什么只训练投影层？**
- 这个阶段的目标不是"做任务"，而是"打通视觉到语言的通道"
- CLIP 的视觉特征和 Vicuna 的嵌入空间是完全不同的"语言"——投影层就是翻译器
- 冻结 CLIP 和 LLM 确保不破坏它们各自的预训练知识

$$\mathcal{L}_{\text{Stage1}} = -\sum_{t} \log P(y_t | y_{<t}, \text{Proj}(\text{CLIP}(I)))$$

**LLaVA-1.0**：投影层 = 单个线性矩阵 $W \in \mathbb{R}^{d_v \times d_t}$（CLIP ViT-L/14 的 1024 维 → LLM 的嵌入维度）

**LLaVA-1.5 改进**：投影层 = 两层 MLP + GELU 激活

$$v_{\text{LLM}} = W_2 \cdot \text{GELU}(W_1 \cdot v_{\text{CLIP}})$$

两层 MLP 提升了视觉特征的表达能力——CLIP 的 1024 维特征通过扩展→压缩学习到更丰富的跨模态映射。

#### 第二阶段：视觉指令微调（Visual Instruction Tuning）

**目标**：让模型学会遵循包含图像的指令。

**数据构建（LLaVA 的核心创新）**：

LLaVA 使用 GPT-4（纯文本）生成多模态指令数据：

1. 取 COCO 数据集的图像 + 标注（物体框、描述）
2. 将标注转化为纯文本上下文（如"The image contains a cat, a table..."）
3. 让 GPT-4 根据这些上下文生成对话/描述/复杂推理问题
4. 构造 158K 条多模态指令数据

三类数据：
| 数据类型 | 数量 | 示例 |
|---------|------|------|
| 对话 | 58K | "图中的猫是什么颜色？""橘色。"|
| 详细描述 | 23K | "这张照片展示了..."|
| 复杂推理 | 77K | "图中的人为什么在笑？可能原因是..."|

**训练设置**：
- CLIP ViT 继续冻结
- 投影层 + LLM（Vicuna）**全量微调**
- 损失：仅计算 Assistant 回复部分的交叉熵

> 这个两阶段设计巧妙地将"对齐"和"指令"解耦——第一阶段解决视觉-语言空间对齐，第二阶段专注学会遵循指令。解耦后的训练更稳定、更高效。

### 为什么 LLaVA 如此高效？

#### LLaVA vs Flamingo vs BLIP-2

| 维度 | Flamingo | BLIP-2 | LLaVA |
|------|---------|--------|-------|
| 桥接模块 | Gated XATTN（84 层交叉注意力） | Q-Former（188M） | 线性/MLP 投影 |
| 可训练参数 | ~5B | 188M | ~50M（投影层） |
| LLM 微调 | 否 | 可选 | 是（SFT） |
| 训练数据 | 1.8B 图文对 | ~200M 图文对 | 3M + 158K |
| 训练GPU | 1500 TPUv4 | 16 A100 | 8 A100 |
| 架构复杂度 | 高 | 中 | **极低** |

LLaVA 成功的关键是：**它不试图教 CLIP 和 LLM 如何"深度融合"——它只是教会 LLM"CLIP 的视觉特征应该被视为一种外语 token"**，然后 LLM 自己就能理解。

### LLaVA-1.5 的关键改进

| 改进 | LLaVA-1.0 | LLaVA-1.5 |
|------|----------|----------|
| 投影层 | 线性 | **MLP (2层)** |
| 输入分辨率 | 224×224 | **336×336** |
| 视觉编码器 | CLIP ViT-L/14 | **CLIP ViT-L/14 @336** |
| 训练数据 | 158K LLaVA | **+ShareGPT +VQA +OCR +区域** |
| 数据格式 | 固定 prompt | **统一格式** |

LLaVA-1.5 在高分辨率输入下使用 **anyres** 策略：将图像分割为多个 patch，每个 patch 单独编码后拼接——类似 ViT 的 patchfy，但以更大粒度。

### 详细训练配置

| 参数 | 第一阶段 | 第二阶段 |
|------|---------|---------|
| 数据 | CC3M (3M) | LLaVA-Instruct (158K) |
| 可训练 | 投影层 | 投影层 + LLM |
| 冻结 | CLIP + LLM | CLIP |
| 优化器 | AdamW | AdamW |
| 学习率 | 2e-3 | 2e-5 |
| 学习率调度 | 余弦衰减 | 余弦衰减 |
| Batch Size | 256 | 128 |
| Epoch | 1 | 3 |
| 训练硬件 | 8×A100 | 8×A100 |
| 训练时间 | ~4 小时 | ~10 小时 |

### 预训练的实用价值

1. **架构极简主义**：证明了"投影层"足以桥接视觉和语言——不需要复杂的 Q-Former 或交叉注意力
2. **GPT-4 数据蒸馏**：开创了用 LLM 生成训练数据的范式，被后续大量工作借鉴
3. **多模态 LLM 的民主化**：8 块 A100 就能训练的多模态模型，极大降低了研究门槛
4. **统一了视觉理解任务**：LLaVA 用"对话格式"统一了 VQA、描述、OCR、推理等各种视觉任务
5. **LLaVA-1.5/Next/NeXT 等衍生系列的起点**
