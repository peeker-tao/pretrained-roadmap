# DALL·E

## 基本信息

- **论文**: [Zero-Shot Text-to-Image Generation](https://arxiv.org/abs/2102.12092)
- **作者**: Aditya Ramesh et al. (OpenAI)
- **发表**: ICML 2021

## 创新点

1. **文本到图像生成**: 从文本描述直接生成图像
2. **VQ-VAE + Transformer**: 将图像离散化为 token 序列，用自回归 Transformer 生成
3. **零样本生成**: 无需微调即可生成训练数据中不存在的概念组合

## 核心原理

### 两阶段框架

**阶段 1: dVAE (Discrete VAE)**
- 将 256×256 图像压缩为 32×32 的离散 token 网格
- 码本大小: 8192

**阶段 2: Transformer**
- 自回归预测图像 token 序列
- 以文本 token 为条件（BPE 编码）

### DALL·E 2 的改进

- 使用 CLIP 潜空间 + 扩散模型
- 生成质量大幅提升

### DALL·E 3 的改进

- 使用 ChatGPT 增强文本 prompt
- 更好的文字渲染能力

## 预训练方法

### 核心思想：像写作文一样"画"图

DALL·E 的本质思路是：如果一段文字可以被 token 化成一个"词序列"，一张图片是否也可以被 token 化成一个"视觉词序列"？如果可以，那么生成图片就可以像 GPT 写文章一样——**从左到右、从上到下，逐个"视觉词"地生成**。

> DALL·E = VQ-VAE（将图像压缩为离散 token）+ 自回归 Transformer（以文本为条件的图像生成）。它把"写文字"变成了"画像素"。

### 训练流水线（Step by Step）

#### 第一阶段：dVAE（离散变分自编码器）训练

这阶段的目标是**学习一个"视觉词汇表"**，将连续图像像素转化为离散 token 序列。

**编码（图像 → 离散 token）**：

1. 输入图像 $256 \times 256 \times 3$ 通过 dVAE 编码器（基于 ResNet 的 CNN）
2. 编码器输出特征图：$32 \times 32 \times d$（1024 个潜向量）
3. 每个潜向量通过**向量量化（VQ）**，映射到码本 $\mathcal{E} = \{e_1, e_2, ..., e_{8192}\}$ 中最接近的条目

$$z_q = \arg\min_{e_k \in \mathcal{E}} \| z(x) - e_k \|_2$$

4. 最终得到一个 $32 \times 32 = 1024$ 的离散 token 序列，每个 token 是 1-8192 之间的整数

**解码（离散 token → 图像）**：

1. 码本向量查表：用离散 token 索引从码本中取出对应的向量
2. dVAE 解码器将这些向量重建为 $256 \times 256$ 图像

**训练损失**：

$$\mathcal{L}_{\text{dVAE}} = \|x - \hat{x}\|^2 + \|\text{sg}[z_e(x)] - e\|_2^2 + \beta \|z_e(x) - \text{sg}[e]\|_2^2$$

- 第一项：重建损失（ELBO 中的负对数似然上界）
- 第二项：码本训练（让码本向量向编码器输出靠拢）
- 第三项：承诺损失（让编码器输出向码本向量靠拢）
- $\text{sg}[\cdot]$ 是 stop-gradient 操作，$\beta=0.25$

> 训练完成后，码本的 8192 个条目成为了"视觉词"——任意图像都可以被转化为一串 1024 个"词"的"文字描述"（只是这些"词"不是人类可读的文字，而是 VQ-VAE 自动发现的视觉模式）。

#### 第二阶段：自回归 Transformer 训练

以文本+图像为联合条件，用自回归 Transformer 学习生成图像 token。

**输入构造**：

```text
[BPE(text)] + [图像token₁] + [图像token₂] + ... + [图像token₁₀₂₄]
```

- 文本经过 BPE 编码（16384 词汇量），最长 256 token
- 图像是第一阶段 dVAE 编码的 1024 个离散 token
- 文本和图像 token 共享 Transformer 的参数

**前向传播**：

使用 64 层稀疏 Transformer（sparse attention）。
由于 1024 + 256 = 1280 token 的全注意力计算量太大（$O(1280^2)$），DALL·E 使用了**三种不同的注意力模式**而非标准密集注意力：

| 注意力模式 | 说明 |
|-----------|------|
| Row Attention | 每个 token 只关注同一行（前 32 个位置）的图像 token |
| Column Attention | 每个 token 只关注同一列（前 32 列）的图像 token |
| Full Attention | 文本 token 对所有 token 使用全注意力；自回归 causal 注意力 |

三种注意力在 64 层中以特定模式交替使用，使得全局信息可以在不付出 $O(N^2)$ 代价的情况下传播。

**损失函数**：

$$\mathcal{L} = -\sum_{i=1}^{1024} \log P(\text{img\_token}_i | \text{text\_tokens}, \text{img\_token}_{<i})$$

标准的下一个 token 预测交叉熵（8192 类分类）。

**推理（生成）**：

1. 输入一段文本描述
2. 自回归生成图像 token：逐个位置采样
3. 将生成的 1024 个 token 送入 dVAE 解码器
4. 解码器重建为 $256 \times 256$ 图像

### 为什么这样设计——深入理解

#### 1. 为什么要用 VQ-VAE 做 token 化？

直接生成像素是不现实的——$256 \times 256 \times 3$ 意味着 196608 个值需要生成。VQ-VAE 将 196608 个连续值压缩为 1024 个离散值，压缩比约 192 倍，使得自回归 Transformer 能够处理。

**为什么是 32×32 而不是更小？** 这是一个分辨率 vs 语义的权衡：
- 更大的网格（如 64×64）→ 4096 token → 序列太长，Transformer 无法有效处理
- 更小的网格（如 16×16）→ 256 token → 信息丢失太多，重建质量差
- 32×32 是经验最优

#### 2. 为什么使用稀疏注意力？

64 层 × 1280 长度的全注意力需要 $\approx 1280^2 \times 64 \approx 104M$ 次注意力计算（每次）。稀疏注意力降低了约 $\sqrt{N}$ 倍的复杂度。

#### 3. DALL·E 2 为什么转向扩散模型？

DALL·E（v1）的缺点：
- **生成速度慢**：自回归逐个 token 生成，8192 类 softmax 计算量大
- **低分辨率的分辨**：VQ-VAE 压缩损失导致细节丢失
- **CLIP 引导困难**：文本条件与图像 token 之间的对齐不够直接

DALL·E 2 的改进：
- 使用 **CLIP** 的文本编码和图像编码对齐的潜空间
- 用**扩散模型**在 CLIP 潜空间中生成，然后解码
- 分辨率和细节显著提升

### 预训练的实用价值

1. **开创文本到图像生成**：首次证明自回归 Transformer + VQ-VAE 可以从文本生成高质量图像
2. **零样本组合能力**：无需训练就能生成训练数据中不存在的概念组合（如"鳄梨形状的扶手椅"）
3. **VQ-VAE 的工业级验证**：推动了 VQ-VAE/VQ-GAN 在大规模生成中的应用
4. **为 DALL·E 2/3、Stable Diffusion、Imagen 等铺路**：DALL·E 的范式启发了整个文生图领域
